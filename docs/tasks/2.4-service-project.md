## Описание

Создать сервис бизнес-логики для проектов. Это ключевой сервис Sprint 2 — он оркестрирует создание проекта (с автоматическим созданием команды в одной транзакции), шифрование Git-credentials, проверку владения (ABAC), валидацию входных данных и установку дефолтов пагинации. Репозитории используются как зависимости через интерфейсы (DI).

---

## Интерфейс

```go
type ProjectService interface {
    Create(ctx context.Context, userID uuid.UUID, req dto.CreateProjectRequest) (*models.Project, error)
    GetByID(ctx context.Context, userID uuid.UUID, projectID uuid.UUID) (*models.Project, error)
    List(ctx context.Context, userID uuid.UUID, req dto.ListProjectsRequest) ([]models.Project, int64, error)
    Update(ctx context.Context, userID uuid.UUID, projectID uuid.UUID, req dto.UpdateProjectRequest) (*models.Project, error)
    Delete(ctx context.Context, userID uuid.UUID, projectID uuid.UUID) error
}
```

**Все методы принимают `userID`** — идентификатор текущего пользователя из JWT (через `gin.Context`). Используется для ABAC (пользователь видит/изменяет только свои проекты).

## Методы

### Create

Самый сложный метод — создание проекта **в одной транзакции** с автоматическим созданием команды.

**Шаги:**
1. Валидация DTO (`name` обязателен, `git_provider.IsValid()`, `status.IsValid()`)
2. Если передан `git_credential_id` — проверить, что credential принадлежит пользователю
3. **Транзакция:**
   a. Создать `Project` (с `user_id` из JWT)
   b. Создать `Team` (name: `"Development Team"`, type: `development`, project_id: project.ID)
4. Вернуть созданный проект

**Ошибки:**
- `ErrProjectNameExists` — дубликат имени
- `ErrGitCredentialNotFound` — credential не найден
- `ErrGitCredentialForbidden` — credential принадлежит другому пользователю

```go
func (s *projectService) Create(ctx context.Context, userID uuid.UUID, req dto.CreateProjectRequest) (*models.Project, error) {
    // 1. Валидация
    if req.GitCredentialID != nil {
        cred, err := s.gitCredRepo.GetByID(ctx, *req.GitCredentialID)
        if err != nil { return nil, ErrGitCredentialNotFound }
        if cred.UserID != userID { return nil, ErrGitCredentialForbidden }
    }

    // 2. Транзакция: Project + Team
    var project *models.Project
    err := s.db.WithContext(ctx).Transaction(func(tx *gorm.DB) error {
        project = &models.Project{
            Name: req.Name,
            // ... заполнение полей из DTO ...
            UserID: userID,
        }
        if err := s.projectRepo.CreateWithTx(ctx, tx, project); err != nil {
            return err
        }

        team := &models.Team{
            Name:      "Development Team",
            ProjectID: project.ID,
            Type:      models.TeamTypeDevelopment,
        }
        return s.teamRepo.CreateWithTx(ctx, tx, team)
    })

    return project, err
}
```

**Примечание к транзакциям:** репозиторий должен поддерживать вариант `CreateWithTx(ctx, tx, ...)` или принимать `*gorm.DB` для работы внутри транзакции. Альтернатива — передать `tx` через контекст (pattern `TxFromContext`). Выбор реализации остаётся за разработчиком, но транзакция обязательна.

### GetByID

1. Загрузить проект через `projectRepo.GetByID`
2. **ABAC:** проверить `project.UserID == userID` (или userRole == admin)
3. Вернуть проект (или `ErrProjectForbidden`)

### List

1. Установить дефолты пагинации: `Limit=20` если 0, clamp `Limit` ≤ 100
2. Вызвать `projectRepo.ListByUserID` (обычный пользователь видит только свои)
3. Для admin — вызвать `projectRepo.List` (все проекты)

### Update

1. Загрузить проект через `projectRepo.GetByID`
2. **ABAC:** проверить `project.UserID == userID`
3. Применить изменения из DTO к загруженной модели (partial update pattern: GetByID → изменить поля → Save)
4. Если обновляется `git_credential_id` — проверить владение credential
5. Сохранить через `projectRepo.Update`

### Delete

1. Загрузить проект через `projectRepo.GetByID`
2. **ABAC:** проверить `project.UserID == userID`
3. Удалить через `projectRepo.Delete` (CASCADE удалит teams, tasks, conversations, mcp_server_configs)

## ABAC (Attribute-Based Access Control)

Проверка владения ресурсом выполняется в Service (не в middleware, не в repository):

```go
func (s *projectService) checkOwnership(project *models.Project, userID uuid.UUID) error {
    if project.UserID != userID {
        return ErrProjectForbidden
    }
    return nil
}
```

В будущем этот метод может учитывать роли (admin видит всё) и командный доступ (участники команды видят проект).

## Кастомные ошибки

```go
var (
    ErrProjectNotFound         = errors.New("project not found")
    ErrProjectNameExists       = errors.New("project with this name already exists")
    ErrProjectForbidden        = errors.New("access to project denied")
    ErrGitCredentialNotFound   = errors.New("git credential not found")
    ErrGitCredentialForbidden  = errors.New("git credential belongs to another user")
)
```

Маппинг ошибок из repository:
- `repository.ErrProjectNotFound` → `ErrProjectNotFound`
- `repository.ErrProjectNameExists` → `ErrProjectNameExists`
- `repository.ErrGitCredentialNotFound` → `ErrGitCredentialNotFound`

## Валидация и дефолты

| Поле | Валидация (Service) | Дефолт |
|------|---------------------|--------|
| `name` | Обязательно, непустое | — |
| `git_provider` | `IsValid()` | `local` |
| `status` | `IsValid()` | `active` |
| `git_credential_id` | Если задан — проверить владение | `nil` |
| `git_default_branch` | — | `main` |
| `Limit` (List) | 0 → 20, clamp ≤ 100 | 20 |
| `Offset` (List) | ≥ 0 | 0 |

## Конвенции (по существующему коду)

1. **Интерфейс + приватная struct** — как `AuthService`, `PromptService`
2. **DI через конструктор** — все зависимости инжектируются
3. **Кастомные sentinel errors** — маппинг из repository-ошибок
4. **Partial update** — GetByID → изменить поля → Save (как `PromptService.Update`)
5. **DTO на входе** — Service принимает DTO, не сырые модели (как `PromptService.Create`)
6. **context.Context** — во всех методах
