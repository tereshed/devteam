**Sprint:** 1 — Новые модели данных и миграции
**Файл миграции:** `backend/db/migrations/019_create_conversations.sql`
**Go-модели:** `backend/internal/models/conversation.go`
**Статус:** ✅ Выполнена

---

## Описание

Создать две таблицы в одной миграции: `conversations` (чаты пользователя с системой) и `conversation_messages` (сообщения внутри чата). Conversation — основной интерфейс взаимодействия пользователя с DevTeam: пользователь описывает идею, оркестратор отвечает планом, в чат приходят live-статусы задач. Каждое сообщение может быть связано с задачами, созданными из него.

---

## Схема связей

```
conversations
  ├──> projects (FK: project_id, NOT NULL, ON DELETE CASCADE)
  └──> users (FK: user_id, NOT NULL, ON DELETE CASCADE)

conversation_messages
  ├──> conversations (FK: conversation_id, NOT NULL, ON DELETE CASCADE)
  └──  tasks (через linked_task_ids UUID[], без FK — массив ссылок)
```

## Решение по `linked_task_ids`

Используем PostgreSQL-массив `UUID[]` вместо отдельной junction-таблицы. Причины:
- Связь read-heavy: задачи создаются один раз при обработке сообщения
- Количество задач на сообщение невелико (1-10)
- GIN-индекс даёт эффективный поиск по содержимому массива (`@>` оператор)
- Проще запросы: не нужен JOIN для получения linked tasks

FK на `tasks.id` не ставим — массив UUID[] не поддерживает FK в PostgreSQL. Валидация в service-слое.

## SQL миграция

### UP

```sql
-- +goose Up
-- +goose StatementBegin

-- === 1. conversations ===
CREATE TABLE conversations (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id  UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    title       VARCHAR(500) NOT NULL DEFAULT '',
    status      VARCHAR(50) NOT NULL DEFAULT 'active',
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at  TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

    CONSTRAINT chk_conversations_status CHECK (status IN ('active', 'completed', 'archived'))
);

CREATE INDEX idx_conversations_project_id ON conversations(project_id);
CREATE INDEX idx_conversations_user_id ON conversations(user_id);
CREATE INDEX idx_conversations_project_status ON conversations(project_id, status);

-- === 2. conversation_messages ===
CREATE TABLE conversation_messages (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    conversation_id   UUID NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
    role              VARCHAR(50) NOT NULL,
    content           TEXT NOT NULL,
    linked_task_ids   UUID[] NOT NULL DEFAULT '{}',
    metadata          JSONB NOT NULL DEFAULT '{}',
    created_at        TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

    CONSTRAINT chk_conv_messages_role CHECK (role IN ('user', 'assistant', 'system'))
);

CREATE INDEX idx_conv_messages_conversation_id ON conversation_messages(conversation_id);
CREATE INDEX idx_conv_messages_conv_created ON conversation_messages(conversation_id, created_at);
CREATE INDEX idx_conv_messages_role ON conversation_messages(role);
CREATE INDEX idx_conv_messages_linked_tasks ON conversation_messages USING GIN (linked_task_ids);

-- +goose StatementEnd
```

### DOWN

```sql
-- +goose Down
-- +goose StatementBegin

DROP INDEX IF EXISTS idx_conv_messages_linked_tasks;
DROP INDEX IF EXISTS idx_conv_messages_role;
DROP INDEX IF EXISTS idx_conv_messages_conv_created;
DROP INDEX IF EXISTS idx_conv_messages_conversation_id;
DROP TABLE IF EXISTS conversation_messages;

DROP INDEX IF EXISTS idx_conversations_project_status;
DROP INDEX IF EXISTS idx_conversations_user_id;
DROP INDEX IF EXISTS idx_conversations_project_id;
DROP TABLE IF EXISTS conversations;

-- +goose StatementEnd
```

## Go-модели

```go
package models

type ConversationStatus string

const (
    ConversationStatusActive    ConversationStatus = "active"
    ConversationStatusCompleted ConversationStatus = "completed"
    ConversationStatusArchived  ConversationStatus = "archived"
)

type Conversation struct {
    ID        uuid.UUID          `gorm:"..." json:"id"`
    ProjectID uuid.UUID          `gorm:"..." json:"project_id"`
    Project   *Project           `gorm:"foreignKey:ProjectID" json:"project,omitempty"`
    UserID    uuid.UUID          `gorm:"..." json:"user_id"`
    User      *User              `gorm:"foreignKey:UserID" json:"user,omitempty"`
    Title     string             `gorm:"..." json:"title"`
    Status    ConversationStatus `gorm:"..." json:"status"`
    Messages  []ConversationMessage `gorm:"foreignKey:ConversationID" json:"messages,omitempty"`
    CreatedAt time.Time          `gorm:"..." json:"created_at"`
    UpdatedAt time.Time          `gorm:"..." json:"updated_at"`
}

type ConversationRole string

const (
    ConversationRoleUser      ConversationRole = "user"
    ConversationRoleAssistant ConversationRole = "assistant"
    ConversationRoleSystem    ConversationRole = "system"
)

type ConversationMessage struct {
    ID             uuid.UUID        `gorm:"..." json:"id"`
    ConversationID uuid.UUID        `gorm:"..." json:"conversation_id"`
    Conversation   *Conversation    `gorm:"foreignKey:ConversationID" json:"conversation,omitempty"`
    Role           ConversationRole `gorm:"..." json:"role"`
    Content        string           `gorm:"..." json:"content"`
    LinkedTaskIDs  UUIDSlice        `gorm:"type:uuid[]" json:"linked_task_ids"` // UUIDSlice + pq.Array
    Metadata       datatypes.JSON   `gorm:"..." json:"metadata"`
    CreatedAt      time.Time        `gorm:"..." json:"created_at"`
}
```

**Зависимость:** `github.com/lib/pq` для `pq.Array` (маппинг PostgreSQL `UUID[]` в Go через тип `UUIDSlice` в `conversation.go`).
