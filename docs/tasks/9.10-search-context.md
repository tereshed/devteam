## Описание

Задача заключается в реализации метода `SearchContext` в `CodeIndexer`, который позволяет выполнять семантический (векторный) поиск по проиндексированному коду проекта. Этот метод является ключевым для предоставления агентам релевантного контекста из кодовой базы проекта на основе их запросов.

Метод уже объявлен в интерфейсе `CodeIndexer` и имеет базовую реализацию в `codeIndexer`. Необходимо убедиться в корректности его работы, настроить параметры релевантности и обеспечить полное покрытие тестами.

---

## Зависимости

- `VectorRepository` — для выполнения гибридного поиска в Weaviate.
- `slog.Logger` — для логирования метрик поиска (latency, количество результатов).

---

## Функциональные требования

### 1. Параметры поиска и валидация
- **Query:** Строка запроса. Ограничение по длине (`MaxSearchQueryLen` = 4096). Пустой запрос возвращает пустой список.
- **Limit:** Количество возвращаемых чанков. По умолчанию 10, максимум 50 (`MaxSearchLimit`).
- **ProjectID:** Обязательный параметр для фильтрации результатов в рамках конкретного проекта.

### 2. Гибридный поиск (Hybrid Search)
- Использовать метод `vectorRepo.Search`, который реализует гибридный поиск Weaviate (векторный + ключевые слова).
- **Alpha:** Установить значение `0.7` (смещение в сторону семантического поиска, но с учетом точных совпадений имен функций/переменных).
- **Category:** Использовать фильтр `category = "project_code"`.

### 3. Фильтрация по релевантности (Certainty)
- Результаты Weaviate содержат `Distance`. Необходимо вычислять `Certainty = 1 - (Distance / 2)`.
- Внедрить порог релевантности `RelevanceThreshold = 0.7`. Чанки с уверенностью ниже этого порога должны отсекаться.

### 4. Маппинг метаданных
- Корректно преобразовывать `vectordb.SearchResult` в `indexer.Chunk`.
- Извлекать из метаданных: `file_path`, `language`, `start_line`, `end_line`, `symbol`, `content_hash`.
- Обрабатывать отсутствие полей в метаданных без паник (graceful degradation).

### 5. Логирование и Наблюдаемость
- Логировать каждый вызов `SearchContext` с указанием:
  - `project_id`
  - `query_len`
  - `latency` (время выполнения запроса к БД)
  - `results_count` (сколько чанков вернули после фильтрации)
  - `total_found` (сколько всего нашел Weaviate)
