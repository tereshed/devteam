## Описание

Создать репозиторий для работы с командами (`teams`). Команда — связующее звено между проектом и агентами (1 проект = 1 команда). Репозиторий проще, чем `ProjectRepository`: нет пагинации (у проекта одна команда), основные операции — создание при создании проекта, получение с Preload агентов, обновление имени.

---

## Интерфейс

```go
type TeamRepository interface {
    Create(ctx context.Context, team *models.Team) error
    GetByID(ctx context.Context, id uuid.UUID) (*models.Team, error)
    GetByProjectID(ctx context.Context, projectID uuid.UUID) (*models.Team, error)
    Update(ctx context.Context, team *models.Team) error
    Delete(ctx context.Context, id uuid.UUID) error
}
```

---

## Методы

### Create

Создаёт команду для проекта. Вызывается автоматически из `ProjectService` при создании проекта (в одной транзакции). Возвращает `ErrTeamAlreadyExists` при попытке создать вторую команду для одного проекта (UNIQUE violation на `project_id`).

| Параметр | Тип | Описание |
|----------|-----|----------|
| `team` | `*models.Team` | Заполненная модель (ProjectID обязателен) |

### GetByID

Получает команду по UUID с Preload агентов. Возвращает `ErrTeamNotFound` при отсутствии.

**Preload:** `Agents` (через обратную связь `Agent.TeamID`), отсортированные по `role ASC`.

### GetByProjectID

Основной метод — получить команду по ID проекта. Используется в `GET /projects/:id/team`.

**Preload:** `Agents` с вложенным Preload `Prompt` (для отображения промпта агента в UI).

### Update

Обновляет команду (`gorm.Save`). Контракт аналогичен `ProjectRepository`: Service передаёт полностью заполненную модель (GetByID → изменить → Update).

Типичный use case: переименование команды через `PUT /projects/:id/team`.

### Delete

Удаление команды по ID. На практике вызывается редко — команда удаляется каскадно при удалении проекта. Агенты, привязанные к команде, получают `team_id = NULL` (ON DELETE SET NULL из миграции 016).

## Кастомные ошибки

```go
var (
    ErrTeamNotFound      = errors.New("team not found")
    ErrTeamAlreadyExists = errors.New("team for this project already exists")
)
```

- `gorm.ErrRecordNotFound` → `ErrTeamNotFound`
- pgErr code `23505` (unique_violation на `project_id`) → `ErrTeamAlreadyExists`

---

## Preload стратегия

| Метод | Preload | Описание |
|-------|---------|----------|
| `GetByID` | `Agents` (order by role) | Список агентов команды |
| `GetByProjectID` | `Agents` → `Prompt` | Агенты с промптами для полного отображения в UI |
| `Create` / `Update` / `Delete` | — | Не нужен |

**Preload Agents через обратный FK:**

У модели `Agent` есть `TeamID *uuid.UUID`, а у `Team` нет поля `Agents []Agent`. Нужно добавить это поле в модель `Team` (или использовать явный Preload с условием).

**Рекомендация:** добавить в `models.Team`:

```go
Agents []Agent `gorm:"foreignKey:TeamID" json:"agents,omitempty"`
```

## Конвенции

1. **Интерфейс + приватная struct** — `NewTeamRepository(db *gorm.DB) TeamRepository`
2. **`r.db.WithContext(ctx)`** — во всех запросах
3. **`fmt.Errorf("failed to ...: %w", err)`** — оборачивание ошибок
4. **Кастомные sentinel errors** — `ErrTeamNotFound`, `ErrTeamAlreadyExists`
5. **Обработка unique_violation** — как `UserRepository` (pgconn.PgError code 23505)
