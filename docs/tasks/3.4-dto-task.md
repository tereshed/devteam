### 2. TaskResponse

Ответ на `GET /tasks/:id`. Полная карточка задачи с вложенными summary.

```go
type TaskResponse struct {
    ID              string              `json:"id"`
    ProjectID       string              `json:"project_id"`
    ParentTaskID    *string             `json:"parent_task_id,omitempty"`
    Title           string              `json:"title"`
    Description     string              `json:"description"`
    Status          string              `json:"status"`
    Priority        string              `json:"priority"`
    AssignedAgent   *AgentSummary       `json:"assigned_agent,omitempty"`
    CreatedByType   string              `json:"created_by_type"`
    CreatedByID     string              `json:"created_by_id"`
    Context         datatypes.JSON      `json:"context" swaggertype:"string"`
    Result          *string             `json:"result,omitempty"`
    Artifacts       datatypes.JSON      `json:"artifacts" swaggertype:"string"`
    BranchName      *string             `json:"branch_name,omitempty"`
    ErrorMessage    *string             `json:"error_message,omitempty"`
    SubTasks        []TaskSummary       `json:"sub_tasks,omitempty"`
    MessageCount    *int64              `json:"message_count,omitempty"`
    StartedAt       *time.Time          `json:"started_at,omitempty"`
    CompletedAt     *time.Time          `json:"completed_at,omitempty"`
    CreatedAt       time.Time           `json:"created_at"`
    UpdatedAt       time.Time           `json:"updated_at"`
}
```

### 6. TaskListItem

Элемент списка — легче, чем полный `TaskResponse` (без sub_tasks, context, artifacts, result).

```go
type TaskListItem struct {
    ID              string        `json:"id"`
    ProjectID       string        `json:"project_id"`
    ParentTaskID    *string       `json:"parent_task_id,omitempty"`
    Title           string        `json:"title"`
    Status          string        `json:"status"`
    Priority        string        `json:"priority"`
    AssignedAgent   *AgentSummary `json:"assigned_agent,omitempty"`
    CreatedByType   string        `json:"created_by_type"`
    CreatedByID     string        `json:"created_by_id"`
    BranchName      *string       `json:"branch_name,omitempty"`
    StartedAt       *time.Time    `json:"started_at,omitempty"`
    CompletedAt     *time.Time    `json:"completed_at,omitempty"`
    CreatedAt       time.Time     `json:"created_at"`
    UpdatedAt       time.Time     `json:"updated_at"`
}
```

**Почему `TaskListItem` отдельно от `TaskResponse`:** В списке не нужны `context` (JSONB, может быть большим), `artifacts`, `result`, `sub_tasks`, `error_message` — это экономит трафик и упрощает UI.

## Конвертеры (model → DTO)

```go
func ToTaskResponse(t *models.Task) TaskResponse
func ToTaskListItem(t *models.Task) TaskListItem
func ToTaskListResponse(tasks []models.Task, total int64, limit, offset int) TaskListResponse
func ToTaskSummary(t *models.Task) TaskSummary
func ToAgentSummary(a *models.Agent) *AgentSummary
func ToTaskMessageResponse(m *models.TaskMessage) TaskMessageResponse
func ToTaskMessageListResponse(msgs []models.TaskMessage, total int64, limit, offset int) TaskMessageListResponse
```

### Нюансы конвертеров

**`ToTaskResponse`:**
- `ParentTaskID` — `*uuid.UUID` → `*string` (nil если нет parent)
- `AssignedAgent` — из Preload `task.AssignedAgent` → `AgentSummary` (nil если не назначен)
- `SubTasks` — из Preload `task.SubTasks` → `[]TaskSummary`
- `MessageCount` — **не заполняется конвертером** (handler запрашивает отдельно через `taskMsgRepo.CountByTaskID` или оставляет nil)

**`ToTaskListItem`:**
- `AssignedAgent` — из Preload (List preload'ит только AssignedAgent, без SubTasks)
- Нет `context`, `artifacts`, `result`, `error_message`, `sub_tasks`

**`ToAgentSummary`:**
- nil-safe: если `agent == nil` → возвращает `nil`
