| Режим | Условие в спеке (строка ниже заполняется в PR) | UI |

| **`openai`** | `OpenAI` | `OpenAI.APIKey` |
| **`anthropic`** | `Anthropic` | `Anthropic.APIKey` |

## SSOT: пустое поле ключа и сброс (🔴 один инвариант)

**Зафиксировано для клиента и DTO (сервер обязан принять ту же семантику):**

| Действие пользователя | Тело **`PATCH`** |
|------------------------|------------------|
| Поле «новый ключ» **пустое** (включая только пробелы после **`trim()`**) при Submit | Ключ **`openai_api_key`** / аналог **для этого провайдера отсутствует** в JSON → сервер **не меняет** сохранённый секрет. |
| Явное действие «Сбросить ключ провайдера X» (кнопка / control, не Submit пустой формы) | В JSON включается булев флаг сброса; **имена полей `*_api_key` и `clear_*_key`** — строго по таблице SSOT в [13.5-backend-user-llm-credentials.md](./13.5-backend-user-llm-credentials.md) (раздел «SSOT: имена полей»). **В том же запросе** не слать новое значение ключа для этого провайдера. |

**Запрещено:** трактовать «пустое поле + Save» как удаление ключа — риск массового wipe при ошибке UX.

## Цель (MVP) в режиме **A**

- Карточка на каждый **`id`** из таблицы провайдеров: **`masked_preview`** / «не задан», поле ввода **нового** ключа (**obscureText: true** по умолчанию), явный **«Сбросить»** → **`clear_*`**, глобальный **Save** (disabled при отсутствии dirty).
- После успешного сохранения: **invalidate** провайдера состояния; **SnackBar** (l10n **без** секрета).
- Ошибки **`PATCH`:** отдельные сообщения для **400 / 401 / 403 / 409 / 502** (**13.6**); **401** → **`errorUnauthorized`**.

## Безопасность и UX (обязательно)

### Платформа и снимки экрана

- **Android:** при отображении экрана с полем ввода секрета (режим **A**) включать **`WindowManager.LayoutParams.FLAG_SECURE`** (через **`MainActivity`** / **`MethodChannel`** / согласованный пакет). Снимать флаг при уходе с экрана. Цель — исключить секреты из системных скриншотов и типичных screen recorders в области окна приложения.
- **iOS:** нет прямого аналога **`FLAG_SECURE`**. Минимум: при **`AppLifecycleState.inactive` / `paused`** накрывать контент **privacy overlay** (непрозрачный плейсхолдер), чтобы миниатюра в multitasking не показывала введённые символы. **Отказ** от overlay допускается **только** с явным **risk acceptance** одной строкой в **`Backend SSOT`** PR и дублем в чек-листе мёржа.

### Поле ввода и состояние

- **`obscureText: true`** по умолчанию; тестом зафиксировано.
- **«Показать ключ»:** удержание видимости **не дольше 3 секунд** с момента включения; таймер **сбрасывается** при повторном нажатии; **немедленное** скрытие при **`FocusNode` lost** поля (срабатывает раньше таймера, если пользователь ушёл с поля). При dispose — отмена таймера + принудительно **`obscureText: true`**.
- **`autofillHints: null`** (или пустой список) для секретных полей; **`enableSuggestions: false`**, **`autocorrect: false`**.
- **Restoration:** **не** задавать **`restorationId`** у **`TextField`** с секретом; экран **не** использовать с **`RestorationMixin`**, сохраняющим текст полей. Если родительский **`MaterialApp`** включает **`restorationScopeId`**, для subtree с секретами — изоляция без восстановления значений (или отключение restoration на этом route — зафиксировать в PR одним способом).
- **`TextEditingController`:** перед **`dispose`** вызвать **`clear()`** и, при необходимости, **`text = ''`** после **`clear()`**, чтобы строка не оставалась в куче дольше нужного.

### Логи и мониторинг

- **Dio:** не логировать body запросов/ответов этого репозитория.
- **Sentry / Crashlytics / аналог:** в репозитории **пока не подключены** (`frontend/pubspec.yaml`). **При появлении:** обязательный **scrub** полей ввода ключей и маршрута **`/settings`** из breadcrumbs / attachments; зафиксировать в чек-листе мёржа PR, подключающего SDK.

### Прочее

- **Clipboard:** не предлагать «скопировать сохранённый ключ» (его нет на клиенте). Копирование только что введённого текста — воле пользователя, без подсказок от продукта.
- **Semantics:** у каждой секции провайдера — **`Semantics`** с **`label`** из l10n (заголовок секции); кнопка показа ключа — **`semanticLabel`** без раскрытия значения. После успешного **Save** — краткий **`SemanticsService.announce`** (или эквивалент для web) **нейтральной** фразой из l10n (**без** ключа).

## UX-инварианты

- **Несохранённые изменения:** при **`PopScope`** (назад / жест) и при наличии **dirty** — **`AlertDialog`** (l10n): «Выйти без сохранения?» — отмена / выйти. Без диалога dirty-сброс **запрещён** молча.
- **Валидация формата ключа на клиенте (MVP):** **не** делаем префикс-локин (**`sk-`**, **`sk-ant-`** и т.д.) — избегаем lock-in и ложных отказов. Невалидный ключ — реакция на **400** с сервера + l10n **`globalSettingsKeyRejected`** (или общий ключ **400** из **13.6**). Фаза 2 (опциональные **подсказки** формата без блокировки Save) — отдельная задача.
