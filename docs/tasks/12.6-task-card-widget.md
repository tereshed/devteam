## Канон (SSOT)

| Тема | Решение |
|------|---------|
| Источник данных | Только **`TaskListItemModel`** (**12.1**). **`TaskCard` не** ходит в репозиторий и **не** читает **`TaskModel`** целиком. |
| Локализация в виджете | Все строки через **`AppLocalizations.of(context)!`** внутри **`build`** (или эквивалент с **`context`**). **Запрещено** требовать **`AppLocalizations`** отдельным **обязательным** параметром публичного **`TaskCard`** — лишний boilerplate на call-site и ломает превью/дашборды без обёртки **l10n**. Тесты: **`MaterialApp`** с **`localizationsDelegates`** / **`supportedLocales`**, **`locale: const Locale('ru')`** — как в остальных widget-тестах фичи tasks (единый выбор локали для детерминированных **`find.text`**). |
| Статус / приоритет | Только API из **`task_status_display.dart`** (**`taskStatusTone`**, **`taskStatusIcon`**, **`taskStatusLabel`**, аналогично приоритет). |
| Строка назначенного агента | Одна строка из ключа **`taskCardAgentLine`**: плейсхолдеры **`{name}`** (из **`assignedAgent!.name`**) и **`{role}`** (из **`taskAgentRoleLabel(l10n, assignedAgent.role)`**). Склейка **`name` + символ + `role` вручную в Dart запрещена** — только **`l10n.taskCardAgentLine(name, role)`** (или сгенерированный геттер с тем же смыслом). Если **`assignedAgent == null`** — только **`taskCardUnassigned`**, без **`taskCardAgentLine`**. |
| Время | Поле **`updatedAt`**, **`.toLocal()`**. Форматирование **только в Dart** через **`intl` `DateFormat`**; в **`.arb`** передаётся **уже готовая** строка времени плейсхолдером **`{time}`** (см. **§ Локализация** — тип плейсхолдера). **Два формата по `dense`:** **`dense == false`** (список \< 600 dp) — **`DateFormat.yMMMd(localeTag).add_jm()`** (как метки времени в **12.5**); **`dense == true`** (Kanban) — **`DateFormat.MMMd(localeTag).add_Hm()`** — короче для колонки **280** dp. **Запрещено** один длинный формат для обоих режимов «ради единообразия» — это ломает компактность Kanban. |
| Навигация | **`TaskCard` не** импортирует **`go_router`**. Родитель передаёт **`onTap`**. |
| Заголовок **`title`** | **`maxLines`:** **`dense ? 3 : 4`**; **`overflow: TextOverflow.ellipsis`**. (Явные числа — SSOT **12.6**; не отсылать исполнителя к **12.4** за этими константами.) |
| Компоновка | Ниже заголовка — чипы статуса и приоритета (**`Wrap`** допустим). Далее **строго по вертикали:** строка **`taskCardAgentLine`** или **`taskCardUnassigned`**, затем строка **`taskCardUpdatedAt`**. |
| Плотность | Параметр **`bool dense = false`**. Call-site: список \< 600 dp — опускают **`dense`** (дефолт); Kanban — **`dense: true`**. |
| Доступность: карточка | При **`onTap != null`** — обёртка с **`Semantics`**, кнопка (**`button: true`**, **`label: task.title`** — совпадает с видимым заголовком). При **`onTap == null`** — без семантики кнопки. |
| Доступность: чипы статуса/приоритета | Декоративные **`Icon`** внутри чипов — **`excludeSemantics: true`**, чтобы скринридер не зачитывал «иконка» отдельно от текста; **либо** один **`Semantics(label: …)`** на весь чип с полной подписью (**`Статус: …`**, **`Приоритет: …`** — через существующие **`taskStatusLabel`** / **`taskPriorityLabel`**). Смешивать «иконка с семантикой + текст» без **`excludeSemantics`** **запрещено**. |
| Ключи виджета для prod | **`ValueKey` / `Key` на корне карточки для prod не требуются** — **`ListView.builder`** / **12.4** корректно сопоставляет элементы без ключа на карточке. **Запрещено** вводить **`ValueKey(task.id)`** «для стабильности списка» без отдельного обоснования (**`ReorderableListView`**, кастомные анимации перестановки — вне **12.6**). |
| Ключи для тестов | При необходимости **`find.byKey`**: в тестовой обёртке задать **`Key`** со строковым значением вида **`'task-card-' + task.id`**, либо **`ValueKey`** с тем же **`String`** id; **либо** поиск по тексту **`title`** / **`Semantics`**. **Не** смешивать тестовую необходимость с обязательным prod-API **`TaskCard`**. |
| Тема | **`ColorScheme`** / **`TextTheme`** из **`Theme.of(context)`**; без хардкода **`Colors.*`** для семантики статуса (цвета чипов — из **`task_status_display`**). |

### Альтернативы (отклонены для 12.6)

## Контракт пропсов

Минимальный публичный API (имена полей можно уточнить в PR, смысл фиксирован):

| Параметр | Тип | Назначение |
|----------|-----|------------|
| **`task`** | **`TaskListItemModel`** | Данные строки. |

| **`dense`** | **`bool dense = false`** | Компактные отступы, **`maxLines`** заголовка **3**, короткий **`DateFormat`** для времени (**§ Канон**). Kanban передаёт **`true`**; список — дефолт. |

## Локализация

Импорт в **`task_card.dart`**: **`package:frontend/l10n/app_localizations.dart`**. После правок **`.arb`**: **`make frontend-codegen`** (порядок: build_runner → **`flutter gen-l10n`**), затем **`make frontend-l10n-check`**.

### Новые ключи (минимум)

| Ключ | Назначение |
|------|------------|
| **`taskCardUnassigned`** | Нет назначенного агента (**`assignedAgent == null`**). |
| **`taskCardAgentLine`** | Одна строка агента; плейсхолдеры **`name`**, **`role`** — оба типа **`String`** в **`@taskCardAgentLine` placeholders** (роль уже локализована вызовом **`taskAgentRoleLabel`** в Dart). Пример **ru:** `"@taskCardAgentLine": { "placeholders": { "name": {}, "role": {} } }`, значение: **`"{name} · {role}"`**. Пример **en:** **`"{name} · {role}"`** (тот же визуальный разделитель; при необходимости другой разделитель — только с продуктовым согласованием и зеркалом **l10n-check**). |
| **`taskCardUpdatedAt`** | Обёртка для уже отформатированного времени. Плейсхолдер **`{time}`** — **исключительно `String`**: в Dart сначала **`DateFormat`…format(updatedAt)`**, затем **`l10n.taskCardUpdatedAt(formattedTime)`**. **Запрещено** объявлять в **`.arb`** ICU-форму **`{time, date, …}`** / **`{time, DateTime, …}`** для этого ключа — форматирование даты/времени остаётся **единственной точкой** в **`task_card.dart`** (аналогично тому, как **`task_status_display`** владеет маппингом статуса). Примеры значений: **ru** — **`"Обновлено: {time}"`**; **en** — **`"Updated: {time}"`**. Метаданные плейсхолдера **`time`**: **`"type": "String"`**. |

Остальные подписи статуса/приоритета — **`taskStatus*`**, **`taskPriority*`**.

## Сценарии widget-тестов (`task_card_test.dart`)

Базовая обёртка: **`MaterialApp`** с **`locale: const Locale('ru')`**, **`AppLocalizations`**-делегаты и **`supportedLocales`** — как в прочих widget-тестах модуля **`features/tasks`**.

- Рендер с **`assignedAgent`** — на экране результат **`taskCardAgentLine(name, role)`** (проверка по **`find.text`**, без хардкода английского при зафиксированной **`Locale('ru')`**).
- Рендер без агента — **`taskCardUnassigned`**.
- **`updatedAt`** — подстрока после **`DateFormat`** при фиксированных **`Locale`** и **`DateTime`** (согласованно с **`dense`**).
- **`dense: true`** vs **`false`** — как минимум: разный **`DateFormat`** (косвенно — разная длина/шаблон подстроки времени) **или** различие **`maxLines`** заголовка при длинном **`title`**.
- **`Semantics`**: при **`onTap`** — кнопка; при **`onTap: null`** — не кнопка.
- Поиск по **`Key('task-card-…')`** — опционально, только если обёртка в тесте задаёт ключ; **не** требование к prod-виджету.
- **`TextScaler.linear(2.0)`** на **`MediaQuery`**: после **`pump`** / **`pumpAndSettle`** — **`expect(tester.takeException(), isNull)`** и отсутствие **`FlutterError`** с **`overflow`** в логе теста; визуально заголовок остаётся с **`ellipsis`** (без требования golden).
