## Описание

Создать репозиторий для CRUD-операций с зашифрованными учётными данными Git (`git_credentials`). Credentials хранят токены, SSH-ключи и OAuth-данные для доступа к GitHub/GitLab/Bitbucket. Поле `encrypted_value` зашифровано на уровне приложения (AES-256-GCM) — репозиторий работает с уже зашифрованными данными (шифрование/расшифровка — ответственность Service-слоя).

---

## Интерфейс

```go
type GitCredentialRepository interface {
    Create(ctx context.Context, cred *models.GitCredential) error
    GetByID(ctx context.Context, id uuid.UUID) (*models.GitCredential, error)
    ListByUserID(ctx context.Context, userID uuid.UUID) ([]models.GitCredential, error)
    ListByUserIDAndProvider(ctx context.Context, userID uuid.UUID, provider models.GitCredentialProvider) ([]models.GitCredential, error)
    Update(ctx context.Context, cred *models.GitCredential) error
    Delete(ctx context.Context, id uuid.UUID) error
}
```

---

## Методы

### Create

Создаёт credential. `encrypted_value` должен быть уже зашифрован Service-слоем.

| Параметр | Тип | Описание |
|----------|-----|----------|
| `cred` | `*models.GitCredential` | Заполненная модель (UserID, Provider, AuthType, EncryptedValue, Label обязательны) |

### GetByID

Получает credential по UUID. Возвращает `ErrGitCredentialNotFound` при отсутствии.

**Важно:** возвращает `EncryptedValue` (байты) — расшифровка в Service. В JSON `EncryptedValue` помечен `json:"-"` и не попадёт в ответ API.

### ListByUserID

Список всех credentials пользователя. Без пагинации — у пользователя обычно 1-10 credentials.

**Важно:** возвращает записи **без** `EncryptedValue` (SELECT без этого поля). Список нужен для UI — показать label, provider, auth_type, но не секреты.

### ListByUserIDAndProvider

Фильтрация по провайдеру (github/gitlab/bitbucket). Для UI: "покажи только GitHub credentials". Также без `EncryptedValue`.

### Update

Обновляет credential (`gorm.Save`). Контракт: Service передаёт полностью заполненную модель (GetByID → изменить → Update). Типичные сценарии: обновить label, ротировать токен (перешифровать `encrypted_value`).

### Delete

Удаление credential. Проекты, использующие этот credential, получат `git_credentials_id = NULL` (ON DELETE SET NULL из миграции 014).

## Кастомные ошибки

```go
var (
    ErrGitCredentialNotFound = errors.New("git credential not found")
)
```

- `gorm.ErrRecordNotFound` → `ErrGitCredentialNotFound`

---

## Безопасность: работа с `EncryptedValue`

| Операция | EncryptedValue |
|----------|----------------|
| `Create` | Записывается (уже зашифрован Service) |
| `GetByID` | Возвращается (для расшифровки в Service) |
| `ListByUserID` | **НЕ возвращается** (SELECT без поля) |
| `ListByUserIDAndProvider` | **НЕ возвращается** |
| `Update` | Перезаписывается (если обновлён) |
| `Delete` | Удаляется вместе с записью |

**Реализация исключения поля из SELECT:**

```go
func (r *gitCredentialRepository) ListByUserID(ctx context.Context, userID uuid.UUID) ([]models.GitCredential, error) {
    var creds []models.GitCredential
    err := r.db.WithContext(ctx).
        Select("id, user_id, provider, auth_type, label, created_at, updated_at").
        Where("user_id = ?", userID).
        Order("created_at DESC").
        Find(&creds).Error
    // ...
}
```

## Конвенции

1. **Интерфейс + приватная struct** — `NewGitCredentialRepository(db *gorm.DB) GitCredentialRepository`
2. **`r.db.WithContext(ctx)`** — во всех запросах
3. **`fmt.Errorf("failed to ...: %w", err)`** — оборачивание ошибок
4. **Кастомные sentinel errors** — `ErrGitCredentialNotFound`
5. **Явный SELECT** — для List-методов (исключение `encrypted_value`)
