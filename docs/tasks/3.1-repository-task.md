# 3.1 Repository: TaskRepository

**Sprint:** 3 — CRUD API для Tasks
**Файл:** `backend/internal/repository/task_repository.go`
**Тест:** `backend/internal/repository/task_repository_test.go` (integration)
**Статус:** ✅ Реализовано

---

## Описание

Создать репозиторий для CRUD-операций с задачами (`tasks`). Задача — основная единица работы в оркестраторе: создаётся оркестратором или пользователем, назначается агенту, проходит через pipeline статусов. Репозиторий сложнее, чем `ProjectRepository`: больше фильтров (status, priority, agent, parent_task, branch), полиморфный `created_by`, Preload для связей (Agent, SubTasks). Это фундамент Sprint 3.

---

## Интерфейс

```go
type TaskRepository interface {
    Create(ctx context.Context, task *models.Task) error
    GetByID(ctx context.Context, id uuid.UUID) (*models.Task, error)
    List(ctx context.Context, filter TaskFilter) ([]models.Task, int64, error)
    Update(ctx context.Context, task *models.Task) error
    Delete(ctx context.Context, id uuid.UUID) error
    CountByProjectID(ctx context.Context, projectID uuid.UUID) (int64, error)
    ListByParentID(ctx context.Context, parentTaskID uuid.UUID) ([]models.Task, error)
}
```

---

## Методы

### Create

Создаёт задачу. `ProjectID`, `Title`, `CreatedByType`, `CreatedByID` — обязательные поля (валидация в Service). По умолчанию `Status = "pending"`, `Priority = "medium"` (дефолты в миграции).

| Параметр | Тип | Описание |
|----------|-----|----------|
| `task` | `*models.Task` | Заполненная модель |

**Обработка ошибок:**
- FK violation на `project_id` → `ErrProjectNotFound` (реимпорт)
- FK violation на `assigned_agent_id` → `ErrAgentNotFound`
- FK violation на `parent_task_id` → `ErrTaskNotFound` (parent не существует)

### GetByID

Получает задачу по UUID с Preload связей. Возвращает `ErrTaskNotFound` при отсутствии.

**Preload стратегия:**
| Связь | Preload | Причина |
|-------|---------|---------|
| `AssignedAgent` | ✅ | UI показывает имя/роль назначенного агента |
| `SubTasks` | ✅ | UI показывает дерево подзадач |
| `ParentTask` | ✅ (shallow) | UI показывает breadcrumb к родителю |
| `Project` | ❌ | Не нужно — project_id достаточно, Project загружается отдельно |

```go
db.Preload("AssignedAgent").
   Preload("SubTasks", func(db *gorm.DB) *gorm.DB {
       return db.Order("created_at ASC")
   }).
   Preload("ParentTask").
   First(&task, "id = ?", id)
```

### List

Основной метод — список задач с фильтрацией и пагинацией. Используется для `GET /projects/:id/tasks` и `GET /tasks` (admin).

**Фильтры** определяются структурой `TaskFilter` (см. ниже).

**Пагинация:** `Count` + `Limit/Offset` — аналогично `ProjectRepository`.

### Update

Обновляет задачу (`gorm.Save`). Контракт с Service аналогичен `ProjectRepository`:
1. `GetByID` — загрузить текущее состояние
2. Изменить нужные поля (status, assigned_agent_id, result, и т.д.)
3. `Update` — сохранить

**Важно:** Service управляет логикой переходов статусов (state machine), репозиторий только сохраняет.

### Delete

Полное удаление задачи. Каскадно удаляются `task_messages` (через ON DELETE CASCADE). Подзадачи (`parent_task_id`) получают `NULL` (ON DELETE SET NULL).

### CountByProjectID

Быстрый подсчёт задач в проекте (для badge в UI / summary).

```go
db.Model(&models.Task{}).Where("project_id = ?", projectID).Count(&count)
```

### ListByParentID

Получить все подзадачи данной задачи. Без пагинации — подзадач обычно немного. Сортировка по `created_at ASC`.

## Фильтры и пагинация

```go
type TaskFilter struct {
    ProjectID       *uuid.UUID          // Обязательный для GET /projects/:id/tasks
    Status          *models.TaskStatus  // Фильтр по статусу
    Statuses        []models.TaskStatus // Фильтр по нескольким статусам (IN)
    Priority        *models.TaskPriority
    AssignedAgentID *uuid.UUID          // Задачи конкретного агента
    CreatedByType   *models.CreatedByType
    CreatedByID     *uuid.UUID          // В паре с CreatedByType
    ParentTaskID    *uuid.UUID          // Подзадачи конкретной задачи
    RootOnly        bool                // Только корневые (parent_task_id IS NULL)
    BranchName      *string             // Поиск по git-ветке
    Search          *string             // Поиск по title/description (ILIKE)
    Limit           int
    Offset          int
    OrderBy         string              // Поле сортировки
    OrderDir        string              // ASC / DESC
}
```

### Фильтры: логика применения

```go
func (r *taskRepository) applyFilters(db *gorm.DB, filter TaskFilter) *gorm.DB {
    if filter.ProjectID != nil {
        db = db.Where("project_id = ?", *filter.ProjectID)
    }
    if filter.Status != nil {
        db = db.Where("status = ?", *filter.Status)
    }
    if len(filter.Statuses) > 0 {
        db = db.Where("status IN ?", filter.Statuses)
    }
    if filter.Priority != nil {
        db = db.Where("priority = ?", *filter.Priority)
    }
    if filter.AssignedAgentID != nil {
        db = db.Where("assigned_agent_id = ?", *filter.AssignedAgentID)
    }
    if filter.CreatedByType != nil && filter.CreatedByID != nil {
        db = db.Where("created_by_type = ? AND created_by_id = ?", *filter.CreatedByType, *filter.CreatedByID)
    }
    if filter.ParentTaskID != nil {
        db = db.Where("parent_task_id = ?", *filter.ParentTaskID)
    }
    if filter.RootOnly {
        db = db.Where("parent_task_id IS NULL")
    }
    if filter.BranchName != nil {
        db = db.Where("branch_name = ?", *filter.BranchName)
    }
    if filter.Search != nil && *filter.Search != "" {
        escaped := escapeILIKEWildcards(*filter.Search)
        pattern := "%" + escaped + "%"
        db = db.Where("(title ILIKE ? ESCAPE '\\' OR description ILIKE ? ESCAPE '\\')", pattern, pattern)
    }
    return db
}
```

### Защита от SQL-инъекций (OrderBy / OrderDir)

Whitelist допустимых колонок — аналогично `ProjectRepository`:

```go
var allowedTaskOrderColumns = map[string]bool{
    "created_at": true,
    "updated_at": true,
    "title":      true,
    "status":     true,
    "priority":   true,
    "started_at": true,
}

func sanitizeTaskOrder(orderBy, orderDir string) string {
    if !allowedTaskOrderColumns[orderBy] {
        orderBy = "created_at"
    }
    if orderDir != "ASC" && orderDir != "asc" {
        orderDir = "DESC"
    }
    return orderBy + " " + orderDir
}
```

### Защита лимита (defense in depth)

```go
func normalizeTaskListLimit(limit int) int {
    if limit <= 0 {
        return 50  // default для задач — 50
    }
    if limit > 200 {
        return 200
    }
    return limit
}
```

## Кастомные ошибки

```go
var (
    ErrTaskNotFound = errors.New("task not found")
)
```

- `gorm.ErrRecordNotFound` → `ErrTaskNotFound`
- FK violation на `project_id` → `ErrProjectNotFound` (из `project_repository.go`)
- FK violation на `assigned_agent_id` → обёрнутая ошибка (или `ErrAgentNotFound` если есть)

**Важно:** `ErrTaskNotFound` будет использоваться и в `TaskMessageRepository` (задача 3.2) для FK violation.

## Preload стратегия

| Метод | Preload | Причина |
|-------|---------|---------|
| `GetByID` | `AssignedAgent`, `SubTasks`, `ParentTask` | Полная карточка задачи в UI |
| `List` | `AssignedAgent` | Список показывает имя агента, но не подзадачи |
| `ListByParentID` | — | Минимальная загрузка для дерева |
| `CountByProjectID` | — | Только count |

## Конвенции

1. **Интерфейс + приватная struct** — `NewTaskRepository(db *gorm.DB) TaskRepository`
2. **`r.db.WithContext(ctx)`** — во всех запросах
3. **`fmt.Errorf("failed to ...: %w", err)`** — оборачивание ошибок
4. **Кастомные sentinel errors** — `ErrTaskNotFound`
5. **Paging: `Count` + `Limit/Offset`** — аналогично `ProjectRepository`
6. **`applyFilters`** — вынести в приватный метод (DRY для List)
7. **`sanitizeTaskOrder`** — whitelist колонок для ORDER BY
8. **`escapeILIKEWildcards`** — реимпорт из `project_repository.go` (или вынести в `pkg/dbutil`)
