### `ConsumerStatefulWidget`

Экран — `ConsumerStatefulWidget`, не `ConsumerWidget`:
- `TextEditingController` требует `dispose()`
- `Timer` дебаунса требует `dispose()`
- `_showClearButton` — локальный UI-стейт (без провайдера)
- `_activeStatusFilter` и `_searchQuery` не нужны глобально

`ProjectCard`, `ProjectStatusChip` — `StatelessWidget`.

### RefreshIndicator и scrollable child

`RefreshIndicator` требует прокручиваемого потомка. Проблема: `Center(child: Column(...))` — не прокручивается, swipe-down не работает ни в error, ни в empty state.

**Решение:** вспомогательный метод `_scrollable()` оборачивает любой контент в `CustomScrollView` с `AlwaysScrollableScrollPhysics`:

```dart
Widget _scrollable(Widget child) => CustomScrollView(
  physics: const AlwaysScrollableScrollPhysics(),
  slivers: [SliverFillRemaining(hasScrollBody: false, child: child)],
);
```

Используется для loading / empty / error. `_ProjectsContent` уже прокручиваемый (ListView/GridView).

### Pull-to-refresh без флика

При `ref.refresh(...)` провайдер уходит в `loading` → если использовать `asyncProjects.when(loading: () => spinner)`, экран мигнёт на спиннер.

**Решение (Riverpod 3):** у `AsyncValue` геттер `value` — при loading/error возвращает предыдущие данные, иначе `null` только при первой загрузке (аналог бывшего `valueOrNull`):

```dart
final asyncProjects = ref.watch(projectListProvider(filter: filter, limit: 50));
final staleData = asyncProjects.value;

// isLoading && staleData != null → refresh в процессе → НЕ показывать спиннер
final isFirstLoad  = asyncProjects.isLoading && staleData == null;
final isFirstError = asyncProjects.hasError  && staleData == null;
```

При refresh: `RefreshIndicator` показывает свой индикатор сверху, список остаётся видимым.

### Freezed `==` — предусловие кэширования

`ProjectListFilter` — Freezed-класс (задача 10.3): Freezed генерирует `==` и `hashCode` по всем полям. Это критично, так как `projectListProvider` — family-провайдер: ключ кэша — сам объект фильтра. Без корректного `==` каждый `build()` создавал бы новый объект → инвалидация кэша → лишние запросы.

### Дебаунс поиска

Кнопка очистки (`_showClearButton`) — немедленно. Фактический поиск (`_searchQuery`) — через 400 мс:

```dart
void _onSearchChanged(String value) {
  setState(() => _showClearButton = value.isNotEmpty);
  _debounce?.cancel();
  _debounce = Timer(const Duration(milliseconds: 400), () {
    if (mounted) setState(() => _searchQuery = value.trim());
  });
}

void _onSearchClear() {
  _searchController.clear();
  _debounce?.cancel();
  setState(() { _showClearButton = false; _searchQuery = ''; });
}
```

### Сброс всех фильтров

Метод `_clearAllFilters()` используется в empty state и может понадобиться в будущем в AppBar:

```dart
void _clearAllFilters() {
  _onSearchClear();
  setState(() => _activeStatusFilter = null);
}
```

### Адаптивная верстка

Breakpoint `600 dp` разделяет `ListView` (mobile) от `GridView` (tablet/desktop). В GridView используется `SliverGridDelegateWithMaxCrossAxisExtent` — число колонок вычисляется автоматически, `childAspectRatio` не нужен:

```
maxCrossAxisExtent: 380 →  600 dp: 2 колонки (600/380 ≈ 1.6 → ceil = 2)
                          1200 dp: 4 колонки ... нет, 1200/380 ≈ 3.2 → ceil = 4
```

Скорректируйте `maxCrossAxisExtent` под дизайн (рекомендуется 360–420). `mainAxisExtent` задаёт фиксированную высоту карточки — не зависит от содержимого.

### Riverpod 3: `ref.invalidate` вместо `ref.refresh`

В Riverpod 3 (`^3.0.3`) для `onRefresh` рекомендуется `ref.invalidate + ref.read(.future)` вместо `ref.refresh(provider.future)`:

```dart
// ✅ Riverpod 3 — рекомендованный паттерн
final currentFilter = _currentFilter; // snapshot до await — защита от гонки
ref.invalidate(projectListProvider(filter: currentFilter, limit: _limit));
await ref.read(projectListProvider(filter: currentFilter, limit: _limit).future)
    .timeout(const Duration(seconds: 30)); // нет зависшего RefreshIndicator при потере сети

// ❌ Устаревший паттерн (Riverpod 2.x)
await ref.refresh(projectListProvider(filter: filter, limit: _limit).future);
```

Снимок `currentFilter` до `await` предотвращает гонку: если пользователь сменил фильтр пока выполнялся refresh, `ref.invalidate` инвалидирует правильный ключ кэша, а не устаревший.

### `ref.listen` — SnackBar при ошибке refresh

При повторной ошибке (данные уже загружены, `prev?.hasValue == true`) экран показывает стейл-данные — пользователь не увидит error state и не узнает об ошибке. Решение — `ref.listen` в начале `build()`:

```dart
ref.listen<AsyncValue<ProjectListResponse>>(
  projectListProvider(filter: filter, limit: _limit),
  (prev, next) {
    if (next.hasError && prev?.hasValue == true) {
      ScaffoldMessenger.of(context).showSnackBar(
        SnackBar(
          content: Text(_errorMessage(l10n, next.error!)),
          action: SnackBarAction(
            label: l10n.retry,
            onPressed: () {
              final f = _currentFilter;
              ref.invalidate(projectListProvider(filter: f, limit: _limit));
            },
          ),
        ),
      );
    }
  },
);
```

### `_lastSeenData` — стейл данные при смене фильтра

`asyncProjects.value` возвращает `null` при первой загрузке нового фильтра (новый ключ кэша → loading state). Без дополнительного кэша экран мигает спиннером при каждой смене фильтра.

**Решение:** поле `_lastSeenData` в State хранит последние известные данные любого ключа:

```dart
ProjectListResponse? _lastSeenData; // поле State — не UI-стейт, без setState

// В build():
final staleData = asyncProjects.value;
if (staleData != null) _lastSeenData = staleData; // обновить кэш без setState
final effectiveData = staleData ?? _lastSeenData;  // стейл при смене фильтра

final isFirstLoad  = asyncProjects.isLoading && effectiveData == null;
final isFirstError = asyncProjects.hasError  && effectiveData == null;
```

**MVP-компромисс:** при смене фильтра кратковременно видны данные предыдущего запроса. UX лучше мигания спиннером.

### Рефакторинг `_buildBody` → три метода

7-параметровый `_buildBody` нарушает rule of three — три семантических метода читаются яснее:

| Метод | Ответственность |
|-------|----------------|
| `_buildLoading()` | Спиннер внутри `_scrollable()` |
| `_buildError(error, l10n, filter)` | Error state + retry |
| `_buildData(response, filter, l10n)` | Список/сетка или empty state |

Тело `build()` сводится к:
```dart
child: isFirstLoad
    ? _buildLoading()
    : isFirstError
        ? _buildError(asyncProjects.error!, l10n, filter)
        : _buildData(effectiveData!, filter, l10n),
```

### `ProjectsListScreen`

```dart
import 'dart:async';
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:go_router/go_router.dart';
import 'package:frontend/l10n/app_localizations.dart';
import 'package:frontend/features/projects/data/project_providers.dart';
import 'package:frontend/features/projects/domain/models.dart';
import 'package:frontend/features/projects/domain/requests.dart';
import 'package:frontend/features/projects/domain/project_exceptions.dart';
import 'package:frontend/features/projects/presentation/widgets/project_card.dart';

class ProjectsListScreen extends ConsumerStatefulWidget {
  const ProjectsListScreen({super.key});

  @override
  ConsumerState<ProjectsListScreen> createState() => _ProjectsListScreenState();
}

class _ProjectsListScreenState extends ConsumerState<ProjectsListScreen> {
  final _searchController = TextEditingController();
  Timer? _debounce;
  String _searchQuery = '';
  String? _activeStatusFilter;
  bool _showClearButton = false;
  ProjectListResponse? _lastSeenData; // кэш последних известных данных любого фильтра

  static const _limit = 50; // пагинация — задача 10.X

  @override
  void dispose() {
    _debounce?.cancel();
    _searchController.dispose();
    super.dispose();
  }

  void _onSearchChanged(String value) {
    setState(() => _showClearButton = value.isNotEmpty);
    _debounce?.cancel();
    _debounce = Timer(const Duration(milliseconds: 400), () {
      if (mounted) setState(() => _searchQuery = value.trim());
    });
  }

  void _onSearchClear() {
    _searchController.clear();
    _debounce?.cancel();
    setState(() { _showClearButton = false; _searchQuery = ''; });
  }

  void _onStatusFilterChanged(String? status) {
    setState(() => _activeStatusFilter = status);
  }

  void _clearAllFilters() {
    _onSearchClear();
    setState(() => _activeStatusFilter = null);
  }

  ProjectListFilter? get _currentFilter {
    final hasSearch = _searchQuery.isNotEmpty;
    final hasStatus = _activeStatusFilter != null;
    if (!hasSearch && !hasStatus) return null;
    return ProjectListFilter(
      search: hasSearch ? _searchQuery : null,
      status: _activeStatusFilter,
    );
  }

  // Любой non-scrollable контент оборачивается в scrollable,
  // чтобы RefreshIndicator мог поймать swipe-down
  Widget _scrollable(Widget child) => CustomScrollView(
    physics: const AlwaysScrollableScrollPhysics(),
    slivers: [SliverFillRemaining(hasScrollBody: false, child: child)],
  );

  Widget _buildLoading() =>
      _scrollable(const Center(child: CircularProgressIndicator()));

  Widget _buildError(Object error, AppLocalizations l10n, ProjectListFilter? filter) =>
      _scrollable(_ErrorState(
        error: error,
        message: _errorMessage(l10n, error),
        retryLabel: l10n.retry,
        onRetry: () {
          if (mounted) {
            ref.invalidate(projectListProvider(filter: filter, limit: _limit));
          }
        },
      ));

  Widget _buildData(
    ProjectListResponse response,
    ProjectListFilter? filter,
    AppLocalizations l10n,
  ) {
    if (response.projects.isEmpty) {
      return _scrollable(_EmptyState(
        hasFilter: filter != null,
        l10n: l10n,
        onClearFilters: filter != null ? _clearAllFilters : null,
        onCreateProject: () => context.push('/projects/new'),
      ));
    }
    return _ProjectsContent(projects: response.projects);
  }

  @override
  Widget build(BuildContext context) {
    final l10n = AppLocalizations.of(context)!;
    final filter = _currentFilter;
    final asyncProjects = ref.watch(
      projectListProvider(filter: filter, limit: _limit),
    );

    // SnackBar при ошибке refresh когда данные уже загружены
    // (prev.hasValue → пользователь видит стейл-данные, error state не показывается)
    ref.listen<AsyncValue<ProjectListResponse>>(
      projectListProvider(filter: filter, limit: _limit),
      (prev, next) {
        if (next.hasError && prev?.hasValue == true) {
          ScaffoldMessenger.of(context).showSnackBar(
            SnackBar(
              content: Text(_errorMessage(l10n, next.error!)),
              action: SnackBarAction(
                label: l10n.retry,
                onPressed: () {
                  final f = _currentFilter;
                  ref.invalidate(projectListProvider(filter: f, limit: _limit));
                },
              ),
            ),
          );
        }
      },
    );

    // _lastSeenData — стейл данные при смене фильтра (нет мигания спиннером)
    // Логируем только тип и стек — $err.toString() для DioException содержит
    // полный URL с токеном и попадает в Crashlytics/Sentry даже в release-сборке.
    final staleData = asyncProjects.value;
    if (staleData != null) _lastSeenData = staleData; // обновить кэш без setState
    final effectiveData = staleData ?? _lastSeenData;  // стейл при смене фильтра

    final isFirstLoad  = asyncProjects.isLoading && effectiveData == null;
    final isFirstError = asyncProjects.hasError  && effectiveData == null;

    if (isFirstError) {
      debugPrint('projectList error runtimeType=${asyncProjects.error?.runtimeType}');
      debugPrint('projectList stackTrace:\n${asyncProjects.stackTrace}');
    }

    return Scaffold(
      appBar: AppBar(
        title: Text(l10n.projectsTitle),
        actions: [
          IconButton(
            icon: const Icon(Icons.add),
            tooltip: l10n.createProject,
            onPressed: () => context.push('/projects/new'),
          ),
        ],
      ),
      body: Column(
        children: [
          _SearchBar(
            controller: _searchController,
            onChanged: _onSearchChanged,
            showClear: _showClearButton,
            onClear: _onSearchClear,
          ),
          _StatusFilterBar(
            activeFilter: _activeStatusFilter,
            onFilterChanged: _onStatusFilterChanged,
          ),
          Expanded(
            child: RefreshIndicator(
              onRefresh: () async {
                // Снимок filter до await — защита от гонки при смене фильтра
                final currentFilter = _currentFilter;
                try {
                  // Riverpod 3: ref.invalidate + ref.read(.future) вместо ref.refresh
                  ref.invalidate(projectListProvider(filter: currentFilter, limit: _limit));
                  await ref
                      .read(projectListProvider(filter: currentFilter, limit: _limit).future)
                      .timeout(const Duration(seconds: 30)); // нет зависшего спиннера
                } catch (_) {
                  // Ошибка отобразится через ref.listen → SnackBar
                }
              },
              child: isFirstLoad
                  ? _buildLoading()
                  : isFirstError
                      ? _buildError(asyncProjects.error!, l10n, filter)
                      : _buildData(effectiveData!, filter, l10n),
            ),
          ),
        ],
      ),
    );
  }

  // is-проверки вместо object pattern matching: синтаксис `ExceptionClass()`
  // в switch мэтчит только если конструктор без аргументов — ненадёжно для
  // классов с final String message (задача 10.2). is-форма всегда корректна.
  String _errorMessage(AppLocalizations l10n, Object error) {
    if (error is UnauthorizedException) return l10n.errorUnauthorized;
    if (error is ProjectForbiddenException) return l10n.errorForbidden;
    return l10n.errorLoadingProjects;
  }
}
```

```dart
class _ProjectsContent extends StatelessWidget {
  const _ProjectsContent({required this.projects});
  final List<ProjectModel> projects;

  @override
  Widget build(BuildContext context) {
    final width = MediaQuery.sizeOf(context).width;

    if (width < 600) {
```

### `ProjectCard`

**Файл:** `frontend/lib/features/projects/presentation/widgets/project_card.dart`

Ключевые правки от предыдущих ревью:

### `ProjectsListScreen` — интеграционные widget-тесты

**Синтаксис `overrideWith`** зависит от определения провайдера в задаче 10.3:

```dart
// Если 10.3: @riverpod с named-параметрами (riverpod_generator)
// → family принимает record; overrideWith получает (ref, record):
projectListProvider.overrideWith(
  (ref, ({ProjectListFilter? filter, int limit, int offset}) args) async => ...,
);

// Если 10.3: FutureProvider.family с одним аргументом
// → уточнить по фактическому коду задачи 10.3 перед написанием тестов
```

**Паттерн pump для async:** после `pumpWidget` нужна пара `pump()` для прохождения через microtask-очередь (`await future → resolve → setState`). `pumpAndSettle()` дожидается стабилизации, но при зависших таймерах зависнет сам. Безопасная последовательность:

```dart
await tester.pumpWidget(...);  // mount
await tester.pump();            // первый frame: виджет построен, provider запущен
await tester.pump();            // microtask: future resolved → setState
await tester.pumpAndSettle();   // анимации и отложенные rebuild'ы
```

**Loading override без таймера:** `Future.delayed(days: 1)` оставляет зомби-таймер — test runner кидает `A Timer is still pending`. Используйте `Completer` без `complete()`:

```dart
// ✅ Правильно: Completer завершается при dispose контейнера, без pending timer
```
