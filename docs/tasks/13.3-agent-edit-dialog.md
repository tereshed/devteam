## Семантика обнуления полей (PATCH и бэкенд)

| Смысл | JSON |
|-------|------|
| Не менять | ключ **отсутствует** |
| Сброс в NULL | **`"field": null`** |

**Dart:** три-состояние — **«Канон»** (**`Patch<T>`** / ручная **`Map`**). **`teamAgentEditPromptNone`** в UI → для clear уходит **`prompt_id: null`** в JSON.

### Go: `encoding/json` и три-состояние

Для поля вида **`*string`** с тегом **`json:"model"`** пакет **`encoding/json`** трактует тело **`{}`** и **`{"model": null}`** одинаково — **`nil`**. **Указатели без доработки не различают omit и null**. Это **симметричная** проблема фронту; **без** решения на сервере end-to-end семантика обнуления **не работает**.

**Обязательно выбрать один подход в PR бэкенда:**

| # | Подход |
|---|--------|
| **1** | **`UnmarshalJSON`** на **`PatchAgentRequest`** с set-флагами по полям (**fieldSet map[string]bool** или структура **present**). |
| **2** | **`json.RawMessage`** для «три-состояние» полей + ручной разбор веток **omitted / null / value**. |
| **3** | Библиотека уровня **`nullable`** / sentinel (**optional**), явно хранящая **Set** + **Value**. |

После выбора — unit-тесты на **`{}`**, **`{"model":null}`**, **`{"model":"x"}`** для каждого поля.

**Go DTO:** имя **`PatchAgentRequest`** (**handler/dto**); не путать с **`UpdateAgentPatch`** на фронте (**«Канон»**, строка PATCH body).

## Безопасность и IDOR

1. **Клиент:** только whitelist; не **`agent.toJson()`**; типизированное тело (**`UpdateAgentPatch`**, **«Канон»**).

2. **Доступ к проекту и RBAC:** право на **`PATCH`** — **то же**, что на **`GET /api/v1/projects/:id/team`**, если иное не оговорено в бэкенде. **Не** вводить новую матрицу прав в **13.3** без отдельной задачи. Если в коде сейчас **`GET /team`** доступен шире, чем задуманное право на мутацию (например **PATCH** только для owner) — **отдельное предусловие** и явный пункт чек-листа бэкенда **до** мёржа.

3. **Привязка агента к проекту (IDOR):** handler проверяет **agent ∈ team(project `:id`)**; иначе **`404`**, не **`403`**.

4. **Формат `:agentId` в path:** до запроса в БД — **`uuid.Parse(c.Param("agentId"))`** (или эквивалент). **Невалидный UUID** → **`400 Bad Request`** (validation), **не** **`404`** (битый ввод ≠ «не найден»). Парсинг **до** SQL снижает риск странных edge cases.

5. **MCP и Swagger** — ниже.

### HTTP 409 Conflict

**Семантика в 13.3:** **`409`** — **бизнес-конфликт** (сервис отклонил мутацию; состояние агента в БД **не** изменилось). **Оптимистическая блокировка / `ETag`** — **вне скоупа 13.3**.

**UX при 409 (единый канон):** **`SnackBar`** с **`teamAgentEditConflictError`** (или узкоспециализированный ключ). **Диалог остаётся открытым** — как при обычной ошибке сохранения (**п. 5** сценариев). **`invalidate`** и **`pop` при 409 не выполнять** — команда на сервере не обновлена, локальный черновик пользователя сохраняем.

### Обработка ответа в UI

- **Успех (2xx):** **`invalidate` → await `teamProvider.future` → `Navigator.pop`** с **`mounted`** после каждого **`await`** (**«Канон»**, последовательность сохранения).
- **409:** в **`mapDioExceptionForRepository`** — отдельная ветка (**`on409`** / код ответа): только **`SnackBar`**, диалог **не** закрывать (**раздел выше**); **`invalidate`** и **`pop`** **не** выполнять.

## Краевые случаи

### Промпты и race «Save до ответа GET /prompts»

Пока список промптов в состоянии loading, поле **`prompt_id`** для PATCH считается **unchanged** (ключ **omit**), если пользователь **явно** не менял выбор (**`_promptDirty`** / флаг «трогали промпт» — по реализации). **Save** в этом состоянии **разрешён** (вариант **а**): пользователь может сохранить только правки модели / **`is_active`** без ожидания промптов. **Не** блокировать **Save** до завершения **`GET /prompts`** ценой UX (вариант **б** — только если продукт передумает).

Секция промпта: loading / ошибка / пустой список; **`CancelToken`** — **`cancel()`** в **`dispose`**.

### Сохранение и **`_saving`**

- **`barrierDismissible: false`** на время **`_saving`**.
- Guard двойного нажатия «Сохранить».
- **Кнопки:** «Отмена» (**`teamAgentEditCancel`**) — **`onPressed: null`**, пока **`_saving`**; «Сохранить» — **`CircularProgressIndicator`** (индикатор прогресса).
- **Вся форма** при **`_saving`:** обёртка **`IgnorePointer(ignoring: _saving, child: …)`** (**или** **`AbsorbPointer`** с тем же смыслом). Пока PATCH в полёте, пользователь **не** может менять **`Switch`**, допечатывать модель и т.д. — иначе после успешного **`pop`** теряются уведомления о несохранённом вводе, после ошибки PATCH локальный state расходится с отправленным телом. После **ошибки** PATCH (**`_saving = false`**) форма снова интерактивна; «истина последнего запроса» — то тело, что ушло на сервер (**last-applied snapshot** при необходимости синхронизировать с ответом ошибки по ревью).

### Несохранённые изменения

Технический паттерн **`showDialog<bool>`** → **`if (ok != true || !mounted) return`** — **`task_detail_screen.dart`** ~339–362; тексты — **`teamAgentEditDiscard*`**.

**Поиск самой формы редактирования:** только **`find.byKey(const Key('agentEditDialogBody'))`** (**«Размещение API»**). **`find.byType(AlertDialog)`** для основной формы — **запрещено**: оболочка — **`Dialog`** или sheet, не **`AlertDialog`** (**«Адаптив shell»**). **`AlertDialog`** — только discard (**п. 3**).

1. Открытие с **`agent`** — видны поля модели, промпта, code_backend, переключатель активности (**на обоих** shell).
2. **Cancel** / barrier dismiss при **чистой** форме — **`pop`**, **`patchAgent`** не вызывался (**на обоих** shell).
3. **Cancel** при **`_dirty == true`** — сначала **`AlertDialog`** discard (паттерн **`task_detail_screen.dart`** ~339–362), затем выход — здесь **`AlertDialog`** **корректен**.
4. Успешное сохранение — на моке **`TeamRepository`** (или override провайдера): **`verify(mock.patchAgent(any)).called(1)`**; **`verify(mock.getTeam(any)).called(2)`**; **`verifyInOrder([mock.getTeam(any), mock.patchAgent(any), mock.getTeam(any)])`**; после успеха — **`expect(find.byKey(const Key('agentEditDialogBody')), findsNothing)`** (**`pop`**). Проверять итоговый **`AsyncValue`** после **`invalidate`** **не обязательно** — достаточно порядка вызовов репозитория (**на обоих** shell).
5. Ошибка сохранения (**`TeamApiException`**) — **`SnackBar`**, форма остаётся (**`find.byKey(...)` findsOneWidget**).
6. **409** (если мокируется) — **`SnackBar`** с **`teamAgentEditConflictError`**; форма остаётся — **`expect(find.byKey(const Key('agentEditDialogBody')), findsOneWidget)`**; **`invalidate`** и **`pop`** **не** выполняются (как **п. 5** и раздел **HTTP 409**).
7. Двойной тап «Сохранить» — **`patchAgent`** вызван один раз (**guard**).
8. Ошибка **`GET /prompts`** — секция промпта в состоянии ошибки, остальная форма доступна.
9. **`dispose`** диалога во время in-flight **`GET /prompts`** — после **`pumpWidget`** / закрытия: **`expect(tester.takeException(), isNull)`** (нет **`setState()` after dispose**); опционально мок **Dio** фиксирует отмену запроса. **Не** требовать **`verify(CancelToken.cancel)`** через инжект фабрики токена (**over-engineering** для **13.3**).
10. Регресс **13.2:** под **`AgentCard`** нет **`Switch`** (**`find.descendant`** → **`findsNothing`**).
11. **Адаптив:** **`pumpWidget`** с **`Size(400, …)`** — **`autofocus`** поля модели **выключен** (или **`FocusNode`** без фокуса после первого фрейма); с **`Size(800, …)`** — **`autofocus`** на поле модели **включён**. Дополнительно на **< 600** dp — наличие **`Padding`** с **`MediaQuery.viewInsetsOf(context).bottom`** на прокручиваемом содержимом sheet (**«Адаптив shell»**); на **≥ 600** dp — без лишнего дублирования **`viewInsets`** на корне **`Dialog`**. Проверка фокуса: **`Focus.of(...).hasFocus`** / **`tester.testTextInput.isRegistered`** по согласованию ревью.
