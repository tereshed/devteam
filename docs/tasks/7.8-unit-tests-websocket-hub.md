# 7.8 Unit-тесты: WebSocket Hub

**Sprint:** 7 — WebSocket и Реалтайм (Backend)
**Файл:** `backend/internal/ws/hub_test.go`
**Статус:** ✅ Выполнено

---

## Описание

Покрыть **`ws.Hub`** (7.1) **исчерпывающими** unit-тестами. Hub — низкоуровневое ядро WebSocket-стека: register/unregister, broadcast по `project_id`, unicast по `clientID`, slow-client isolation, graceful shutdown. От его корректности зависят 7.2–7.7 и весь Sprint 7.

> **Критично:** Мы пишем тесты не ради покрытия, а чтобы менеджер коннектов не разваливался под нагрузкой и не утекал в проде. Hub живёт всё время работы приложения; одна утечка горутины = ползущий heap до OOM за неделю.

---

## Контекст использования

```
Tests (hub_test.go)
   ├─ harness: NewHub() + Run(ctx) в горутине + helper newFakeClient()
   ├─ Actor Model: ВСЕ мутации Hub'а — внутри Run()
   └─ goleak.VerifyNone(t) обязателен для всех асинхронных тестов
```

Согласно `README.md` (Sprint 7) — это **последняя** задача спринта; зелёные тесты = sign-off спринта.

## Требования к качеству тестов

### Table-Driven Tests — ОБЯЗАТЕЛЬНО

Группа «Edge Cases», «Multiple projects routing» и «Error handling» — table-driven. Иначе 30+ тестов превратятся в копипаст инициализации Hub.

```go
type tc struct {
    name        string
    setup       func(*testing.T, *Hub)
    action      func(*testing.T, *Hub)
    assertion   func(*testing.T, *Hub)
}
```

### Hub harness — единый помощник

```go
// newTestHub стартует Hub.Run в горутине, возвращает Hub и cancel.
// t.Cleanup автоматически: cancel(), wait done, goleak.
func newTestHub(t *testing.T) (*Hub, context.CancelFunc) { ... }

// newFakeClient создаёт *Client без реального websocket.Conn:
//   - Conn = nil или фейковая структура (для тестов мутаций состояния Conn не нужен).
//   - Send буферизован.
func newFakeClient(t *testing.T, id string, sendBuf int) *Client { ... }
```

### Проверка на утечки горутин — `goleak` в `t.Cleanup()`

Каждый тест, который запускает `Run` в горутине или ожидает асинхронных сигналов, **обязан**:

```go
import "go.uber.org/goleak"

func TestX(t *testing.T) {
    hub, cancel := newTestHub(t) // t.Cleanup внутри: cancel → wait done → goleak.VerifyNone(t)
    // ...
}
```

**Не допускается** выносить goleak в `TestMain` — отдельный тест поймает только последнее запущенное; нужно проверять каждый.

### Без реальных TCP-соединений

`Client.Conn` (`*websocket.Conn`) — внешняя зависимость и сетевой ресурс. Hub его **не дёргает** напрямую (только закрывает в `WritePump`/`ReadPump`, см. 7.1). Поэтому в hub-tests **не поднимаем `httptest.Server` + `websocket.Dial`** — это уже область интеграционных тестов (опционально, отдельным файлом с тегом `//go:build integration`).

## Что НЕ тестируем здесь

- **`Client.WritePump`/`ReadPump`** — это поведение `*websocket.Conn`, идёт в integration-тесты (с реальным `httptest.Server`).
- **HTTP-upgrade и JWT** — задача **7.2** и её тестов `handler_test.go`.
- **EventBus и HubBridge** — задача **7.4** и её тестов.
- **Реалистичный multi-process load** — отдельный e2e-тест (вне Sprint 7).

---

## Дизайн-решения

1. **Без `*websocket.Conn` в hub-tests:** Hub его не использует напрямую (только закрывает `Send`-канал). Имитация коннектов через `Client{Conn: nil}` валидна и быстра.
2. **`goleak` per-test, не `TestMain`:** локализация утечки, никаких ложных пропусков.
3. **Table-driven для multi-projects routing и edge cases:** уменьшает копипаст ~30 раз и читабельнее.
4. **`-race` обязателен для всей C/D/E/F секции:** Hub == concurrent component, без `-race` тесты бесполезны.
5. **Бенч broadcast'а:** одна простая защита от случайного `O(N)` в будущих рефакторингах структуры данных (см. 7.1 «O(1) Broadcast»).

## Анти-паттерны (что ЗАПРЕЩЕНО в тестах)

| Анти-паттерн | Почему |
|--------------|--------|
| `t.Parallel()` в `hub_test.go` | goleak.VerifyNone(t) сканирует ВЕСЬ процесс; параллельные тесты создают ложные срабатывания в CI. Hub-запускам не нужна параллельность — тесты мокают сеть и выполняются за миллисекунды. |
| `time.Sleep` для синхронизации | Flaky CI; используем `chan struct{}` / `sync.WaitGroup` / `assert.Eventually` с timeout |
| `Run` без последующего `cancel` | Утечка горутины и Hub'а между тестами |
| Глобальный `hub` на пакет | Гонка между тестами, нарушает изоляцию |
| Запуск нескольких `Run` для одного Hub | Hub допускает один Run; double-run = race |
| Прямая мутация `hub.projects[...]` из теста | Нарушение Actor Model (`backend.mdc` §2.1: только `Run` мутирует) |
