## Канон режима отображения (SSOT)

**Зафиксировано одно решение для всей задачи — не выбирается в PR.**

**Гибрид по ширине экрана** (тот же порог, что в **10.4** и в **`docs/rules/frontend.md`** § 2.2):
- **ширина \< 600 dp** — вертикальный **компактный список** (одна колонка, строки/карточки **`TaskListItemModel`**);
- **ширина ≥ 600 dp** — **Kanban**: колонки по **`taskStatuses`** (**12.1**), горизонтальный скролл доски, вертикальный скролл внутри колонки.

Отдельный режим «только таблица (**DataTable**)» для этой задачи **не** реализуется. Если команда позже решит добавить таблицу — отдельная задача в roadmap, не расширение **12.4**.

### Альтернативы (отклонены для 12.4)

| Вариант | Почему не канон |
|---------|------------------|
| Только Kanban на всех ширинах | На mobile колонки неуместны без горизонтального скролла всей доски — хуже UX при длинных заголовках. |
| Только таблица (**DataTable**) | Отдельный режим не входит в **12.4**; таблица без отдельной задачи размывает объём. |

## Предусловия перед началом

1. **Flutter ≥ 3.27.0** локально — для API вроде **`Color.withValues`**, как в **10.4** (см. **10.4**, предусловие **0**). Команда: `flutter --version`. В **`frontend/pubspec.yaml`** зафиксирован только constraint Dart SDK, **не** минимальная версия Flutter — формально «политика» подтверждается выбором SDK на машине разработчика и в CI. Если CI/контейнер младше **3.27.0** — отдельная задача на обновление пайплайна, **в скоуп 12.4 не входит**.
2. **`l10n.yaml`:** `synthetic-package: false` — иначе скорректировать импорт **`AppLocalizations`** (см. **10.4**).
3. **`taskListControllerProvider`** — family с **именованным** параметром **`projectId`**:

   `taskListControllerProvider(projectId: uuid)`  

   В widget-тестах **`overrideWith`** зависит от сигнатуры codegen (**12.3**); перед написанием тестов сверить фактический **`task_list_controller.g.dart`**.
4. **`TaskListFilter`** — дефолты при первом **`build`** контроллера совпадают с **12.3** (рабочий state: фильтр не **`null`**); для теста empty state зафиксировать ожидаемый запрос (**например** отсутствие активных фильтров → пустой список при **`total == 0`**).
5. Роут **`/projects/:id/tasks`** обрабатывается веткой shell (**10.7**). Невалидный **`projectId`** или отсутствие проекта перехватываются **`ProjectDashboardScreen`** (**оверлей / fallback**, см. **`project_dashboard_screen.dart`**) — контент ветки **`tasks`** в этом случае **не** строится. **`TasksListScreen`** в нормальном flow получает уже согласованный **`projectId`**; **дополнительная валидация формата на экране 12.4 не обязательна** (при желании — только **`assert`/`debug`**, не отдельный UX-путь).

## Описание

Presentation-слой: **`TasksListScreen`** получает **`projectId`** (конструктор или из родителя, но **источник истины** — параметр маршрута **`GoRouterState`**, см. ниже). Данные только через **`TaskListController`** (**12.3**).

**Запрещено:** собирать query **`listTasks`** вручную на экране — только **`TaskListFilter`** и **`setFilter`** на нотифайере (**12.3**).

**Drag-and-drop** между колонками Kanban и любые жесты смены статуса перетаскиванием — **вне скоупа 12.4** во всех случаях; смена статуса через **`updateTask`** / мутации — только **12.8**.

**Вне скоупа 12.4:** публичный виджет **`TaskCard`** (**12.6**) — до него временная строка/карточка в **`tasks_list_screen.dart`**; **`DiffViewer`**, Pause/Cancel — **12.5** / **12.8**; кнопка «Создать задачу» в верхней области (**12.8**). Подписка на WebSocket — **12.9**; экран отображает данные из **`TaskListState`**, но поля **`realtimeMutationBlocked`**, **`realtimeSessionFailure`**, **`realtimeServiceFailure`** **не** выводить в UI **12.4** (**12.8** / **12.9**). Текст «чат-ленты» по realtime на списке — **не** делать. **Сохранение поиска/фильтров при переключении вкладок shell** задаётся **`@Riverpod(keepAlive: true)`** на **`taskListControllerProvider`** (**12.3**) и реализацией **`StatefulNavigationShell`** / деревья веток — **12.4** **не** вводит отдельный **`InheritedWidget`** / локальный стейт «поверх» контроллера; при потере фильтров в QA — задача на **12.3** или shell, не точечная доработка только **12.4**.

## Контракт данных (**12.3**)

- **`ProjectDashboardScreen`** (**`project_dashboard_screen.dart`**) при загрузке / ошибке **`projectProvider`** показывает **overlay поверх области веток** (**`Stack`** над **`navigationShell`**, см. **`project_dashboard_shell.dart`**): закрывается **контент вкладок**, **не** верхний **`AppBar`** с именем проекта и **не** боковая/нижняя навигация shell. **`TasksListScreen`** может уже быть в дереве, но быть перекрытым. **Не** дублировать второй **`CircularProgressIndicator`** на всю область экрана для первичной загрузки списка задач — индикатор загрузки списка только **внутри тела** вкладки (как в **`projects_list_screen.dart`** / **10.4**), чтобы при переключении с другой вкладки не было эффекта «двойного спиннера».
- **`ref.watch(taskListControllerProvider(projectId: …))`** → **`AsyncValue<TaskListState>`**.
- **Чтение `AsyncValue` без флика при refresh** — как **`ProjectsListScreen`** (**10.4**): переменная результата **`ref.watch`** (например **`listAsync`**) — использовать **`listAsync.value`** / **`valueOrNull`**, **`listAsync.isLoading`**, **`listAsync.hasError`**; различать первую загрузку (нет **`value`**) и **`refresh`** при уже показанных данных. Для основного тела экрана **не** использовать только **`listAsync.when(data:, loading:, error:)`** без доступа к предыдущему **`value`** — при **`refresh`** будет мигание полноэкранного спиннера.
- Пагинация: **`hasMore`**, **`loadMore()`**, **`refresh()`** на нотифайере; при падении **`loadMore`** ошибка в **`state.loadMoreError`** (провайдер остаётся **`AsyncData`**) — UI см. п. **8** ниже.
- Ошибки: **`AsyncError`** при первой загрузке; при повторной ошибке при уже показанных данных — **`AsyncValue.value`** + **`ref.listen`** → SnackBar (**10.4**). Тексты — функции **`taskListErrorTitle`**, **`taskErrorDetail`** из **`task_errors.dart`** (§ ниже); **не** вводить ключ **`tasksLoadError`** и **не** дублировать смысл **`taskErrorGeneric`**.
- Экран **не** вызывает **`ref.invalidate(taskListControllerProvider(...))`** без причины из контракта **12.3**.

## UI: обязательные элементы

**Тип виджета:** **`TasksListScreen`** — **`ConsumerStatefulWidget`** (как **`ProjectsListScreen`** в **10.4**): в **`State`** — **`TextEditingController`** поиска, при необходимости **`ScrollController`**(ов) для **`NotificationListener`** / **`extentAfter`**; **`dispose()`** — **`controller.dispose()`**. **`ConsumerWidget`** без **`State`** для этого экрана **не** использовать — нет жизненного цикла для контроллеров.

1. **Без собственного `Scaffold` + `AppBar` для заголовка раздела.** Родитель — **`ProjectDashboardScreen`**: уже есть **`Scaffold`** и **`AppBar(title: имя проекта)`**; вкладка «Задачи» обозначена **`NavigationRail` / `NavigationBar`** shell. Оборачивать **`TasksListScreen`** во второй **`Scaffold(appBar: AppBar(title: projectDashboardTasks))`** **запрещено** — получится двойной AppBar и вложенные **`Scaffold`**. **`TasksListScreen`** — только **тело** ветки **`tasks`** (как замена **`ProjectDestinationPlaceholder`** — там был просто **`Center(Text(title))`** без **`Scaffold`**). Если нужна подпись раздела внутри контента — **`SliverAppBar`** / **`Container`** с текстом **внутри прокручиваемого тела**, а не второй верхний **`AppBar`** через вложенный **`Scaffold`**.
2. **Фильтры** — минимум: статус и/или приоритет + **поле поиска** с **`TaskListFilter.search`**. Дебаунс ввода поиска: **400 ms** (как **10.4**); после дебаунса обновление фильтра в контроллере. Если **`state.filter.search`** изменился **извне** (**`setFilter`** / **`refresh`** после WS и т.д., **12.3**) и расходится с **`_searchController.text`** — синхронизировать текст поля (**`_searchController.text = новое значение`**) через **`ref.listen`** по провайдеру (например **`select`** на **`filter.search`** из успешного **`TaskListState`**), **без** повторного запуска дебаунса на то же значение.
3. **Индикатор загрузки** — различать первый заход (**нет данных**) и refresh со стейлом (**10.4**); только в теле вкладки, см. контракт про overlay области веток выше.
4. **Пустое состояние** — две ветки: **(а)** список пуст без активных фильтров — **`tasksEmpty`**; **(б)** пусто из-за фильтра — **`tasksEmptyFiltered`** + кнопка с подписью **`tasksEmptyFilteredClear`**. По нажатию сброса: **`ref.read(taskListControllerProvider(projectId: …).notifier).setFilter(TaskListFilter.defaults())`** (**`TaskListFilter.defaults()`** — **`frontend/lib/features/tasks/domain/requests.dart`**); локальное состояние полей фильтра (**`TextEditingController`** поиска → **`clear()`**, выбранные чипы статуса/приоритета → значения, согласованные с **`defaults()`**).
5. **Ошибка верхнего уровня** (первичная загрузка / полный сбой — **`AsyncError`**) и SnackBar при ошибке refresh: заголовок **`taskListErrorTitle(l10n, error)`**, при необходимости вторая строка **`taskErrorDetail(error)`** (может быть **`null`**); кнопка повтора — **`retry`**. **Не** добавлять ключ **`tasksLoadError`** — общий fallback уже **`l10n.taskErrorGeneric`** внутри **`taskListErrorTitle`**. Если нужна отдельная формулировка именно «не удалось загрузить список» — расширение **`taskListErrorTitle`** / контракта **12.3**, отдельное согласование, не молчаливый новый `.arb` в **12.4**.
6. **Обновление списка (один и тот же `notifier.refresh()`, `invalidate` для pull-to-refresh не использовать):**
   - **ширина \< 600 dp** — **`RefreshIndicator`** оборачивает **основной вертикальный** **`ListView`** (один **`Scrollable`** для pull-down).
   - **ширина ≥ 600 dp** (Kanban: горизонтальный скролл доски + вертикальные скроллы в колонках) — **`RefreshIndicator` не использовать** (конфликт осей / несколько вертикальных областей). Вместо него — **`IconButton(icon: Icons.refresh)`** в **правом верхнем углу** области доски (рядом с фильтрами или над колонками), **`onPressed`** → **`ref.read(taskListControllerProvider(projectId: …).notifier).refresh()`**.
7. **Бесконечная прокрутка** — триггер **`loadMore`** зафиксирован: на соответствующем **`Scrollable`** (вертикальный список на mobile **или** вертикальный скролл внутри колонки Kanban) — **`NotificationListener<ScrollNotification>`**, вызывать **`loadMore()`**, если **`notification.metrics.extentAfter < 600`** (логические пиксели), и **`state.hasMore`**, **`!state.isLoadingMore`**. **Альтернатива:** только для одного плоского **`ListView`** без вложенных вертикальных скроллов — построение элемента с индексом **`>= state.items.length - 5`** инициирует **`loadMore`**. **Не** комбинировать порог **`extentAfter`** и порог по индексу в одном PR без мотивации; для режима с Kanban использовать только **`extentAfter < 600`**. Реентрантность **`loadMore`** обеспечивает контроллер (**`_loadMoreInFlight`** в **`task_list_controller.dart`**); экран **не** дублирует mutex — только условия для UX и вызова метода.
8. **Ошибка пагинации (`state.loadMoreError != null`):** нижний **inline-баннер** (над футером списка / под последней строкой): заголовок **`taskListErrorTitle(l10n, state.loadMoreError!)`**, при необходимости **`taskErrorDetail(state.loadMoreError!)`**, кнопка **retry** → **`ref.read(...notifier).loadMore()`**. **SnackBar** для этого случая **не** использовать (избежать дублирования с ошибками верхнего уровня и пустых состояний).

## Навигация и `projectId` (разрешение противоречия с **12.5**)

- **Источник `projectId` для экрана, провайдера и любой навигации:** только параметр маршрута **`/projects/:id/`** (конструктор **`TasksListScreen(projectId: …)`** заполняется из **`GoRouterState.pathParameters['id']`**). Поле **`TaskListItemModel.projectId`** не используется для навигации (контракт с API/WS; риск рассинхрона при неконсистентных данных).
- **До появления маршрута детали (**12.5**):** по тапу на строку/карточку — **`SnackBar`** с **`taskDetailNotImplementedYet`**. Использовать **`ScaffoldMessenger.of(context)`**, где **`context`** — из **`TasksListScreen`**; подъём по дереву находит **`ScaffoldMessenger`**, связанный с **`Scaffold`** **`ProjectDashboardScreen`**. Связь **`tasksNavigatorKey`** с **`ScaffoldMessenger`** отсутствует (**`tasksNavigatorKey`** — **`GlobalKey<NavigatorState>`** для стека ветки); для SnackBar ключ навигатора **не** нужен. В **12.5** переход на детали — **`GoRouter.of(context).push(...)`** на навигатор ветки **`tasks`** (**`tasksNavigatorKey`** в **`buildProjectDashboardShellBranches`**, **`project_dashboard_routes.dart`**): **без** **`parentNavigatorKey: rootNavigatorKey`**, чтобы маршрут оставался внутри ветки и нижняя панель shell сохранялась (канон URI — **12.5**).
- Экран **обязан** быть зарегистрирован в роутере в рамках **12.4** (см. критерии приёмки): противоречия «не регистрировать до 12.5» нет — регистрируем список и временный SnackBar.

### Ошибки и `task_errors.dart`

Использовать готовые функции из **`frontend/lib/features/tasks/presentation/controllers/task_errors.dart`**:

- **`String taskListErrorTitle(AppLocalizations l10n, Object error)`** — заголовок для списка задач (маппинг исключений → существующие ключи `.arb`: **`taskListErrorProjectNotFound`**, **`taskErrorGeneric`**, **`errorUnauthorized`**, …). Это **не** имя ключа в `.arb`, функцию **не** дублировать новым ключом **`taskListErrorTitle`**.
- **`String? taskErrorDetail(Object error)`** — короткий хвост сообщения API (до 200 символов) или **`null`**; уместна вторая строка в error-state / баннере.

Паттерн: **title** = **`taskListErrorTitle(l10n, error)`**, **subtitle** = **`taskErrorDetail(error)`** (если не **`null`**). Там же объявлены **`TaskMutationOutcome`** и заголовки для детали (**`taskDetailErrorTitle`**) — для **12.4** нужны только функции выше для списка.

## Архитектурная связь

Экран **не** обращается к **`TaskRepository`** и **не** импортирует Dio. Ошибки — через **`task_errors.dart`**, не через самописный разбор в виджете.

```
/projects/:id/tasks
        │
        ▼
   TasksListScreen (ref.watch / ref.read → только контроллер)
        │
        ▼
   taskListControllerProvider(projectId)  ←── 12.3
        │
        ▼
   taskRepositoryProvider → TaskRepository (12.2) → HTTP API
```

## Частые ошибки

1. Локальная фильтрация по **`title.contains`** при активном серверном **`search`** (**12.3** § 50).
2. Игнор **`hasMore`** / вызов **`loadMore`** при **`isLoadingMore`** (контроллер уже сериализует вызовы — экран не добавляет второй mutex).
3. **`invalidate`** списка без необходимости — ломает позицию скролла Kanban.
4. Хардкод строк; импорты **`../../`**.
5. Использование **`TaskListItemModel.projectId`** для **`push`** вместо параметра экрана.
6. Ожидание поля **`description`** у **`TaskListItemModel`** (его нет в DTO списка — см. **12.1**); копипаст из **10.4**/`ProjectModel` недопустима.
7. Дублирование маппинга статуса вместо **`task_status_display.dart`** (сломает вынос **12.6**).
8. Путаница: **`taskListErrorTitle`** — функция Dart в **`task_errors.dart`**, не ключ `.arb`; заведение ключа с тем же именем или дубликата **`tasksLoadError`** рядом с **`taskErrorGeneric`**.
9. Отображение **`realtimeMutationBlocked`** / **`realtimeSessionFailure`** / **`realtimeServiceFailure`** на экране списка (**12.8** / **12.9**).
10. Второй **`Scaffold`** с **`AppBar(title: …)`** внутри **`TasksListScreen`** поверх **`ProjectDashboardScreen`**.
11. Вызов **`loadMore`** только при последнем элементе без порога **`extentAfter`** / индекса — «прыжки» контента и лишние round-trip.
