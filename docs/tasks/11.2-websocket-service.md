## Конечный автомат (FSM) и гонки

**Состояния внутреннего автомата** (имена на усмотрение, семантика фиксирована):

| Состояние | Смысл |
|-----------|--------|
| **`idle`** | Нет активного сокета, нет ожидающего handshake, backoff может быть запланирован только из `reconnecting`. |
| **`connecting`** | Выполняется `channelFactory` / handshake; сокёт ещё не считается «открытым» для приложения. |
| **`open`** | Handshake успешен, читается стрим. |
| **`closing`** | Закрытие сокета по запросу; дождаться завершения, затем `idle` или `reconnecting`. |
| **`reconnecting`** | Ожидание таймера backoff / фиксированной задержки перед следующим `connecting`. |
| **`disposed`** | **Терминал:** любой вызов кроме no-op — **`StateError`**. |

**Допустимые переходы** (все прочие вызовы API из «не того» состояния → **`StateError`**):

```
idle ──connect()──► connecting ──ok──► open ──socket lost──► reconnecting ──timer──► connecting
  ▲                      │                │                        │
  │                      │ fail           │ disconnect/pause       │
  └──────────────────────┴──► idle         └──► closing ────────────┘
disposed ◄── dispose() из любого состояния (кроме disposed)
```

- Параллельно **не** допускается два живых сокета: при **`connect(B)`** во время **`connecting`** для `A` — **инкремент `_sessionId`**, отмена логики «привязки» к устаревшему `Future`, закрытие канала, если он успел открыться устаревшей сессии (см. ниже).  
- **Таймер backoff** из сессии `A` при переходе на проект `B` или `disconnect`/`dispose` — **отменяется** до срабатывания.

## Отмена handshake «в полёте»

`WebSocketChannel.connect` / аналог **не** отменяем снаружи. Обязательный паттерн:

1. Перед `await` handshake инкрементировать **`int _sessionId`** (или `Object _sessionToken`).  
2. После **каждого** `await` (handshake, задержка) сравнить `sessionId` с текущим; если **не совпадает** — немедленно **`close()`** полученного канала, **не** публиковать события в `StreamController`, **не** переводить FSM в `open` для этой сессии.  
3. Так **`disconnect()` / `dispose()` / `connect(другой id)`** во время висящего handshake **не** приводят к «открытому сокету после dispose».

---

## `disconnect()` vs `dispose()` (семантика)

| Метод | Сокет | FSM после | Повторный `connect()` |
|-------|--------|-----------|-------------------------|
| **`disconnect()`** | Штатно закрыть текущий канал; отменить **все** pending backoff/idle таймеры. | `idle` (сервис **жив**). | **Разрешён** — новая сессия. |
| **`dispose()`** | Закрыть канал + отменить таймеры + **закрыть** broadcast-контроллер событий. | **`disposed`**. | **`StateError`** — нужен новый экземпляр сервиса. |

**`pause()`:** перевести в состояние без авто-реконнекта и закрыть TCP (как договорено в предыдущей версии задачи); **не** вызывать `dispose()`. **`resume()`:** снова `connect` с новым `await WsAuthProvider()`.

## Маппинг WebSocket close codes (RFC 6455 + согласование с 7.2/7.7)

До тех пор, пока бэк **не** зафиксировал custom-коды в **7.2/7.7**, клиент использует таблицу ниже; при расхождении — обновить оба документа одним PR-пакетом.

| Close code | Источник (типично) | Реакция клиента |
|------------|-------------------|-----------------|
| **1000** Normal | Штатное закрытие сервера | Сетевой реконнект (exponential full jitter), **если** не было штатного `disconnect()`. |
| **1001** Going away | Shutdown / вкладка | Как **1000** (реконнект с backoff). |
| **1006** Abnormal closure | Нет close frame (сеть) | Exponential backoff. |
| **1008** Policy violation | Bad Origin / auth handshake | **Терминал:** **`WsServiceFailure.policyCloseCode(1008)`**, **без** авто-реконнекта. |
| **1011** Internal error | Ошибка на бэке | Exponential backoff (как сеть). |
| **4401** (custom, если введён в 7.x) | Invalid / expired token | **`WsServiceFailure.authExpired`** + **`WsAuthFailure`** по согласованности с union событий; **без** backoff с тем же токеном. |
| **4429** (custom, если введён) | **MaxConnsPerUserProject** исчерпан (см. 7.7, лимит **5**) | Первый **4429** в эпизоде: **`WsServiceFailure.tooManyConnections`** + одна попытка через **≥ 60 s**; второй **4429** без сброса эпизода: **`WsServiceFailure.tooManyConnectionsTerminal`**. Сброс счётчика эпизода: **connect/disconnect**, первый envelope, **open ≥ 30 s** непосредственно перед **4429**, и при **1000/1001/1006/1011** если соединение прожило ≥ 30 s (как сброс backoff). HTTP **429** на upgrade vs WS close — **TODO(7.7)**. |

**Дедлайн выбора по 4429 / 429:** не позднее **первого коммита кода по 11.2** исполнителем **или** официального **старта имплементации** спринта 11 (что наступит раньше по процессу команды): одна строка в таблице выше + тест/stub в клиенте; до дедлайна допускается временный stub с пометкой `TODO(7.7)` и ссылкой на issue/док бэка.

**Согласование с 7.7:** что именно отдаёт сервер при 6-й вкладке (HTTP 429, close **4429**, `error.code=forbidden` в кадре до close) — **один** источник правды; пункт в **критериях приёмки** после ответа бэка.

**Запрещено:** реконнектить бесконечно на **1008** / **4401** как на «сетевой обрыв».

## Backoff: формула, сброс `attempt`, `server_shutdown`

### Full jitter (единственная формула в репозитории)

Параметры: **`baseMs = 500`**, **`capMs = 30000`**, **`attempt`** начинается с **0** и увеличивается на каждой **неуспешной** попытке открытия / после обрыва, к которой применяется exponential ветка.

```
upperExclusive = min(capMs, baseMs * pow(2, attempt)).toInt();
delayMs = Random().nextInt(max(1, upperExclusive)); // [0, upperExclusive)
```

Использовать инъекцию **`Clock`** из **`package:clock`**: в проде **`Clock()`** (дефолтный конструктор = системное время по доке пакета), в тестах — **`Clock.fixed(...)`** или связка с **`fake_async`**; **не** вызывать «голый» `DateTime.now()` внутри backoff/idle/circuit window.

### Сброс `attempt`

**`attempt := 0` только если выполнено одно из:**

- Получен и успешно распарсен **хотя бы один** полноценный JSON-конверт сервера в **`WsServerEvent`** (включая `type` unknown → `WsUnknownEvent`); **или**  
- FSM пробыл в **`open`** непрерывно **≥ 30 s** (настраиваемо в `WsConfig` как `backoffResetAfterStableOpen`).

**Запрещено:** сбрасывать `attempt` сразу при переходе `connecting → open`, если сразу после этого сокет закрылся с auth-fail — иначе получится молотилка **500 ms**.

### `server_shutdown` (7.3) + anti-thundering herd

Базовая задержка **`WsConfig.serverShutdownReconnectDelay`** (**5 s**) + малый jitter **0–500 ms** + **дополнительно** **`Random().nextInt(8001) + 2000` ms** (**2–10 s** случайно) перед следующей попыткой TCP, чтобы размазать стадо клиентов.

## Downstream: `WsServiceFailure` для 11.9

**`sealed class WsServiceFailure`** (Freezed или `sealed` в Dart 3) — **отдельный** тип от полезной нагрузки 7.3, чтобы **11.9** не писал `if (event is WsErrorEvent && event.code == …)`.

Рекомендуемые варианты (имена можно слегка подправить, набор фиксирован):

| Вариант | Когда |
|---------|--------|
| **`transient(Object cause)`** | Идёт exponential backoff, сокет ещё вернётся (опционально эмитить throttled, не чаще 1/сек). |
| **`authExpired()`** | **4401**, HTTP 401 на upgrade (VM), и т.п. по таблице с бэком. |
| **`policyForbidden()`** | Кадр **7.3** `error.code=forbidden`. |
| **`policySubprotocolMismatch()`** | Рассогласование **Sec-WebSocket-Protocol** после connect. |
| **`policyCloseCode(int code)`** | Закрытие с кодом политики (напр. **1008**). |
| **`tooManyConnections()`** | Первый **4429** в эпизоде — сигнал к одной отложенной попытке. |
| **`tooManyConnectionsTerminal()`** | Второй **4429** в том же эпизоде (см. сброс счётчика в таблице close codes). |
| **`protocolBroken()`** | Circuit-breaker парсинга (см. ниже). |

**Канал доставки:** либо **второй** broadcast-стрим `Stream<WsServiceFailure> get failures`, либо **один** `Stream<WsClientEvent>` с union — **выбрать один** в репозитории и **не плодить** три способа. Выбранный вариант **обязан** быть зафиксирован в **dartdoc комментарии к классу `WebSocketService`** (какие публичные геттеры стримов, что в них попадает, рекомендуемый паттерн `listen` для **11.9**).

## Circuit breaker: шторм `WsParseError`

Если за **`T = 10 s`** (скользящее окно по `Clock`) набралось **`N = 50`** событий **`WsParseError`** (или эквивалентных сбоев декодирования конверта):

1. Эмит **`WsServiceFailure.protocolBroken()`**.  
2. Остановить авто-реконнект до **`disconnect()` + `connect()`** или **`dispose()`** нового сервиса.  

Тест: быстрый спам мусором с мок-канала → срабатывание circuit breaker.

## Парсинг: `ts`, BOM, `type`, размер фрейма

| Тема | Правило |
|------|---------|
| **`ts` UTC** | После `jsonDecode` поле `ts` **обязано** парситься в **`DateTime` с `isUtc == true`**. Если строка **без** суффикса **`Z`** и без явного numeric offset (`+00:00`) — **`WsParseError`** на конверт (не silent local). **Тест:** валидная строка с **`Z`** → `isUtc`; без таймзоны → ошибка. **Тест:** дробная часть **≥ 6 цифр** (`…33.123456789Z`) — не бросает, обрезка до микросекунд Dart допустима. |
| **UTF-8 BOM** | Перед `jsonDecode`: если `text.startsWith('\uFEFF')` — `text = text.substring(1)`. **Тест.** |
| **`type`** | Сравнение с перечнем 7.3 **case-sensitive**, **без** `toLowerCase()`. |
| **Размер входящего текстового кадра** | Текст из `web_socket_channel` уже приходит как **`String`** (Dart: UTF-16 code units). Лимит **256 KiB** считать по **байтам UTF-8**: **`utf8.encode(text).length`** **перед** `jsonDecode` (не `text.length` — для кириллицы/emoji байты ≠ числу code units). Превышение → не парсить гигантский JSON; **`WsParseError`** / отброс + вклад в circuit breaker. (Бэк лимитирует **64 KiB** — клиентский предел выше для proxy, но конечен.) |

## Аутентификация: `WsAuthProvider` и отсутствие `null`

```dart
typedef WsAuthProvider = Future<WsAuth> Function();
```

- **`Future<WsAuth?>` не использовать:** вместо «нет токена» возвращать **`WsAuth.none()`** (cookie / неавторизованный режим по контракту с бэком).  
- **`WsAuth.bearer(String jwt)`:** JWT **не** содержит «лишних» точек в префиксе протокола: в подпротоколе строка **`bearer.` + `<полный JWT>`**, где **после первых 7 символов `bearer.` весь суффикс целиком — JWT** (включая внутренние `.` трёх частей JWT). Парсер **бэка** должен использовать тот же принцип; для **`apikey.`** — аналогично префикс + секретный хвост **целиком**.  
- **Cookie-only будущее:** `WsAuth.none()` без subprotocol для токена.

## Riverpod (обязательный паттерн lifecycle)

```dart
@Riverpod(keepAlive: false)
WebSocketService webSocketService(Ref ref) {
  final service = WebSocketService(
    channelFactory: /* IOWebSocketChannel.connect или обёртка */,
    authProvider: () async {
      final token = await ref.read(authRepositoryProvider).getAccessToken();
      return (token != null && token.isNotEmpty)
          ? WsAuth.bearer(token)
          : WsAuth.none();
    },
    clock: Clock(),
  );
  ref.onDispose(service.dispose);
  return service;
}
```

**Без `ref.onDispose(service.dispose)`** — утечка сокета и таймеров при `invalidate`.

## Критерии приёмки (дополнение к уже существующим из итераций 1–2)

- [ ] Реализована **FSM**; недопустимые переходы → **`StateError`**.  
- [ ] **Session-token** после `await` handshake: устаревшая сессия закрывает канал, не пишет в закрытый контроллер.  
- [ ] **`disconnect()`** во время backoff: таймер отменён; **после `dispose()`** счётчик попыток / таймер **не** срабатывает (тест **`fake_async`**).  
- [ ] Таблица **close codes** в коде совпадает с разделом выше (после синка с 7.x).  
- [ ] **MaxConns** / 6-я вкладка: поведение согласовано с бэком и покрыто тестом или документированным stub до ответа бэка.  
- [ ] Backoff по формуле **full jitter** с **`baseMs`/`capMs`**; сброс **`attempt`** только по правилам раздела выше.
