# 2.6 Handler: ProjectHandler

**Sprint:** 2 — CRUD API для Project и Team

**Статус:** ✅ Выполнена

---

## Описание

Создать HTTP-обработчик для проектов. Handler — тонкий слой: парсинг запроса, извлечение `userID` из `gin.Context`, вызов Service, маппинг ошибок в HTTP-статусы, формирование ответа через DTO. Бизнес-логика в handler **запрещена**.

---

## Структура

```go
type ProjectHandler struct {
    service service.ProjectService
}

func NewProjectHandler(service service.ProjectService) *ProjectHandler
```

---

## Эндпоинты

| Метод | Путь | Handler метод | Описание |
|-------|------|---------------|----------|
| POST | `/api/v1/projects` | `Create` | Создать проект |
| GET | `/api/v1/projects` | `List` | Список проектов (query params) |
| GET | `/api/v1/projects/:id` | `GetByID` | Получить проект |
| PUT | `/api/v1/projects/:id` | `Update` | Обновить проект |
| DELETE | `/api/v1/projects/:id` | `Delete` | Удалить проект |

Все эндпоинты требуют авторизации (`AuthMiddleware`).

---

## Методы

### Create (`POST /projects`)

1. `ShouldBindJSON(&dto.CreateProjectRequest)` — парсинг body
2. `c.Get("userID")` → `uuid.UUID` — извлечение userID из JWT
3. `service.Create(ctx, userID, req)` — вызов Service
4. Ответ: `201 Created` + `ProjectResponse`

**Маппинг ошибок:**
| Service error | HTTP | apierror code |
|---------------|------|---------------|
| `ErrProjectNameExists` | 409 Conflict | `already_exists` |
| `ErrGitCredentialNotFound` | 400 Bad Request | `bad_request` |
| `ErrGitCredentialForbidden` | 403 Forbidden | `forbidden` |
| прочие | 500 | `internal_server_error` |

### List (`GET /projects`)

1. `ShouldBindQuery(&dto.ListProjectsRequest)` — парсинг query params
2. `c.Get("userID")` → `uuid.UUID`
3. `service.List(ctx, userID, req)`
4. Ответ: `200 OK` + `ProjectListResponse`

### GetByID (`GET /projects/:id`)

1. `uuid.Parse(c.Param("id"))` — парсинг path param
2. `c.Get("userID")` → `uuid.UUID`
3. `service.GetByID(ctx, userID, projectID)`
4. Ответ: `200 OK` + `ProjectResponse`

**Маппинг ошибок:**
| Service error | HTTP | apierror code |
|---------------|------|---------------|
| `ErrProjectNotFound` | 404 Not Found | `not_found` |
| `ErrProjectForbidden` | 403 Forbidden | `forbidden` |

### Update (`PUT /projects/:id`)

1. `uuid.Parse(c.Param("id"))` — path param
2. `ShouldBindJSON(&dto.UpdateProjectRequest)` — body
3. `c.Get("userID")` → `uuid.UUID`
4. `service.Update(ctx, userID, projectID, req)`
5. Ответ: `200 OK` + `ProjectResponse`

**Маппинг ошибок:**
| Service error | HTTP | apierror code |
|---------------|------|---------------|
| `ErrProjectNotFound` | 404 | `not_found` |
| `ErrProjectForbidden` | 403 | `forbidden` |
| `ErrProjectNameExists` | 409 | `already_exists` |
| `ErrGitCredentialNotFound` | 400 | `bad_request` |
| `ErrGitCredentialForbidden` | 403 | `forbidden` |

### Delete (`DELETE /projects/:id`)

1. `uuid.Parse(c.Param("id"))` — path param
2. `c.Get("userID")` → `uuid.UUID`
3. `service.Delete(ctx, userID, projectID)`
4. Ответ: `200 OK` + `{"message": "project deleted successfully"}`

## Извлечение userID

Конвенция из `auth_handler.go`:

```go
userID, exists := c.Get("userID")
if !exists {
    apierror.JSON(c, http.StatusUnauthorized, apierror.ErrAccessDenied, "Unauthorized")
    return
}
uid := userID.(uuid.UUID)
```

**Рекомендация:** вынести в хелпер `getUserID(c *gin.Context) (uuid.UUID, bool)` — переиспользуется во всех методах.

## Swagger-аннотации

Каждый метод **ОБЯЗАН** иметь Swagger-аннотации (задача 2.9, но аннотации пишутся сразу).

Пример для `Create`:

```go
// Create создаёт новый проект
// @Summary Создание проекта
// @Description Создаёт проект с автоматическим созданием команды
// @Tags projects
// @Security BearerAuth
// @Accept json
// @Produce json
// @Param request body dto.CreateProjectRequest true "Данные проекта"
// @Success 201 {object} dto.ProjectResponse
// @Failure 400 {object} apierror.ErrorResponse
// @Failure 403 {object} apierror.ErrorResponse
// @Failure 409 {object} apierror.ErrorResponse
// @Failure 500 {object} apierror.ErrorResponse
// @Router /projects [post]
```

**Важно:** в `@Router` **не указывать** префикс `/api/v1` (он определён в `@BasePath`).

## Конвенции (по существующему коду)

1. **Struct + конструктор** — как `PromptHandler`, `AuthHandler`
2. **`c.Request.Context()`** — передаётся в Service
3. **`apierror.JSON()`** — для всех ошибок
4. **`c.Get("userID")`** — извлечение из JWT (как `auth_handler.go`)
5. **`uuid.Parse(c.Param("id"))`** — парсинг path param с обработкой ошибки
6. **`ShouldBindJSON` / `ShouldBindQuery`** — валидация через binding теги
7. **DTO конвертеры** — `dto.ToProjectResponse()` (не `toPromptResponse` приватная)
