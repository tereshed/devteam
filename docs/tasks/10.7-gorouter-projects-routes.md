# 10.7 Обновить GoRouter: `/projects`, `/projects/:id`, `/projects/:id/*`

**Sprint:** 10 — Frontend: Проекты и навигация  

**Основной файл:** `frontend/lib/core/routing/app_router.dart`

**Терминология (SSOT в этом документе):**
- **project dashboard** — область приложения «проект»: список, создание, **`ProjectDashboardScreen`** и вложенные URL после **`/projects/:id`**.
- **shell** — оболочка навигации по разделам дашборда (**`StatefulShellRoute`** + нижняя панель / rail в **10.6**).

**Дополнительные файлы (обязательные):**
- **`frontend/lib/core/routing/project_dashboard_routes.dart`** — единый источник правды для:
  - **`projectDashboardShellBranchPaths`** — сегменты URL после **`/projects/:id`** (порядок = порядок вкладок в UI);
  - **`projectDashboardDefaultBranch`** — дефолтная ветка после **`/projects/:id`**; **инвариант:** **`projectDashboardDefaultBranch == projectDashboardShellBranchPaths.first`** (проверка в **`frontend/test/core/routing/project_dashboard_routes_test.dart`**);
  - **`ProjectRouteNames`** — имена маршрутов **`GoRoute.name`** для сегмента **`/projects`** (без литералов **`'projects'`** в нескольких файлах);
  - **`projectDashboardRedirectPreservingQuery`** — сборка целевого location при редиректе с сохранением query/fragment;
  - **`projectDashboardDetailRedirect`** — валидация **`:id`**, редирект «голого» **`/projects/:id`** на **`/projects/:id/${projectDashboardDefaultBranch}`**;
  - **`projectDashboardUnknownShellBranchRedirect`** — см. раздел **«Обоснование глобального redirect»**;
  - **`buildProjectDashboardShellBranches`** — **`StatefulShellBranch`** + дочерние **`GoRoute`** веток.
- **`frontend/lib/core/l10n/require.dart`** — **`requireAppLocalizations`**, общая проверка делегатов (см. **`router_error_screen`**, **`project_dashboard_routes`**).
- **`frontend/lib/core/routing/router_error_screen.dart`** — **`buildRouterErrorScreen`**, общий **`errorBuilder`** для **`app_router`** и **`project_dashboard_test_router`** (одна точка изменения текста/логирования).
- **`frontend/lib/core/utils/uuid.dart`** — только **`isValidUuid`**: проверка формата UUID для **`:id`** на уровне роутинга; presentation не дублирует парсинг **`GoRouterState`**.

**Регистрация redirect’ов (go_router v17: `redirect` родительского [GoRoute] **не** применяется к дочерним путям автоматически):**
| Функция | Где подключается |
|---------|------------------|
| **`rootRouterRedirect`** | Первый аргумент **`GoRouter(redirect: …)`** в **`app_router.dart`**. **Композиция:** (1) для путей **`/projects`** и **`/projects/...`** — вызов **`authGuard`**; (2) затем **`projectDashboardUnknownShellBranchRedirect`**. Новые глобальные правила добавлять **в начало или конец этой функции** явной цепочкой — один корневой слот. |
| **`projectDashboardUnknownShellBranchRedirect`** | Только из **`rootRouterRedirect`** (не отдельным top-level redirect). |
| **`projectDashboardDetailRedirect`** | **`redirect:`** на **`GoRoute(path: ':id', …)`** под **`/projects`**. |
| **`authGuard`** для **`/projects`** | **Никогда** не полагаться на «наследование» **`redirect`** с родительского **`GoRoute('/projects')`** на **`/projects/new`** или **`/projects/:id`** — в **go_router v17** это **не** работает. **`authGuard`** вызывается из **`rootRouterRedirect`** для всего префикса **`/projects`** и **`/projects/...`**. Отдельный **`redirect: authGuard`** на **`GoRoute('/projects')`** **убран**, чтобы не дублировать проверку для **`/` списка** (она уже в **`rootRouterRedirect`**). |

**Критично неошибаться:** `redirect` родительского маршрута **не** защищает дочерние пути — только **`rootRouterRedirect`** или свой **`redirect`** на каждом **`GoRoute`**.

**Тесты (обязательные / условные):**
- **`frontend/test/core/routing/auth_guard_test.dart`** — контракт **`authGuard`**: при **`AsyncLoading`** возвращается **`null`** (навигация не режется до выхода из loading — см. **`auth_guard.dart`** и документ ниже).
- **`frontend/test/features/projects/helpers/project_dashboard_test_router.dart`** — матчинг shell + **`projectDashboardUnknownShellBranchRedirect`** **без** цепочки auth (упрощение виджет-тестов дашборда); **`ProviderScope`** не обязателен.
- **`frontend/test/core/routing/root_router_redirect_test.dart`** — **`rootRouterRedirect`** + **`authGuard`** для гостя и авторизованного пользователя на **`/projects/new`**.
- **`frontend/test/core/routing/project_dashboard_routes_test.dart`** — инварианты веток, **`/projects/new`** vs **`:id`**, query/fragment при редиректе.
- Widget-тесты дашборда — см. **10.6**; сценарии из таблицы «Канонический URL и краевые случаи» — **`flutter test`** (не только **`flutter analyze`**).

## Контекст и цель

Зафиксировать дерево **`GoRouter`** для префикса **`/projects`**, правила валидации **`:id`** и **каноническое** поведение URL (включая deep link).

**Цель:**
- Зарегистрировать маршруты списка, создания, корня дашборда и веток **`/projects/:id/<branch>`**.
- Невалидный **`:id`** → **`/projects`** без монтирования дашборда и без вызова API с мусорным id.
- Экран дашборда получает **`projectId: String`** из замыкания **`pageBuilder`** (**не** из **`GoRouterState`** внутри виджета) — инвариант [10.6](./10.6-project-dashboard-screen.md).

## Целевая схема URL (SSOT)

Все редиректы на «дефолтную ветку» используют **`projectDashboardDefaultBranch`** (не литерал **`chat`** в **`app_router.dart`**).

| Путь | Поведение |
|------|-----------|
| `/projects` | Список (**`ProjectsListScreen`**, 10.4). |
| `/projects/new` | Создание (**`CreateProjectScreen`**). Должен матчиться **литералом `new`** раньше **`:id`** (порядок **`routes`** — см. чек-лист и тест). |
| `/projects/:id` | При валидном UUID → редирект на **`/projects/:id/${projectDashboardDefaultBranch}`** с сохранением query/fragment (**`projectDashboardRedirectPreservingQuery`**). При невалидном → **`/projects`**. |
| `/projects/:id/<branch>` | Ветка shell; **`<branch>`** ∈ **`projectDashboardShellBranchPaths`**. |
| `/projects/:id/<unknown>` | При валидном UUID и **`<unknown>`** ∉ списку веток → редирект на **`/projects/:id/${projectDashboardDefaultBranch}`** (см. глобальный redirect). |

## Канонический URL и краевые случаи

| Тема | Правило |
|------|---------|
| **Trailing slash** | **`/projects/:id`** и **`/projects/:id/`** трактуются как корень дашборда одинаково (**`_stripSingleTrailingSlash`** в **`projectDashboardDetailRedirect`**). Ветки без завершающего **`/`** — канон; при необходимости нормализация только через redirect, без ломания истории. |
| **Регистр UUID в пути** | **`isValidUuid`** допускает **`A–F`** (**регекс в `uuid.dart`**). **Нормализация в lowercase в URL в scope 10.7 не требуется**; строка **`projectId`** в экран передаётся **как в path**; при расхождении с API ответственность на слое данных (**10.2**) при необходимости. |
| **Query и fragment при редиректе** | При редиректе с **`/projects/:id`** на дефолтную ветку и при редиректе unknown-branch — **сохранять** query и fragment исходного **`state.uri`** (**`projectDashboardRedirectPreservingQuery`** / **`Uri.replace(path: …)`**). |
| **Литерал `new` vs `:id`** | **`go_router`** отдаёт приоритет статическому сегменту **`new`**; всё равно зафиксировать **тестом**, что **`/projects/new`** **никогда** не обрабатывается как **`/projects/:id`** с id **`new`** и не проходит UUID-валидацию как проект. Порядок: сначала **`GoRoute(path: 'new')`**, затем **`GoRoute(path: ':id')`**. |

## Неавторизованный доступ и deep links

**Факт поведения `authGuard` (проверять по коду):** неавторизованный пользователь при открытии **`/projects/...`** получает redirect (**сейчас — на **`/`**, не на **`/login`**). **Пока `authController` в `loading`, `authGuard` возвращает `null`** (допускает навигацию) — так же в **`auth_guard.dart`** для любых защищённых маршрутов. **Сохранение intended URL (return-after-login) в scope 10.7 не реализуется** — это отдельная задача (расширение **`authGuard`** / query **`redirect=`).  

**Требования к тестам 10.7:**
- **Happy path:** авторизованный пользователь — таблицы URL и редиректы.
- **Гость:** один сценарий «ожидаемый увод с **`/projects/...`** без 500 / без монтирования дашборда» — по фактическому контракту **`auth_guard.dart`**, без требования return-URL.

## Архитектурные решения

### 1. `pageBuilder` в prod

**SSOT:** в **`AppRouter`** экраны проектов подключаются через **`pageBuilder`** + **`MaterialPage`** / **`NoTransitionPage`** (как в текущем **`app_router.dart`**) — нужно для согласованного стека страниц и Web history.  

**Тестовый хелпер** может использовать **`builder`** для заглушки списка (**`__TEST_PROJECTS_LIST__`**), но **сегмент `:id` + `StatefulShellRoute`** должен совпадать по матчингу и redirect’ам с prod.

### 2. `StatefulShellRoute` и сохранение state

**Напряжение:** стандартный контейнер **`StatefulShellRoute`** держит **несколько** веток смонтированными → локальный state неактивных вкладок обычно **сохраняется**. Текущий **`navigatorContainerBuilder`**, возвращающий только **`children[currentIndex]`**, **снимает с дерева** неактивные ветки → **локальный state виджетов вкладок при переключении не сохраняется** (скролл, поля ввода в дереве).

**Принятое решение Sprint 10 (явный компромисс):**
- Сохраняем **только активную** ветку в дереве — осознанный trade-off для MVP (проще контролировать жизненный цикл тяжёлых поддеревьев **до** появления полноценных экранов в Sprint 11+).
- Долгоживущее состояние (черновик сообщения, позиция чата), которое должно пережить смену вкладки, проектируется в **Riverpod / контроллере**, привязанном к **`projectId`**, **а не** к виджету ветки.
- Если продукт позже потребует **сохранение локального UI-state** вкладок без подъёма в провайдер — отдельная задача: смена **`navigatorContainerBuilder`** на поведение по умолчанию или **`IndexedStack`** с оценкой памяти/фоновых эффектов.

Это **не** «ещё одно молчаливое» противоречие документации и **`StatefulShellRoute`**: назначение shell здесь — **URL + отдельный Navigator на ветку**, а не обещание кэша вкладок уровня браузера.

### 3. Обоснование глобального redirect (**`projectDashboardUnknownShellBranchRedirect`**)

**Наблюдаемое поведение (не спекуляция):** для URL вида **`/projects/<uuid>/<x>`**, где **`<x>`** не является дочерним **`path`** одной из веток shell, полное совпадение маршрута для **`GoRoute(path: ':id')`** с вложенным shell **может не произойти** так, чтобы отработал **`redirect`** на сегменте **`:id`**. В результате без перехвата на уровне **`GoRouter(redirect: …)`** вызывается **`errorBuilder`**.

**Формализация:** комментарий в коде рядом с **`projectDashboardUnknownShellBranchRedirect`** и этот раздел — **основание для workaround**; при смене **`go_router`** перепроверить интеграционным тестом путь **`/projects/<valid-uuid>/typo`**. Удалять глобальный redirect «вслепую» без этого теста **нельзя**.

### 4. Редиректы (два уровня)

| Функция | Назначение |
|---------|------------|
| **`projectDashboardDetailRedirect`** | На **`GoRoute` `:id`**: невалидный UUID → **`/projects`**; корень дашборда → **`/projects/:id/${projectDashboardDefaultBranch}`** с сохранением query/fragment. |
| **`projectDashboardUnknownShellBranchRedirect`** | На корневом **`GoRouter`**: неизвестный третий сегмент при валидном UUID → дефолтная ветка + сохранение query/fragment. |

### 5. `errorBuilder` и локализация

Хардкод пользовательских строк (**`Text('Error: …')`**) в **`errorBuilder`** **противоречит** правилам проекта (**`CLAUDE.md`**, frontend rules).

При этом **`GoRouterState.error`** полезен для отладки: в **`buildRouterErrorScreen`** при **`!kReleaseMode`** ошибка пишется в консоль (**`debugPrint`**), в UI остаётся только строка из **`.arb`**. В release консольный вывод отключён (см. **`TODO`** в коде для возможной отправки в телеметрию в Sprint 12).

**Один из вариантов приёмки Sprint 10:**
- **(A)** В **10.7** или **10.8**: **`errorBuilder`** использует **`AppLocalizations`** (или общий виджет ошибки из shared) и ключ(и) в **`.arb`**;
- **(B)** Явный подпункт в **10.8**: «нет английского хардкода в **`errorBuilder`**» с отдельным чекбоксом.

Допускается перенос только **(B)** при условии, что **10.8** закрывается в том же спринте и ссылка из этого докуента не «потеряется».

### 6. Имена маршрутов

Использовать **`ProjectRouteNames`** из **`project_dashboard_routes.dart`** в **`app_router.dart`** и в тестах/навигации — **без** размножения строковых литералов имён.
