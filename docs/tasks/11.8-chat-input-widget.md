**Симуляция клавиатуры в тестах (устранение flaky CI):**

- Ориентир API **`WidgetTester`**: **`tester.sendKeyDownEvent` / `sendKeyUpEvent`** с **`LogicalKeyboardKey`** (актуальный путь для **`HardwareKeyboard`** в версии Flutter проекта).
- **Ctrl+Enter:** последовательность вроде **`controlLeft` down → enter down/up → controlLeft up`** (точный порядок — как требует связка **`Shortcuts`** в дереве; зафиксировать рабочую последовательность в PR и не менять без причины).
- **Meta+Enter:** отдельный кейс с **`LogicalKeyboardKey.metaLeft`** (или **`metaRight`**).
- **Не** смешивать проверку **Ctrl** и **Meta** в одном тесте так, что падение не различает платформу — два именованных теста (**`..._ctrl_enter`** и **`..._meta_enter`**).
- Цель: оба сочетания нажимаемы **независимо от** **`defaultTargetPlatform`** раннера CI (Linux vs macOS), как и два **`SingleActivator`** в **`Shortcuts`**.

**TextScaler (`chat_input_text_scaler_2_no_overflow_usable_send`):**

- Обёртка **`MediaQuery(textScaler: TextScaler.linear(2.0))`** над **`ChatInput`**.

- **`tester.tap`** по **`ValueKey('chat_send_button')`** при непустом поле — без исключения.
- Длинный многострочный текст + **`maxLines: 6`** — **`scrollUntilVisible`** к концу текста **или** эквивалент доступности прокрутки поля.

## Контракт с контроллером (11.4 ↔ 11.5 ↔ 11.8)

### Владение `TextEditingController` и `FocusNode`

- **`ChatInput`** **не владеет** переданными **`TextEditingController`** и **`FocusNode`**.
- **`dispose()`** для них вызывает **только родитель** (**`ChatScreen`** / **`ConsumerState`** экрана). В **`ChatInput`** **запрещено** вызывать **`controller.dispose()`** / **`focusNode.dispose()`** (в т.ч. из **`State.dispose`** обёртки-стейта).
- Внутри **`ChatInput`** не держать **`StatefulWidget`** ради «освобождения» контроллера. Если когда-либо нужна подписка на **`controller`** вне **`ValueListenableBuilder`** — симметричный **`addListener`/`removeListener`** только в том же виджете, который их добавил, и **без** **`dispose`** контроллера.
- Виджет **не мутирует** **`controller`** программно **кроме** того, что делает привязанный **`TextField`** (пользовательский ввод). **`controller.clear()`** после успешной отправки выполняет **родитель** (см. § **Состояние отправки**).

### Отправка и `onSend`

- При пользовательском действии «отправить» **`ChatInput`** вызывает **`onSend(controller.text)`** только если:
  - **`controller.text.trim().isNotEmpty`**,
  - **`!isSending`**,
  - для shortcut — дополнительно § **IME-композиция**,
  - **`trim`** для тела HTTP по-прежнему только в **`ChatController.send`**.
- Родитель: **`ref.read(...notifier).send(raw)`**. Идемпотентность при двойном клике / гонке shortcut+tap — **гард на стороне родителя** (игнор второго **`send`** пока **`Future`** активен **или** **`isSending`** синхронно **`true`**).

### Расширение под вложения

Не ломать **`send(String raw)`**; **`file_picker`** не в **11.8** (§ **Вне скоупа**).

## Состояние отправки (in-flight)

Зафиксировать единое UX; иначе реализации разъедутся.

| Вопрос | Решение |
|--------|---------|
| Кнопка «Отправить» пока **`send`** не завершился | **`onPressed == null`** при **`isSending == true`** (блокировка и клика, и визуально неактивное состояние по теме). |
| Shortcut **Ctrl/Meta+Enter** во время отправки | **Не** вызывать **`onSend`** при **`isSending`**. |
| Двойной клик / клик + shortcut | Два вызова **`onSend`** недопустимы: родитель выставляет **`isSending`** и/или игнорирует повтор до резолва; **`ChatInput`** не вызывает **`onSend`**, если **`isSending`**. |
| Очистка поля | Только после **успешного** завершения **`send`** (**или** принятой политики «сообщение принято сервером»): **`controller.clear()`** вызывает **родитель**. |
| Ошибка отправки | Текст **остаётся** в поле для правки / **`retrySend`** (**11.4**); **`ChatInput`** ничего не затирает при ошибке. |
| Фокус после успеха | **`focusNode.requestFocus()`** — **родитель** после **`clear()`** (как дух **11.5**). |

**Параметр API:** **`final bool isSending;`** (**default `false`**). Источник — родитель (например **`asyncChat.isLoading`** по операции **`send`**, отдельный флаг в **`ChatController`** — зафиксировать в PR один способ).

## Публичный API (канон)

```dart
class ChatInput extends StatelessWidget {
  const ChatInput({
    super.key,
    required this.controller,
    required this.focusNode,
    required this.onSend,
    this.onStop,
    this.onAttach,
    this.isStopActive = false,
    this.isSending = false,
    this.hintText,
  });

  final TextEditingController controller;
  final FocusNode focusNode;

  /// Сырой [controller.text]; только если trim не пустой, !isSending, IME — § ниже.
  final ValueChanged<String> onSend;

  final VoidCallback? onStop;
  final VoidCallback? onAttach;

  /// Подсветка «идёт операция»; видимость стопа только при [onStop != null].
  final bool isStopActive;

  /// Блокирует кнопку отправки и shortcut (§ Состояние отправки).
  final bool isSending;

  /// Placeholder для InputDecoration.hintText: уже локализованная строка от родителя
  /// (например AppLocalizations.of(context)!.chatInputHint или перенесённый chatScreenInputHint).
  /// При null hint не показывается. ChatInput не импортирует AppLocalizations — проще юнит-тесты.
  final String? hintText;
}
```

**Порядок в `Row` (LTR):** слева направо — **`attach`** (если **`onAttach != null`**), затем **`Expanded`** с **`TextField`**, затем **`stop`** (если **`onStop != null`**), затем **`send`**. Отступы между слотами — **`SizedBox(width: 8)`** или эквивалент из темы; цель — один ряд без bikeshedding на ревью.

**Видимость кнопок:** отправка — всегда; стоп — **`onStop != null`**; attach — **`onAttach != null`**.

**Активация «Отправить»:** **`onPressed`** только если **`controller.text.trim().isNotEmpty`** **и** **`!isSending`**.

## Производительность

- **Не** оборачивать весь **`ChatInput`** в **`ListenableBuilder`** по **`controller`**: на каждый символ перерисовывались бы **attach/stop**, не зависящие от текста.
- Слушать **`controller`** **точечно** для кнопки отправки: например отдельный приватный виджет **`_SendButton`** с **`ValueListenableBuilder<TextEditingValue>`** / **`ListenableBuilder`** **только** вокруг этой кнопки (и учёт **`isSending`** снаружи без лишних ребилдов **`TextField`**).

## IME-композиция (CJK / диктовка)

Пока **`controller.value.composing.isValid`** (**активная композиция IME**), shortcut **Ctrl/Meta+Enter** **не** должен вызывать **`onSend`** (иначе уходит недопечатанное слово). Реализация: проверка **`composing`** в **`CallbackAction`** / общем обработчике перед **`onSend`**.

Тест — **`chat_input_on_send_not_called_during_ime_composition`** (см. таблицу имён).

- **`IconButton.filled`** (M3): передать **`tooltip`** через **встроенный параметр** **`IconButton.tooltip`** — **не** оборачивать во **второй** **`Tooltip`** (двойная обёртка ломает hit-test и дублирует semantics).
- Активация — § **Производительность** и **`isSending`**.

## Кнопка «Стоп» / «Вложения»

| Тема | Правило |
|------|---------|
| Видимость | Кнопка **Стоп** только при **`onStop != null`**; кнопка **Вложения** только при **`onAttach != null`**. Если **`null`** — слота нет (**не** «disabled навсегда» на пустом месте). |
| Подписи | **`tooltip`** / семантика из **`l10n`**: **`chatInputStopTooltip`** — явно про **отмену исходящего запроса** (HTTP **`send`**); **не** про «остановить ответ ассистента». **`chatInputAttachTooltip`** — вложения. Разделение с «ответом бота» — чтобы не путать со **Sprint 12** (отмена **задачи** на сервере — другой UX/API). |
| **`isStopActive`** | Визуальная подсветка «идёт операция» (ожидание **`send`** или позже стрим **11.9**); **не** заменяет правило видимости по **`onStop`**. |
| Нажатие | При **`onStop != null`** и активном **`onPressed`** → **`onStop()`**. Attach → **`onAttach()`**. |
| Отмена HTTP | Целевое расширение **11.4**: **`CancelToken`** на **`sendMessage`**; **`onStop`** → **`cancel()`** → **`ConversationCancelledException`**. До готовности — поведение зафиксировать в PR (может быть no-op кнопка «Стоп», если токена ещё нет — тогда **`tooltip`** не обещает отмену). |

## Доступность и семантика

- **`Tooltip`** (desktop/web hover): параметр **`IconButton`** / **`tooltip`** у кнопки.
- **Мобильные** (нет hover): экранные читалки опираются на **`Semantics`** — убедиться, что подпись непустая (**`IconButton`** с **`tooltip`** задаёт семантику на большинстве платформ; при необходимости **`Semantics(label: …)`** не дублирует конфликтующий текст).
- Тест: **`find.bySemanticsLabel(...)`** с строкой из **`l10n`** для кнопки отправки — см. таблицу тестов.
