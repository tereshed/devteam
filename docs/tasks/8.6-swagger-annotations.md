# 8.6 Swagger-аннотации для Conversation API

**Sprint:** 8 — Conversation API (Backend)
**Файлы:** `backend/internal/handler/conversation_handler.go`
**Статус:** ✅ Завершено

---

## Описание

Добавление и стандартизация Swagger-аннотаций для эндпоинтов `ConversationHandler`. Задача гарантирует, что API чатов корректно отображается в Swagger UI, поддерживает три способа авторизации и содержит полные описания параметров и ответов.

---

## Стандарт авторизации

Согласно общему стандарту проекта (см. [2.9-swagger-annotations.md](2.9-swagger-annotations.md)), каждый защищённый эндпоинт **ОБЯЗАН** иметь три `@Security` на **отдельных строках**:

```go
// @Security BearerAuth
// @Security ApiKeyAuth
// @Security OAuth2Password
```

---

## Каталог аннотаций для ConversationHandler

### 1. CreateConversation (`POST /projects/{project_id}/conversations`)

```go
// CreateConversation создает новый чат
// @Summary Создание чата
// @Description Создает новый чат в указанном проекте для текущего пользователя
// @Tags conversations
// @Security BearerAuth
// @Security ApiKeyAuth
// @Security OAuth2Password
// @Accept json
// @Produce json
// @Param project_id path string true "Project ID"
// @Param request body dto.CreateConversationRequest true "Данные чата"
// @Success 201 {object} dto.ConversationResponse
// @Failure 400 {object} apierror.ErrorResponse "Невалидный UUID или JSON"
// @Failure 401 {object} apierror.ErrorResponse "Не авторизован"
// @Failure 403 {object} apierror.ErrorResponse "Нет доступа к проекту"
// @Failure 404 {object} apierror.ErrorResponse "Проект не найден"
// @Failure 500 {object} apierror.ErrorResponse "Внутренняя ошибка"
// @Router /projects/{project_id}/conversations [post]
```

### 2. ListConversations (`GET /projects/{project_id}/conversations`)

```go
// ListConversations возвращает список чатов проекта
// @Summary Список чатов проекта

// @Tags conversations

// @Security ApiKeyAuth
// @Security OAuth2Password
// @Produce json

// @Success 200 {object} dto.ConversationListResponse
// @Failure 400 {object} apierror.ErrorResponse "Невалидный UUID или параметры пагинации"

```

### 4. SendMessage (`POST /conversations/{conversation_id}/messages`)

```go
// SendMessage отправляет сообщение в чат
// @Summary Отправка сообщения
// @Description Добавляет сообщение пользователя в чат и запускает процесс оркестрации. Поддерживает идемпотентность через заголовок X-Client-Message-ID.
// @Tags conversation-messages
// @Security BearerAuth
// @Security ApiKeyAuth
// @Security OAuth2Password
// @Accept json
// @Produce json
// @Param conversation_id path string true "Conversation ID"
// @Param X-Client-Message-ID header string false "Ключ идемпотентности (UUIDv4)"
// @Param request body dto.CreateMessageRequest true "Текст сообщения"
// @Success 201 {object} dto.MessageResponse
// @Failure 400 {object} apierror.ErrorResponse "Невалидный JSON, пустой контент или невалидный ID идемпотентности"
// @Failure 401 {object} apierror.ErrorResponse "Не авторизован"
// @Failure 403 {object} apierror.ErrorResponse "Нет доступа к чату"
// @Failure 404 {object} apierror.ErrorResponse "Чат не найден"
// @Failure 429 {object} apierror.ErrorResponse "Слишком много запросов"
// @Failure 500 {object} apierror.ErrorResponse "Внутренняя ошибка"
// @Router /conversations/{conversation_id}/messages [post]
```

## Группировка (@Tags)

| Tag | Эндпоинты | Описание |
|-----|-----------|----------|
| `conversations` | Create, List, Get, Delete | Управление чатами как сущностями |
| `conversation-messages` | SendMessage, GetHistory | Работа с контентом внутри чата |

---

## Критерии приёмки

- [x] Все 6 эндпоинтов имеют корректные аннотации `@Security` (OR-семантика).
- [x] `@Router` пути не содержат префикс `/api/v1`.
- [x] Описан заголовок `X-Client-Message-ID` для `SendMessage`.
- [x] Используются правильные DTO для `@Success` и `@Param body`.
- [x] `make swagger` выполняется без ошибок.
- [x] Swagger UI корректно отображает разделение на `conversations` и `conversation-messages`.
