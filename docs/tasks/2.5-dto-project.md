## Что нужно добавить

### ProjectResponse

Ответ на `GET /projects/:id`. Содержит все поля проекта + вложенные summary для credential и team.

```go
type ProjectResponse struct {
    ID               string                `json:"id"`
    Name             string                `json:"name"`
    Description      string                `json:"description"`
    GitProvider      string                `json:"git_provider"`
    GitURL           string                `json:"git_url"`
    GitDefaultBranch string                `json:"git_default_branch"`
    GitCredential    *GitCredentialSummary `json:"git_credential,omitempty"`
    VectorCollection string                `json:"vector_collection"`
    TechStack        datatypes.JSON        `json:"tech_stack" swaggertype:"string"`
    Status           string                `json:"status"`
    Settings         datatypes.JSON        `json:"settings" swaggertype:"string"`
    CreatedAt        time.Time             `json:"created_at"`
    UpdatedAt        time.Time             `json:"updated_at"`
}
```

**`GitCredential`** — не полный объект, а summary (без секретов):

```go
type GitCredentialSummary struct {
    ID       string `json:"id"`
    Provider string `json:"provider"`
    AuthType string `json:"auth_type"`
    Label    string `json:"label"`
}
```

### ProjectListResponse

Обёртка для пагинированного списка.

```go
type ProjectListResponse struct {
    Projects []ProjectResponse `json:"projects"`
    Total    int64             `json:"total"`
    Limit    int               `json:"limit"`
    Offset   int               `json:"offset"`
}
```

### TeamSummaryResponse

Краткая информация о команде (для вложения в `ProjectResponse` при необходимости).

```go
type TeamSummaryResponse struct {
    ID         string `json:"id"`
    Name       string `json:"name"`
    Type       string `json:"type"`
    AgentCount int    `json:"agent_count"`
}
```

## Конвертеры (model → DTO)

```go
func ToProjectResponse(p *models.Project) ProjectResponse
func ToProjectListResponse(projects []models.Project, total int64, limit, offset int) ProjectListResponse
func ToGitCredentialSummary(gc *models.GitCredential) *GitCredentialSummary
```

**Почему конвертеры, а не метод модели:**
- Модель не должна знать о DTO (Clean Architecture)
- Конвертер живёт в пакете `dto` — ближе к handler-слою
- Упрощает тестирование (чистые функции)

## Валидация binding-тегов

### CreateProjectRequest (уточнения)

| Поле | Текущий binding | Дополнить |
|------|----------------|-----------|
| `name` | `binding:"required"` | Добавить `binding:"required,min=1,max=255"` |
| `git_provider` | — | Валидация в Service (`IsValid()`) |
| `git_url` | — | Добавить `binding:"omitempty,url"` (если указан — должен быть валидным URL) |
| `status` | — | Валидация в Service |

### ListProjectsRequest (query params)

Все поля приходят через query string. Handler парсит их из `c.Query()` / `c.DefaultQuery()`.

```go
type ListProjectsRequest struct {
    Status      *string `form:"status"`
    GitProvider *string `form:"git_provider"`
    Search      *string `form:"search"`
    Limit       int     `form:"limit"`
    Offset      int     `form:"offset"`
    OrderBy     string  `form:"order_by"`
    OrderDir    string  `form:"order_dir"`
}
```

**Примечание:** сменить `json` теги на `form` для query params (Gin `ShouldBindQuery` использует `form` тег).
