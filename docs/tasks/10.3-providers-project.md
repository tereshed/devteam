## Цель

Создать слой Riverpod-провайдеров для чтения данных проектов, используемых на экранах списка и деталей. Провайдеры должны:

- Использовать существующий `projectRepositoryProvider`
- Поддерживать отмену запросов через `CancelToken` при dispose провайдера
- Соответствовать стилю аналогичных провайдеров в проекте — см. `features/admin/workflows/data/workflows_providers.dart` (function-based `@riverpod` с параметрами)

---

## Архитектурные решения

### Function-based, а не class-based

Используем **функциональный** стиль `@riverpod` (как в `workflows_providers.dart`), а не class-based AsyncNotifier.

**Обоснование:**
- В задаче 10.3 нет мутирующих методов (только чтение и инвалидация извне)
- Class-based AsyncNotifier оправдан только когда нужно изменять состояние из UI (например, optimistic updates, локальная фильтрация)
- Refresh/инвалидация выполняются через `ref.refresh(...)` / `ref.invalidate(...)` — это методы `Ref/WidgetRef`, а не методы провайдера
- Для будущих задач 10.5+ (создание/обновление проекта) будут отдельные class-based notifier'ы — там это оправдано

### Жизненный цикл (autoDispose vs keepAlive)

| Провайдер | Lifecycle | Обоснование |
|-----------|-----------|-------------|
| `projectRepositoryProvider` | `keepAlive: true` | Stateless DI — нет смысла пересоздавать |
| `projectListProvider` | autoDispose (по умолчанию) | Список перезагружается при возврате на экран; объёмные данные не должны висеть в памяти |
| `projectProvider(id)` | autoDispose (по умолчанию) | Аналогично — конкретный проект освобождается при уходе с экрана |

Для `projectListProvider` в будущем можно добавить `@Riverpod(keepAlive: true)` если UX потребует мгновенный возврат на экран без перезагрузки.

### Отмена запросов (CancelToken)

`ProjectRepository.listProjects` и `getProject` принимают `CancelToken`. Провайдеры **обязаны** его создавать и привязывать к жизненному циклу через `ref.onDispose`. Это:

- Отменяет запросы при смене параметров (`filter`, `id`)
- Отменяет запросы при уходе с экрана (autoDispose → dispose → cancel)
- Предотвращает race conditions при быстром переключении фильтров

### Валидация входных данных

**Не дублируется в провайдере.** `ProjectRepository` уже:
- Валидирует пустой `id` (бросает `ArgumentError`)
- Нормализует `limit` (1..100) и `offset` (≥0)

Провайдер просто проксирует параметры в репозиторий.

## Реализация

### `projectListProvider`

Function-based async-провайдер для списка проектов с пагинацией и фильтрацией.

```dart
@riverpod
Future<ProjectListResponse> projectList(
  Ref ref, {
  ProjectListFilter? filter,
  int limit = 20,
  int offset = 0,
}) {
  final cancelToken = CancelToken();
  ref.onDispose(cancelToken.cancel);

  return ref.watch(projectRepositoryProvider).listProjects(
        filter: filter,
        limit: limit,
        offset: offset,
        cancelToken: cancelToken,
      );
}
```

**Поведение Riverpod при параметрах:** разные комбинации `(filter, limit, offset)` создают **разные кэши**. Например, `projectListProvider(filter: active, limit: 10)` и `projectListProvider(filter: archived, limit: 10)` живут независимо.

### `projectProvider(String id)`

Function-based async-провайдер для одного проекта.

```dart
@riverpod
Future<ProjectModel> project(Ref ref, String id) {
  final cancelToken = CancelToken();
  ref.onDispose(cancelToken.cancel);

  return ref.watch(projectRepositoryProvider).getProject(
        id,
        cancelToken: cancelToken,
      );
}
```

**Кэширование:** каждый `id` кэшируется отдельно. `projectProvider('abc')` и `projectProvider('def')` — независимые экземпляры.

### Полный список новых импортов

```dart
import 'package:dio/dio.dart';                                    // CancelToken
import 'package:frontend/features/projects/domain/models.dart';   // ProjectModel (re-export)
import 'package:frontend/features/projects/domain/requests.dart'; // ProjectListFilter, ProjectListResponse
```

`models.dart` — это re-export файл (`export 'models/project_model.dart'`), импорт корректен.

## Управление кэшем (вызывается из UI / других провайдеров)

`refresh` и `invalidate` — это методы `Ref` / `WidgetRef`, **не методы провайдера**. Они описаны здесь как референс для UI-кода.

### `ref.refresh(provider)`

Принудительно перезагружает провайдер и возвращает новый Future.

```dart
final freshData = await ref.refresh(projectListProvider(filter: active).future);
```

**Когда использовать:** pull-to-refresh, кнопка "Обновить".

### `ref.invalidate(provider)`

Помечает провайдер как устаревший. Перезапрос происходит **лениво** — при следующем `watch`.

```dart
// После создания/удаления проекта (в будущей задаче 10.5):
ref.invalidate(projectListProvider);  // инвалидирует ВСЕ комбинации параметров
ref.invalidate(projectProvider(id));  // инвалидирует конкретный проект
```

**Когда использовать:** мутации (POST/PUT/DELETE), которые могли изменить серверные данные.

## Обработка ошибок

Провайдеры **не ловят** исключения из репозитория — они пробрасываются как `AsyncError` в `AsyncValue`. UI обрабатывает ошибки через `.when(error: ...)`.

Возможные исключения (определены в `domain/project_exceptions.dart`):

| Класс | Когда | UI-реакция |
|-------|-------|------------|
| `UnauthorizedException` | 401 — токен истёк | Redirect на `/login` через `go_router` |
| `ProjectForbiddenException` | 403 — нет прав | Экран "Нет доступа" |
| `ProjectNotFoundException` | 404 — проект не найден | Экран "Проект не найден" |
| `ProjectConflictException` | 409 — конфликт (релевантно для будущих задач 10.5+) | Сообщение из `e.message` |
| `ProjectApiException` | прочие ошибки API + сетевые таймауты | Общий экран ошибки + retry |
| `ArgumentError` | пустой `id` | Программная ошибка — не должна доходить до UI |

### Маскирование ошибок в UI

**КРИТИЧНО:** Не выводить `err.toString()` напрямую — может содержать URL с токеном, stack trace, внутренние сообщения бэкенда. Логировать отдельно, в UI показывать локализованный текст.

```dart
error: (err, stack) {
  debugPrint('projectList error: $err\n$stack');
  return ErrorView(message: context.l10n.errorLoadingProjects);
}
```

## Примеры использования в UI

> ⚠️ Все строки в примерах ниже **обязаны** браться из `context.l10n.*` (см. `frontend.md` §2.3). Литералы используются здесь только для краткости — в реальном коде заменить на ключи локализации в `l10n_en.arb` / `l10n_ru.arb`.

### Список проектов

```dart
class ProjectListScreen extends ConsumerWidget {
  const ProjectListScreen({super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final asyncProjects = ref.watch(
      projectListProvider(
        filter: const ProjectListFilter(status: 'active'),
        limit: 20,
      ),
    );

    return RefreshIndicator(
      onRefresh: () => ref.refresh(
        projectListProvider(
          filter: const ProjectListFilter(status: 'active'),
          limit: 20,
        ).future,
      ),
      child: asyncProjects.when(
        data: (response) => ListView.builder(
          itemCount: response.projects.length,
          itemBuilder: (_, i) => ProjectCard(project: response.projects[i]),
        ),
        loading: () => const Center(child: CircularProgressIndicator()),
        error: (err, stack) {
          debugPrint('projectList error: $err');
          return ErrorView(message: context.l10n.errorLoadingProjects);
        },
      ),
    );
  }
}
```

### Деталь проекта

```dart
class ProjectDetailScreen extends ConsumerWidget {
  final String projectId;
  const ProjectDetailScreen({required this.projectId, super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    return ref.watch(projectProvider(projectId)).when(
          data: (project) => ProjectDetailView(project: project),
          loading: () => const Center(child: CircularProgressIndicator()),
          error: (err, stack) {
            debugPrint('project($projectId) error: $err');
            if (err is ProjectNotFoundException) {
              return ErrorView(message: context.l10n.projectNotFound);
            }
            if (err is ProjectForbiddenException) {
              return ErrorView(message: context.l10n.projectForbidden);
            }
            return ErrorView(message: context.l10n.errorLoadingProject);
          },
        );
  }
}
```

### Интеграция с `go_router`

```dart
GoRoute(
  path: '/projects/:id',
  builder: (context, state) => ProjectDetailScreen(
    projectId: state.pathParameters['id']!,
  ),
),
```

`projectProvider(id)` сам по себе кэширует данные по `id` — никакого дополнительного state-менеджмента в роутере не требуется.

## Тесты

**Файл:** `frontend/test/features/projects/data/project_providers_test.dart`

### Стратегия мокирования

В проекте используется **`mocktail`** (см. `test/features/auth/data/auth_repository_test.dart`). Использовать его для согласованности.

> Существующий `project_repository_test.dart` использует `mockito` — это исключение, оставленное для совместимости с задачей 10.2. Новые тесты пишем на `mocktail`.

Мокаем **`ProjectRepository`**, а не `Dio`. Это:
- Изолирует тесты провайдеров от детали репозитория
- Не дублирует уже существующие тесты `ProjectRepository`
- Не делает реальных сетевых запросов

### Обязательные сценарии

1. **`projectListProvider`:**
   - Успешная загрузка списка с фильтром
   - Проброс `UnauthorizedException` в `AsyncError`
   - Проброс `ProjectApiException` (5xx, network) в `AsyncError`
   - Разные параметры → разные вызовы репозитория (проверка кэширования по параметрам)

2. **`projectProvider`:**
   - Успешная загрузка по `id`
   - Проброс `ProjectNotFoundException` (404)
   - Проброс `ProjectForbiddenException` (403)
   - Проброс `ArgumentError` для пустого `id` (репозиторий бросает — провайдер пропускает)

3. **CancelToken:**
   - При `container.dispose()` вызывается `cancelToken.cancel()`
   - Проверяется через мок: захват `CancelToken` из аргументов и проверка `isCancelled`

### Шаблон теста

```dart
import 'package:dio/dio.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:mocktail/mocktail.dart';
import 'package:frontend/features/projects/data/project_providers.dart';
import 'package:frontend/features/projects/data/project_repository.dart';
import 'package:frontend/features/projects/domain/models.dart';
import 'package:frontend/features/projects/domain/project_exceptions.dart';
import 'package:frontend/features/projects/domain/requests.dart';

class MockProjectRepository extends Mock implements ProjectRepository {}

class FakeProjectListFilter extends Fake implements ProjectListFilter {}

void main() {
  setUpAll(() {
    registerFallbackValue(FakeProjectListFilter());
  });

  late MockProjectRepository repo;

  ProviderContainer makeContainer() {
    final container = ProviderContainer(
      overrides: [
        projectRepositoryProvider.overrideWithValue(repo),
      ],
    );
    addTearDown(container.dispose);
    return container;
  }

  setUp(() {
    repo = MockProjectRepository();
  });

  group('projectListProvider', () {
    test('loads projects via repository', () async {
      const response = ProjectListResponse(
        projects: [],
        total: 0,
        limit: 20,
        offset: 0,
      );
      when(() => repo.listProjects(
            filter: any(named: 'filter'),
            limit: any(named: 'limit'),
            offset: any(named: 'offset'),
            cancelToken: any(named: 'cancelToken'),
          )).thenAnswer((_) async => response);

      final container = makeContainer();
      final result = await container.read(projectListProvider().future);

      expect(result, response);
      verify(() => repo.listProjects(
            filter: null,
            limit: 20,
            offset: 0,
            cancelToken: any(named: 'cancelToken'),
          )).called(1);
    });

    test('cancels request on dispose', () async {
      CancelToken? captured;
      when(() => repo.listProjects(
            filter: any(named: 'filter'),
            limit: any(named: 'limit'),
            offset: any(named: 'offset'),
            cancelToken: any(named: 'cancelToken'),
          )).thenAnswer((invocation) async {
        captured = invocation.namedArguments[#cancelToken] as CancelToken?;
        return const ProjectListResponse(
          projects: [],
          total: 0,
          limit: 20,
          offset: 0,
        );
      });

      final container = makeContainer();
      await container.read(projectListProvider().future);
      container.dispose();

      expect(captured?.isCancelled, isTrue);
    });

    test('propagates UnauthorizedException as AsyncError', () async {
      when(() => repo.listProjects(
            filter: any(named: 'filter'),
            limit: any(named: 'limit'),
            offset: any(named: 'offset'),
            cancelToken: any(named: 'cancelToken'),
          )).thenThrow(UnauthorizedException('token expired'));

      final container = makeContainer();

      await expectLater(
        container.read(projectListProvider().future),
        throwsA(isA<UnauthorizedException>()),
      );
    });
  });

  group('projectProvider', () {
    test('loads project by id', () async {
      final project = ProjectModel(
        id: 'abc-123',
        name: 'Test',
        // ... остальные обязательные поля из ProjectModel
      );
      when(() => repo.getProject(
            'abc-123',
            cancelToken: any(named: 'cancelToken'),
          )).thenAnswer((_) async => project);

      final container = makeContainer();
      final result = await container.read(projectProvider('abc-123').future);

      expect(result.id, 'abc-123');
    });

    test('propagates ProjectNotFoundException as AsyncError', () async {
      when(() => repo.getProject(
            any(),
            cancelToken: any(named: 'cancelToken'),
          )).thenThrow(ProjectNotFoundException('not found'));

      final container = makeContainer();

      await expectLater(
        container.read(projectProvider('xxx').future),
        throwsA(isA<ProjectNotFoundException>()),
      );
    });
  });
}
```
