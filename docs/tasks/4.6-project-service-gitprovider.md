# 4.6 Service: интеграция GitProvider в ProjectService

**Sprint:** 4 — Git Provider Integration (Backend)
**Файл:** `backend/internal/service/project_service.go`
**Статус:** ⬜ Не начата

---

## Описание

Интегрировать `gitprovider.Factory` в `ProjectService`, чтобы при создании проекта с remote-провайдером (GitHub, GitLab, Bitbucket) выполнялись:

1. **Расшифровка credentials** — из `GitCredential.EncryptedValue` в `gitprovider.Credentials`
2. **Валидация доступа** — `provider.ValidateAccess(ctx, gitURL)` перед сохранением в БД
3. **Получение метаданных** — `provider.GetRepoInfo(ctx, gitURL)` для автозаполнения (default branch)
4. **Клонирование для индексации** — `provider.Clone(ctx, gitURL, opts)` (подготовка к Sprint 6: векторная индексация)

**Текущее состояние:** `ProjectService.Create` только сохраняет проект в БД и создаёт команду «Development Team» в одной транзакции. Никаких Git-операций не выполняется; `encryptionKey` передаётся в конструктор, но не используется.

### 5. Вспомогательный метод: создание GitProvider из проекта

```go
// buildGitProvider расшифровывает credentials и создаёт экземпляр GitProvider.
// Вызывается из Create и Update при наличии remote-провайдера.
func (s *projectService) buildGitProvider(
    ctx context.Context,
    providerType models.GitProvider,
    credentialID *uuid.UUID,
    userID uuid.UUID,
) (gitprovider.GitProvider, error) {
    creds := gitprovider.Credentials{}

    if credentialID != nil {
        // 1. Получить credential
        gitCred, err := s.gitCredRepo.GetByID(ctx, *credentialID)
        if err != nil {
            return nil, mapGitCredRepoErr(err)
        }

        // 2. Проверить принадлежность
        if gitCred.UserID != userID {
            return nil, ErrGitCredentialForbidden
        }

        // 3. Расшифровать
        decrypted, err := s.decryptor.Decrypt(gitCred.EncryptedValue)
        if err != nil {
            return nil, fmt.Errorf("%w: %v", ErrDecryptionFailed, err)
        }

        // 4. Маппинг: models.GitCredentialAuthType → gitprovider.Credentials
        switch gitCred.AuthType {
        case models.GitCredentialAuthToken, models.GitCredentialAuthOAuth:
            creds.Token = string(decrypted)
        case models.GitCredentialAuthSSHKey:
            creds.SSHKey = string(decrypted)
        }
    }

    // 5. Создать провайдер через фабрику
    provider, err := s.gitFactory.Create(string(providerType), creds)
    if err != nil {
        return nil, fmt.Errorf("create git provider: %w", err)
    }

    return provider, nil
}
```

### 6. Обновление метода `Create`

Добавить Git-операции **перед** транзакцией (Git-вызовы не должны быть внутри транзакции БД).

**⚠️ Ключевое изменение:** Убран вызов `ensureGitCredentialOwned` — его работу (SELECT + проверка владельца) теперь выполняет `buildGitProvider`, чтобы не было дублирования запросов в БД. Провайдер, созданный до транзакции, **переиспользуется** в фоновой горутине клонирования (не лезем в БД и AES повторно).

```go
func (s *projectService) Create(ctx context.Context, userID uuid.UUID, req dto.CreateProjectRequest) (*models.Project, error) {
    // === Валидация полей ===
    name := strings.TrimSpace(req.Name)
    if name == "" {
        return nil, ErrProjectInvalidName
    }
    gp, err := parseGitProvider(req.GitProvider)
    if err != nil {
        return nil, err
    }
    status, err := parseProjectStatus(req.Status)
    if err != nil {
        return nil, err
    }

    isRemote := gp != models.GitProviderLocal

    // Local-провайдер не работает с credentials — если клиент передал, это ошибка ввода.
    if !isRemote && req.GitCredentialID != nil {
        return nil, ErrGitCredentialNotSupportedForLocal
    }

    branch := strings.TrimSpace(req.GitDefaultBranch)
    if branch == "" {
        branch = "main"
    }

    gitURL := strings.TrimSpace(req.GitURL)

    // === Git-валидация для remote-провайдеров ===
    // buildGitProvider сам проверяет ownership credential (заменяет ensureGitCredentialOwned).
    // Провайдер сохраняется для переиспользования в cloneForIndexing.
    var provider gitprovider.GitProvider

    if isRemote && gitURL != "" {
        provider, err = s.buildGitProvider(ctx, gp, req.GitCredentialID, userID)
        if err != nil {
            return nil, err
        }

        // 1. Валидация доступа к репозиторию
        if err := provider.ValidateAccess(ctx, gitURL); err != nil {
            return nil, mapGitProviderErr(err)
        }

        // 2. Получение метаданных (автозаполнение default branch)
        repoInfo, err := provider.GetRepoInfo(ctx, gitURL)
        if err == nil && repoInfo != nil {
            if branch == "main" && repoInfo.DefaultBranch != "" {
                branch = repoInfo.DefaultBranch
            }
        }
        // Ошибка GetRepoInfo не критична — продолжаем с дефолтной веткой
    } else if isRemote && req.GitCredentialID != nil {
        // Remote без URL, но с credential — проверяем ownership через buildGitProvider
        // (provider не сохраняем, clone не запускаем — нет URL).
        if _, err := s.buildGitProvider(ctx, gp, req.GitCredentialID, userID); err != nil {
            return nil, err
        }
    }

    // === Создание записи в БД ===
    project := &models.Project{
        Name:             name,
        Description:      req.Description,
        GitProvider:      gp,
        GitURL:           gitURL,
        GitDefaultBranch: branch,
        GitCredentialsID: req.GitCredentialID,
        VectorCollection: req.VectorCollection,
        TechStack:        req.TechStack,
        Status:           status,
        Settings:         req.Settings,
        UserID:           userID,
    }

    err = s.transactions.WithTransaction(ctx, func(txCtx context.Context) error {
        if err := s.projectRepo.Create(txCtx, project); err != nil {
            return mapProjectRepoErr(err)
        }
        team := &models.Team{
            Name:      devTeamDefaultName,
            ProjectID: project.ID,
            Type:      models.TeamTypeDevelopment,
        }
        return s.teamRepo.Create(txCtx, team)
    })
    if err != nil {
        return nil, err
    }

    // === Асинхронное клонирование для индексации ===
    // Передаём уже созданный provider (без повторных запросов в БД и расшифровки).
    // Передаём примитивы вместо указателя *models.Project (защита от data race).
    if provider != nil && s.importDir != "" {
        go s.cloneForIndexing(provider, project.ID, gitURL, branch)
    }

    return project, nil
}
```

### 7. Метод клонирования для индексации

**⚠️ Ключевые решения по ревью:**
- Метод принимает **уже созданный** `provider` — нет повторных запросов в БД и расшифровки.
- Принимает **примитивы** (`projectID`, `gitURL`, `branch`), а не указатель `*models.Project` — защита от data race, если другой горутиной/middleware объект будет мутирован.
- Первой строкой — `defer recover()` — panic внутри Clone (или глубже) не убьёт весь сервер.
- Использует `context.WithTimeout` (10 минут) — клонирование больших репо не зависнет навсегда и будет корректно отменено при graceful shutdown (если заменить `context.Background()` на контекст с cancel при shutdown).
- Использует `log.Printf` (стандартный логгер), так как в проекте пока не внедрён структурированный логгер. **TODO:** при внедрении `slog`/`zap` заменить на `s.logger.Info(...)` / `s.logger.Error(...)`.

```go
const cloneTimeout = 10 * time.Minute

// cloneForIndexing клонирует репозиторий во временную директорию для будущей индексации.
// Запускается в фоне (горутина) — ошибки логируются, не влияют на ответ API.
// Sprint 6+: после клонирования будет запущена индексация в Weaviate.
//
// provider — уже инициализированный GitProvider (переиспользуется из Create, без повторных обращений в БД).
// Принимает примитивы вместо *models.Project для защиты от data race.
func (s *projectService) cloneForIndexing(
    provider gitprovider.GitProvider,
    projectID uuid.UUID,
    gitURL string,
    branch string,
) {
    defer func() {
        if r := recover(); r != nil {
            log.Printf("[project:%s] panic in cloneForIndexing: %v", projectID, r)
        }
    }()

    cloneCtx, cancel := context.WithTimeout(context.Background(), cloneTimeout)
    defer cancel()

    workDir := filepath.Join(s.importDir, projectID.String())

    if err := provider.Clone(cloneCtx, gitURL, gitprovider.CloneOptions{
        Branch:   branch,
        DestPath: workDir,
        Depth:    0,
    }); err != nil {
        log.Printf("[project:%s] clone for indexing failed: %v", projectID, err)
        os.RemoveAll(workDir)
        return
    }

    log.Printf("[project:%s] clone for indexing completed: %s", projectID, workDir)

    // TODO (Sprint 6+): запустить индексацию в Weaviate
    // s.vectorIndexer.IndexRepository(ctx, projectID, workDir)
}
```

#### 8c. Обновление `projectServiceMCPError` (`internal/mcp/tools_project.go`)

Добавить обработку новых ошибок сервиса в MCP-маппинг:

Сигнатура хелперов из `result.go`:
- `Err(details string, internalErr error) (*mcp.CallToolResult, any, error)` — `details` — user-facing сообщение для LLM, `internalErr` — полная ошибка для серверных логов.
- `ValidationErr(details string) (*mcp.CallToolResult, any, error)` — ошибка валидации входных параметров.

```go
func projectServiceMCPError(err error) (*mcp.CallToolResult, any, error) {
    switch {
    // ... существующие кейсы ...
    case errors.Is(err, service.ErrGitValidationFailed):
        return Err("Git repository validation failed", err)
    case errors.Is(err, service.ErrGitCloneFailed):
        return Err("Git clone failed", err)
    case errors.Is(err, service.ErrDecryptionFailed):
        return Err("Failed to process git credentials", err)
    case errors.Is(err, service.ErrGitURLRequired),
        errors.Is(err, service.ErrGitCredentialRequired),
        errors.Is(err, service.ErrGitCredentialNotSupportedForLocal):
        return ValidationErr(err.Error())
    default:
        return Err("project operation failed", err)
    }
}
```

> **Важно:** Не дублируйте `err.Error()` в `details` — `Err()` уже логирует `internalErr` на сервере. В `details` передаётся чистое, user-facing сообщение для LLM-клиента. Новая ошибка `ErrGitCredentialNotSupportedForLocal` маппится как `ValidationErr` (ошибка ввода клиента).

### 9. Гранулярный маппинг ошибок gitprovider → service

```go
// mapGitProviderErr маппит ошибки gitprovider в ошибки сервиса.
func mapGitProviderErr(err error) error {
    if err == nil {
        return nil
    }
    switch {
    case errors.Is(err, gitprovider.ErrAuthFailed):
        return fmt.Errorf("%w: authentication failed for repository", ErrGitValidationFailed)
    case errors.Is(err, gitprovider.ErrRepoNotFound):
        return fmt.Errorf("%w: repository not found", ErrGitValidationFailed)
    case errors.Is(err, gitprovider.ErrPermissionDenied):
        return fmt.Errorf("%w: insufficient permissions", ErrGitValidationFailed)
    case errors.Is(err, gitprovider.ErrRateLimited):
        return fmt.Errorf("%w: rate limit exceeded, try later", ErrGitValidationFailed)
    case errors.Is(err, gitprovider.ErrCloneFailed):
        return ErrGitCloneFailed
    case errors.Is(err, gitprovider.ErrUnknownProvider):
        return ErrProjectInvalidProvider
    default:
        return fmt.Errorf("%w: %v", ErrGitValidationFailed, err)
    }
}
```

### 11. Обновление метода `Update` (опционально, но рекомендуется)

При изменении `GitURL`, `GitProvider` или `GitCredentialID` — перевалидировать доступ и обновить клон.

```go
// Внутри метода Update, после применения изменений к project, но до вызова projectRepo.Update:

// Запоминаем старый URL для сравнения (до мутации)
oldGitURL := project.GitURL // перед применением req.GitURL

// ... (код применения полей) ...

// Local-провайдер не работает с credentials (аналогично Create)
if project.GitProvider == models.GitProviderLocal && project.GitCredentialsID != nil {
    return nil, ErrGitCredentialNotSupportedForLocal
}

// Если изменился GitURL или GitProvider или GitCredentialID — ревалидация
needsRevalidation := req.GitURL != nil || req.GitProvider != nil || req.GitCredentialID != nil
isRemote := project.GitProvider != models.GitProviderLocal
var provider gitprovider.GitProvider

if needsRevalidation && isRemote && project.GitURL != "" {
    provider, err = s.buildGitProvider(ctx, project.GitProvider, project.GitCredentialsID, userID)
    if err != nil {
        return nil, err
    }
    if err := provider.ValidateAccess(ctx, project.GitURL); err != nil {
        return nil, mapGitProviderErr(err)
    }
}

// ... projectRepo.Update(ctx, project) внутри транзакции ...

// ⚠️ КРИТИЧНО: все файловые операции выполняются СТРОГО ПОСЛЕ
// успешного коммита транзакции БД. Если Update/транзакция вернёт ошибку —
// файловая система остаётся нетронутой (идемпотентность).
gitURLChanged := req.GitURL != nil && project.GitURL != oldGitURL
if gitURLChanged && s.importDir != "" {
    oldWorkDir := filepath.Join(s.importDir, project.ID.String())
    os.RemoveAll(oldWorkDir)

    if provider != nil && project.GitURL != "" {
        go s.cloneForIndexing(provider, project.ID, project.GitURL, project.GitDefaultBranch)
    }
}
```

> **Важно:**
> - `oldGitURL` нужно сохранить **до** применения `req.GitURL` к project, иначе сравнение будет бессмысленным.
> - Файловые операции (`os.RemoveAll`, `cloneForIndexing`) выполняются **строго после** успешного коммита транзакции БД. Если транзакция откатилась — файловая система не затрагивается. Это гарантирует, что состояние БД и файловой системы не рассинхронизируются.

## Архитектурные решения

### 1. Git-операции ВНЕ транзакции БД

**Правило:** `ValidateAccess` и `GetRepoInfo` вызываются **ДО** `transactions.WithTransaction`.

**Обоснование:**
- Git API вызовы могут занимать секунды (сеть) — держать открытую транзакцию БД опасно (блокировки, таймауты YugabyteDB).
- Если ValidateAccess упал — нет смысла начинать транзакцию.
- Порядок: validate → DB transaction (create project + team) → async clone.

### 2. Клонирование — асинхронное (горутина)

**Правило:** `Clone` запускается в фоне **ПОСЛЕ** успешного сохранения в БД.

**Обоснование:**
- Клонирование может занимать минуты (большой репозиторий).
- Пользователь не должен ждать клонирования, чтобы увидеть созданный проект.
- При ошибке клона проект всё равно создан — пользователь может перезапустить индексацию вручную.

**TODO (Sprint 6+):** Заменить горутину на задачу в очереди (Task Queue) с retry и мониторингом. Обновлять статус индексации в проекте (`project.settings.indexing_status`).

### 3. `local` провайдер — без Git-операций при Create

Для `GitProviderLocal`:
- `ValidateAccess` проверяет существование директории и `.git` — это работает только на сервере, а при создании проекта через API директория может не существовать.
- `Clone` не нужен — для local провайдера пути задаются вручную.
- **Решение:** пропускаем Git-операции при `gp == models.GitProviderLocal`.

### 4. `Decryptor` интерфейс vs прямая зависимость от `pkg/crypto`

**Выбор:** Интерфейс `Decryptor` в слое `service`.

**Обоснование:**
- Clean Architecture: `internal/service` не зависит от `pkg/crypto` напрямую.
- Тестируемость: в unit-тестах подставляем мок-декриптор.
- Развязка: можно поменять алгоритм шифрования без изменения сервиса.
- Параллельная разработка: задача 4.6 не блокируется задачей 4.7.

### 5. `GitCredentialID` не обязателен для public-репозиториев

Публичные GitHub-репозитории доступны без токена. Поэтому:
- Если `GitCredentialID == nil` и `gitURL != ""` → создаём провайдер с пустыми `Credentials`.
- `ValidateAccess` пройдёт для публичных репозиториев.
- `Clone` также сработает для публичных.

Но для приватных репозиториев без credentials получим `ErrAuthFailed` от `ValidateAccess` — ожидаемое поведение.

### 6. Файловые операции — строго ПОСЛЕ коммита транзакции БД

**Правило:** `os.RemoveAll`, `cloneForIndexing` и любые другие операции с файловой системой выполняются **только после** успешного коммита транзакции в БД.

**Обоснование:**
- Если транзакция откатилась (ошибка БД, конфликт уникальности) — файловая система остаётся нетронутой, состояние консистентно.
- Обратный порядок (сначала файловые операции, потом БД) привёл бы к "осиротевшим" директориям или потере данных при откате транзакции.
- В `Create`: `go cloneForIndexing(...)` запускается после `transactions.WithTransaction`.
- В `Update`: `os.RemoveAll(oldWorkDir)` и `go cloneForIndexing(...)` запускаются после `projectRepo.Update(ctx, project)` (внутри транзакции) завершился успешно.

### 7. `local` провайдер — валидация: credential не принимается

**Правило:** если `git_provider=local` и `git_credential_id != nil` — возвращаем `ErrGitCredentialNotSupportedForLocal`.

**Обоснование:**
- Local-провайдер работает с файловой системой сервера, credentials ему не нужны.
- Передача credential для local — логическая ошибка клиента, которую лучше отловить рано.
- Это предотвращает зависание credentials, привязанных к проектам, где они бесполезны.

### 8. Автозаполнение `GitDefaultBranch` из `GetRepoInfo`

Если пользователь не указал ветку (или указал "main"), а у репозитория default branch = "master" — заполняем из API.

**Логика:**
- Если пользователь явно указал ветку (не пустую и не "main") — оставляем как есть.
- Если ветка пустая или "main" — пытаемся получить из `GetRepoInfo`.
- Если `GetRepoInfo` вернул ошибку — продолжаем с "main" (не критично).

## Безопасность

### Token Leakage

- **ЗАПРЕЩЕНО** логировать расшифрованные credentials.
- В `cloneForIndexing` логируем только `project.ID` и факт ошибки, **НЕ** URL с токеном.
- `buildGitProvider` возвращает `GitProvider` — токен инкапсулирован внутри.

### Race Condition и защита горутины

- Горутина использует **свежий** `context.Background()` (не `ctx` из запроса — тот будет отменён).
- `cloneForIndexing` принимает **примитивы** (`projectID uuid.UUID`, `gitURL string`, `branch string`) вместо `*models.Project` — нет shared mutable state между горутиной и HTTP-хендлером.
- `defer recover()` внутри горутины предотвращает crash всего сервера при panic в Clone.
- Если пользователь удалит проект до завершения клона — клон продолжится, но результат не критичен (директория будет удалена при следующем cleanup).

### Path Traversal

- `importDir` задаётся через конфигурацию (env variable), не через пользовательский ввод.
- `workDir = filepath.Join(s.importDir, project.ID.String())` — UUID безопасен (нет `/`, `..`).
