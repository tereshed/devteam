## Контекст и цель

В Sprint 10 вся фича `projects` опирается на `AppLocalizations`, Riverpod 3.x (`@riverpod`) и `go_router`. Виджет-тесты должны:

1. Верифицировать **поведение** UI (loading / empty / error / success / debounce / cancel / 404 / redirect / адаптив).
2. Не дублировать литералы переводов — **сравнивать с геттерами `l10n.<key>`**, иначе любая правка ARB ломает тесты до прогона `gen-l10n`.
3. Покрывать **обе локали** для тех экранов и сценариев, в которые идёт smoke-эстафета из задачи 10.8 (см. ниже «Локализация»).
4. Не запускать сетевые вызовы — все `Dio`-вызовы мокируются через `mockito` + `ProviderScope.overrides`.

Цель 10.9 — **закрыть** покрытие фичи проектов виджет-тестами, чтобы регрессии в UI проектов ловились до интеграционных прогонов и до code-review.

## Предусловия

1. **Riverpod 3.x и порядок codegen.** Используется `@riverpod` + сгенерированные `*.g.dart`. Перед ручным `flutter test` без `make frontend-test` обязательно **`make frontend-codegen`** (`dart run build_runner build --delete-conflicting-outputs`, затем `flutter gen-l10n`). Иначе тесты не скомпилируются: `projectListProvider` и `createProjectControllerProvider` определены в `*.g.dart`. Подробности и запрет на обратный порядок — в `docs/rules/frontend.md` (раздел про `gen-l10n`) и в `Makefile` целях `frontend-setup` / `frontend-codegen` / `frontend-test`.

2. **Mocks (`mockito` `^5.6.1`).** Mockito-моки регенерируются `build_runner` из `@GenerateNiceMocks([MockSpec<…>()])`. Файлы `*.mocks.dart` **в git** — правило зафиксировано в **`docs/rules/frontend.md`** (подпункт «Mockito» в разделе про генерацию и git). После нового мока — `make frontend-codegen`.

3. **Делегаты локализации.** Любой виджет под тестом, тянущий `AppLocalizations`, обязан быть обёрнут через `wrapSimple` / `wrapRouter` или явно `MaterialApp(localizationsDelegates: AppLocalizations.localizationsDelegates, supportedLocales: AppLocalizations.supportedLocales, …)`. Иначе `requireAppLocalizations` ([frontend/lib/core/l10n/require.dart](../../frontend/lib/core/l10n/require.dart)) бросит `StateError` с подсказкой `where`.

4. **Изоляция таймеров и retry.** В `wrapRouter` уже стоит `ProviderScope(retry: (_, _) => null)` — без этого `AsyncNotifier` фейлы зацикливаются. Для дебаунса поиска использовать **`kProjectsListSearchDebounce`** из [projects_list_screen.dart](../../frontend/lib/features/projects/presentation/screens/projects_list_screen.dart) (тот же `Duration`, что в проде), например `tester.pump(kProjectsListSearchDebounce)` или суммы долей, а не «магические» 400/401 мс, скопированные с потолка. Не использовать `pumpAndSettle` для ожидания дебаунса.

## Архитектурные решения / конвенции

### 1. Один источник правды для обвязки — `helpers/test_wrappers.dart`

Все новые тесты обязаны идти через `wrapSimple` (без роутера) или `wrapRouter` (с роутером). Не копировать `MaterialApp(...)` руками, иначе через две правки разойдутся `localizationsDelegates` или `supportedLocales`. Если нужен **специальный** `GoRouter` (как в `project_dashboard_screen_test.dart`) — использовать `helpers/project_dashboard_test_router.dart` или вынести аналогичный фабричный конструктор в `helpers/`.

### 2. Моки только через `ProviderScope.overrides`

```dart
ProviderScope(
  retry: (_, _) => null,
  overrides: [
    projectRepositoryProvider.overrideWithValue(ProjectRepository(dio: mockDio)),
    // или — для более точного контроля: override провайдеров, не репозитория
    projectListProvider.overrideWith((ref) => Future.value(makeResponse([...]))),
  ],
  child: ...,
);
```

Не использовать `GetIt`, не подменять глобальные синглтоны, не ходить в реальный `Dio`. Для интеграционных HTTP-тестов есть отдельная цель (`make frontend-test-integration`), но это **не** widget-тесты.

### 3. Стабильные фикстуры из `project_fixtures.dart`

Все ProjectModel-инстансы в тестах — через `makeProject(...)` с дефолтами. Стабильные `createdAt` / `updatedAt` (`DateTime(2026, …)`) — критично для будущего golden-сравнения и логов.

### 4. Никаких ожиданий на литералах перевода

```dart
// ❌ плохо
expect(find.text('Создать проект'), findsOneWidget);

// ✅ хорошо
final ctx = tester.element(find.byType(ProjectsListScreen));
final l10n = AppLocalizations.of(ctx)!;
expect(find.text(l10n.createProject), findsOneWidget);
```

Это и есть тот контракт, который 10.8 закрепляет в ARB: один источник правды для строки — геттер.

**`create_project_screen_test.dart`:** для тапов по кнопке отправки и пункту «локальный» провайдер используйте **`find.text(l10n.create)`** и **`find.text(l10n.gitProviderLocal).last`** (в файле — вспомогательная **`l10nForCreateScreen(tester)`** на контексте `CreateProjectScreen`); не дублируйте английские литералы `Create` / `Local` в `find.text`.

### 5. Латинские литералы в `expect(find.text('…'))` — только whitelist

Строки **пользовательского UI**, которые приходят из ARB (включая SnackBar, overlay ошибки/404, подписи retry, названия вкладок shell), в тестах сравниваются **только** с `l10n.<key>` (или с переменной, равной `l10n.<key>`), без дублирования английского текста в кавычках.

**Whitelist** — разрешены **только** литералы, которые по смыслу **не** являются переводимым UI-продуктом:

- **Данные модели / фикстуры:** дефолты `makeProject` (`Test Project`, `A test project`), явные `name:` / `description:` в тестах (`Alpha`, `Beta`, …), а также имена дашборда из **`project_fixtures.dart`**: `kTestDashboardProjectNameFixtureAlpha`, `kTestDashboardProjectNameAfterLoading`, `kTestDashboardProjectNameAfterRetry`, `kTestDashboardProjectNamePopped`, **`kTestDashboardNavProjectNameA`**, **`kTestDashboardNavProjectNameB`** (одна точка правды для строки в `makeProject` и в `expect`).
- **Служебные маркеры** в тестах роутера: `__TEST_PROJECTS_LIST__`, `__OPEN_PROJECT__`.
- **Ветка «неизвестный статус»** в `project_status_chip_test`: `unknown_xyz`, `Unknown`.
- **Git-провайдер** в `project_card_test`: литералы в духе `github` / `GitHub`, где проверяется отображение кода vs локализованная подпись.

Интерполяция вроде `'hub-$id'` — не новый зафиксированный продуктовый текст, допустима.

Новый латинский литерал в `expect(find.text('…'))` вне whitelist = либо заменить на `l10n` / константу из `project_fixtures.dart`, либо **расширить whitelist в этом файле** одним коммитом с явным обоснованием в PR.

## Анти-паттерны (что НЕ делать)

- **`pumpAndSettle` для дебаунса/таймеров.** В `projects_list_screen` используется `Timer` с **`kProjectsListSearchDebounce`** — `pumpAndSettle` либо повиснет, либо съест семантику. Прокачивать время через `tester.pump(...)` с опорой на **`kProjectsListSearchDebounce`**, не на копию `400` в тесте.
- **Мок репозитория без `retry: (_, _) => null`.** `AsyncNotifier` будет ретраить ошибку до таймаута теста.
- **Тестировать через интерфейс родителя без `GoRouter`.** `ProjectCard` тапает `context.push('/projects/:id')` → без роутера `tester.tap` бросит `Navigator operation requested with a context that does not include a Navigator`.
- **Глобальные моки `Dio` как `Mock` (без `MockSpec`).** Использовать `@GenerateNiceMocks([MockSpec<Dio>()])` — это запретит несуществующие методы и обновится при апгрейде `dio`.
- **Хардкод дат через `DateTime.now()`** — тесты станут флаки. Только фикстуры из `project_fixtures.dart` или `clock` пакет, если нужен сдвиг времени.
- **`expect(find.text("…русский литерал"))`.** См. раздел про `l10n`.
