## Провайдеры: копия `project()` (SSOT)

Паттерн Sprint 10 зафиксирован в **`frontend/lib/features/projects/data/project_providers.dart`**:

- **`projectRepository`** — **`@Riverpod(keepAlive: true)`**, возвращает **`ProjectRepository`**.
- **`project`** — **`@riverpod Future<ProjectModel> project(Ref ref, String id)`** с **`CancelToken`** и **`ref.onDispose(cancelToken.cancel)`** инлайном, вызов **`getProject(..., cancelToken: cancelToken)`**.

**Для команды в 13.1 — точная аналогия:**

```dart
@Riverpod(keepAlive: true)
TeamRepository teamRepository(Ref ref) {
  final dio = ref.watch(dioClientProvider);
  return TeamRepository(dio: dio);
}

@riverpod
Future<TeamModel> team(Ref ref, String projectId) {
  final cancelToken = CancelToken();
  ref.onDispose(cancelToken.cancel);
  return ref.watch(teamRepositoryProvider).getTeam(
        projectId,
        cancelToken: cancelToken,
      );
}
```

- Имя функции провайдера — **`team`** → в кодгене **`teamProvider`** (family по **`projectId`**).
- **`AsyncNotifier` в 13.1 не вводить.** Обновление списка команды и повтор после ошибки — **`ref.invalidate(teamProvider(projectId))`**, как **`project_dashboard_screen.dart`** (**~строка 58**) для **`projectProvider`**, а не отдельный нотифайер.
- **`AsyncNotifier`** и мутабельный стейт появятся в **13.3** (**`PUT`**, переключатели с сохранением). Граница **13.1 ↔ 13.3** зафиксирована здесь.

## Контракт данных

### Источник и идентификаторы

- **`projectId`** — только маршрут **`/projects/:id/…`**. **`TeamRepository.getTeam`**: **`if (projectId.isEmpty) throw ArgumentError('projectId is required');`**.
- **`TeamScreen({required String projectId})`**: **`assert(projectId.isNotEmpty);`** (debug).
- **`Future<TeamModel> getTeam(...)`** только в **`TeamRepository`**. В **`team`**-провайдере: **`CancelToken` + ref.onDispose** — **обязательно**.
- После **`fromJson`**: если **`team.projectId != projectId`** → **`TeamProjectMismatchException`**.

### 401 / истёкший JWT

Бэкенд требует JWT. При **401** / **`UnauthorizedException`** (как в **`ProjectRepository`**): **не** оставлять экран на бесконечном спиннере — либо тот же UX, что у дашборда (**`dataLoadError`** + **`retry`** после обновления сессии, если приложение так обрабатывает токен), либо редирект на логин **по тому же паттерну**, что уже используется для проектов (**единый** путь с **`ProjectRepository`**, без выдумывания третьего).

### Исключения и HTTP

Файл **`frontend/lib/features/team/domain/team_exceptions.dart`**:

- Абстрактный **`TeamRepositoryException`** с **`originalError`**, **`==` / `hashCode`** без учёта **`originalError`** — **по образцу** **`ProjectRepositoryException`** (`project_exceptions.dart`).
- Подклассы: **`TeamCancelledException`**, **`TeamForbiddenException`**, **`TeamNotFoundException`**, **`TeamApiException`** (и др. по кодам API), **`TeamProjectMismatchException`**.
- Тело ответа: **`requireResponseJsonMap`** (`dio_api_error.dart`); **не** дублировать **`_jsonBody`** из **`project_repository`**.
- **`DioException` → домен**: та же стратегия, что **`_handleDioError`** в **`ProjectRepository`**; при **~80%** совпадения кода — общий хелпер в **`frontend/lib/core/api/`**; иначе зеркало с комментарием **`// mirror ProjectRepository._handleDioError`**.

Presentation **не** импортирует **`dio`**. Тексты ошибок — **«Локализация»**.

## UI: обязательные элементы

1. Без второго **`Scaffold` + AppBar`** раздела (**как 12.4**).
2. Опционально заголовок блока: **`TeamModel.name`**, **`type`**.
3. Только **`ListView.builder`**, одна колонка (**13.1**).
4. **Индикатор активности (`is_active`) — доступность:** без **`Switch`** скринридер не получает семантику «из коробки». Обязательно оборачивать **`Chip`/`Icon`/комбинацию** в **`Semantics`:**

   **`Semantics(label: agent.isActive ? l10n.teamAgentActive : l10n.teamAgentInactive, child: …)`**

   Голый **`Icon(Icons.check_circle)`** без **`Semantics`** / **`tooltip`** — **бракуется на ревью**. Ключи **`teamAgentActive`** / **`teamAgentInactive`** добавить в **13.1** в **`app_*.arb`** (пары **`statusActive`** / «Inactive» в репозитории на момент проверки **нет** — не полагаться на частичное совпадение).

   Если для роли используется только цветной **`Chip`** без текста — та же требование к **`Semantics`** (или явный текст роли рядом).

5. Строка до **13.2:** приватный **`_AgentRowTile`** в **`team_screen.dart`**. После **13.2** — удалить в том же PR, что **`AgentCard`**; в документе **13.2**: **`grep -R "_AgentRowTile" frontend/`** пусто.

6. **`is_active`:** **только** read-only — **`Chip`**, **`Icon`**, **`Text`** + **`Semantics`** (п. 4). **Запрещены** **`Switch`**, **`Checkbox`**, **`ToggleButtons`**, в т.ч. с **`onChanged: null`**.

7. Пустые **`name` / `role`:** см. ключи в локализации.

8. **Загрузка и два спиннера (ортогональность):**
   - **`ProjectDashboardScreen`** (**`project_dashboard_screen.dart`**, ~51–60): пока **`projectProvider(id)`** в **`loading`**, над shell может быть **полноэкранный** **`Center(CircularProgressIndicator())`** — это **оверлей шелла**, не **`TeamScreen`**.
   - **`TeamScreen`** отвечает **только** за **`teamProvider`**: спиннер списка — **`Center(CircularProgressIndicator())`** **в зоне body** вкладки, без **`Stack`**, перекрывающего rail/AppBar дашборда.
   - **Не** подписываться на **`projectProvider`** внутри **`TeamScreen`** ради «синхронизации» с оверлеем шелла: оверлей = метаданные проекта, спиннер вкладки = загрузка команды; сценарии **независимы**.
   - **Deep-link** на **`/projects/:id/team`:** сначала может крутиться оверлей шелла (**`projectProvider`**), затем — спиннер тела вкладки (**`teamProvider`**). Два спиннера **последовательно** — **ожидаемо**, **не** баг UX.
   - Переключение rail на «Команда» при уже загруженном проекте: оверлея нет, крутится только спиннер **`TeamScreen`**.

9. **`RefreshIndicator`:** **`onRefresh`** → **`ref.invalidate(teamProvider(projectId))`** — **обязательно** (после п. «Провайдеры» альтернатив нет).

10. Ошибка загрузки команды: **`title`/`message`** из **`l10n.dataLoadError`**, кнопка **`l10n.retry`** — **как** **`project_dashboard_screen.dart`:56** (**`asyncProject.hasError`**). Повтор — **`ref.invalidate(teamProvider(projectId))`** (и при необходимости общий обработчик сессии — как у проекта).
