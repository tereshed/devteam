# 13.6.1 Унификация ключей роли агента (agentRole*)

**Результат PR:** ключи меток роли в ARB и геттеры в Dart — префикс **`agentRole*`**; нейтральные **`description`** в template **`app_ru.arb`**; в репозитории нет прежнего префикса из Sprint **12.5** (имя см. историю git до merge этой задачи).

**Sprint:** 13 — Frontend (локализация, техдолг именования после Sprint 12)

**Основные файлы:**
- `frontend/lib/l10n/app_ru.arb` — template (`l10n.yaml` → `template-arb-file`); переименование ключей и обновление блоков `@…` с **`description`**.
- `frontend/lib/l10n/app_en.arb` — зеркальные ключи (плейсхолдеров у этих строк нет).
- Сгенерированные файлы после **`make frontend-codegen`**: `frontend/lib/l10n/app_localizations.dart` (абстрактный **`AppLocalizations`** + делегаты), `app_localizations_ru.dart` и `app_localizations_en.dart` (конкретные реализации со строковыми телами геттеров). Дифф по **`app_localizations.dart`** и по **`_ru` / `_en`** не обязан быть идентичным по структуре. **В остальном документе** для краткости эта тройка файлов — **`app_localizations*.dart`**. Коммит вместе с **`.arb`** (канон: **`docs/rules/frontend.md`** §2.3).

**Косвенно затрагивается:**
- `frontend/lib/features/projects/presentation/utils/agent_role_display.dart` — единственная функция маппинга **`role` (wire-строка)** → подпись **`AppLocalizations`**; геттеры **`agentRole…`**.
- UI с **`agentRoleLabel(l10n, …)`** (сигнатура не меняется): `task_detail_screen.dart`, `task_card.dart`, `agent_card.dart`, `agent_edit_dialog.dart`.
- **Тесты:** guard **`frontend/test/features/tasks/presentation/utils/agent_role_l10n_guard_test.dart`** — цикл по **`agentRoles`** из **`task_model.dart`** и проверка геттеров **`agentRole…`**. Остальные тесты UI ходят через **`agentRoleLabel`**.

**Связано с:**
- [12.5 Экран: Детали задачи](./12.5-task-detail-screen.md) — введены ключи меток роли (позже переименованы в **`agentRole*`** в этой задаче); гард на **`agentRoles`**.
- [13.6 Локализация / аудит l10n для Sprint 13](./13.6-l10n-sprint13-arb.md) — таблица префиксов после merge: строка **`agentRole*`** (источник **12.5** + **13.6.1**).

**Предпосылка по коду:** завершённые **12.5** (метки роли в **`.arb`** до унификации).

**Координация с 13.6:** и **13.6**, и **13.6.1** правят **`app_ru.arb`** / **`app_en.arb`** и регенерируют **`app_localizations*.dart`**. Параллельные ветки почти наверняка дают **merge-конфликт** по **`app_localizations*.dart`**. **Мерджить 13.6.1** либо **до** старта активной работы над **13.6** по ARB, либо **после** мерджа **13.6**.

**Статус:** ✅

## Зачем отдельная задача (и почему не в 13.6)

1. **Семантика имени:** прежний префикс намекал на контекст «деталь задачи», но те же строки используются для **роли агента** на карточке задачи, в **команде** (`agent_card`, `agent_edit_dialog`) и в **`agent_role_display.dart`**. Префикс **`agentRole*`** отражает доменную сущность (роль агента в pipeline), а не экран.

2. **Объём PR:** девять ключей × два ARB + **`app_localizations*.dart`** + **`agent_role_display.dart`** + guard-тест (переименование файла) + **`docs/tasks/`**.

3. **Разделение ответственности:** **13.6** — дельта строк Sprint 13; **13.6.1** — **переименование** существующего набора ключей.

### Код и тесты

- **`agent_role_display.dart`** — `switch` по `role` → **`l10n.agentRole…`**.
- **`agent_role_l10n_guard_test.dart`** — **`agentRoles`** из **`task_model.dart`**; фоллбэк неизвестной роли — **`l10n.agentRoleUnknown`**.

**Guard-тест** импортирует **`agentRoles` из `task_model.dart`**. Слияние двух деклараций в одну точку — вне скоупа **13.6.1** (условно **13.6.2**).

## Политика PR (атомарное переименование в ARB)

**Канон:** переименовать ключи на месте одной правкой в каждом ARB, без временных алиасов.

## Предусловия / приёмка ARB

1. Паритет ключей сообщений ru/en — **`make frontend-l10n-check`**.
2. Порядок codegen — **`make frontend-codegen`** (сначала **`build_runner`** с **`--delete-conflicting-outputs`**, затем **`flutter gen-l10n`**).
3. **`app_ru.arb`:** для **`@agentRole*`** — **`description`** нейтрально (роль в UI: карточка задачи, детали, команда), **без** привязки «только 12.5» и без суффикса **`(12.5)`** в тексте описания.
4. **`app_en.arb`:** для этих девяти ключей блоки **`@agentRole*`** с **`description`** не используются (метаданные только в template **`app_ru.arb`**).
