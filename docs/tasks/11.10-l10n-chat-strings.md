# 11.10 Локализация / аудит l10n для Sprint 11 — чат (ru, en)

**Sprint:** 11 — Frontend: Чат (Flutter)

**Основные файлы:**
- `frontend/lib/l10n/app_ru.arb` — **template** для `flutter gen-l10n` (см. `frontend/l10n.yaml` → `template-arb-file: app_ru.arb`).
- `frontend/lib/l10n/app_en.arb` — английские строки; множество **ключей сообщений** совпадает с template.

**Скоуп аудита (код фичи):**
- `frontend/lib/features/chat/**` — все пользовательские строки UI и вызовы локализации; источник дельты ключей и поиска захардкоженного текста.

**Косвенно затрагивается:**
- `frontend/lib/l10n/app_localizations.dart`, `app_localizations_ru.dart`, `app_localizations_en.dart` — генерируются **`flutter gen-l10n`**, в репозитории **коммитятся**. После правок **`.arb`** — **`make frontend-codegen`** (или `build_runner` → `gen-l10n` в том же порядке, что в `Makefile`) и коммит триплета. Канон: **`docs/rules/frontend.md`** (i18n).

**Зависит от:**
- **11.4** — функция **`chatErrorTitle(l10n, error)`** (локализованные заголовки ошибок: `chatError*`, общие `errorForbidden`, `errorUnauthorized`, `errorRequestCancelled`, `errorNetwork`, `errorServer`, …) и логика отправки ([11.4-chat-controller.md](./11.4-chat-controller.md)). **`chatErrorDetail`** — **не** набор ключей ARB: это **`String? chatErrorDetail(Object error)`**, опциональный **сырой** хвост ответа API; **не локализуется** — см. § «Предусловия», п. 4.
- **11.5** — `chat_screen.dart`, placeholder без `conversationId` ([11.5-chat-screen.md](./11.5-chat-screen.md)).
- **11.6** — `ChatMessage`, markdown, стрим ([11.6-chat-message-widget.md](./11.6-chat-message-widget.md)).
- **11.7** — `TaskStatusCard`, роли агентов, статусы задач ([11.7-task-status-card-widget.md](./11.7-task-status-card-widget.md)).
- **11.8** — `ChatInput`, подсказки attach/stop ([11.8-chat-input-widget.md](./11.8-chat-input-widget.md)).
- **11.9** — реалтайм; после мерджа пересмотреть тексты, которые **временно** ссылались на «будущий WebSocket» ([11.9-realtime-websocket-subscription.md](./11.9-realtime-websocket-subscription.md)).

**Блокирует / prerequisite для:**
- **11.11** — widget-тесты чата ([11.11-chat-widget-tests.md](./11.11-chat-widget-tests.md)). Дерево под тестом должно включать **`MaterialApp` / `MaterialApp.router`** с **`localizationsDelegates: [AppLocalizations.delegate, …]`** и **`supportedLocales`**; ожидания в тестах согласовать с **`AppLocalizations`** там, где строка уже в ARB (см. **10.8** / **10.9**).

**Исполнение по времени:** задачу закрывают **после стабилизации** UI **11.5–11.9** в `main`: дельта = любые **новые** пользовательские литералы в `frontend/lib/features/chat/` → ключи в template (`ru`) → `en` → **`make frontend-codegen`**. Если мердж не добавил строк — объём = **verify** по чек-листу + правки устаревших формулировок (например, примечание про реалтайм).

## Предусловия перед началом

1. **Template и порядок ключей** — как в **10.8** ([10.8-l10n-projects-strings.md](./10.8-l10n-projects-strings.md), § «Предусловия»): новый ключ **сначала** в **`app_ru.arb`**, затем тот же ключ в **`app_en.arb`**; для `{placeholder}` — зеркальные **`@messageKey.placeholders`** с одинаковыми именами и типами в **обоих** файлах.

2. **Импорт** — `package:frontend/l10n/app_localizations.dart` (в `l10n.yaml` нет `synthetic-package` — см. **10.8**).

3. **`build_runner` и `gen-l10n`** — не запускать `build_runner` **после** `gen-l10n` с `--delete-conflicting-outputs`. Предпочтительно **`make frontend-codegen`** из корня.

4. **Заголовок vs деталь ошибки** — в UI используют **`chatErrorTitle`** (и общие ключи из того же `switch`), а не «ключ `chatErrorDetail`». Функция **`chatErrorDetail`** в **11.4** возвращает **`String?`**: при необходимости — **сырой** укороченный текст с бэкенда. Это **не** переводимый литерал и **не входит** в аудит ARB по смыслу «все строки должны быть в `.arb`». Исключение осознанное: локализуются **заголовки** и **статичные** подсказки UI.

## Контекст и цель

В Sprint 11 чат, ввод, карточки задач и ошибки должны опираться на **`AppLocalizations`**, без захардкоженных пользовательских строк во **`frontend/lib/features/chat/`** (кроме осознанных исключений: отладочные `assert`, `debugPrint`, технические `ValueKey`, контент сообщений из API, **сырой** хвост **`chatErrorDetail`** — см. п. 4 выше).

**Цель задачи 11.10:**
- Подтвердить **паритет** ключей `ru`/`en` и **зеркало плейсхолдеров** для всех строк чата и связанных общих ключей (`retry`, `errorNetwork`, …).
- Убрать или обновить **устаревшие** формулировки в ARB. Для **`chatLinkedTasksRealtimeNote`**: либо удалить ключ вместе с использованием в UI, либо **обновить пользовательский текст** и в **`app_ru.arb`** метаданные **`@chatLinkedTasksRealtimeNote`**: если блока ещё нет — **добавить**, иначе **обновить** (`description` для переводчиков должен отражать поведение **после 11.9**, без отсылок к «задаче 11.9» как к будущему). В **`app_en.arb`** — синхронизировать строку и зеркальные **`@*.placeholders`**, если есть; **`description` в `en`** — по политике **10.8** (только то, что нужно генератору).
- **`make frontend-codegen`**, **`make frontend-l10n-check`**, **`make frontend-analyze`** — без ошибок.

Подробные правила именования, `description` только в template (`ru`), инвентаризация и команды — **переиспользовать** разделы **10.8**; ниже — ориентир по **областям** Sprint 11.

## Инвентаризация ключей (ориентир по чату)

Таблица — справочник; после правок кода **11.5–11.9** прогнать **`rg`** по новым вызовам локализации и по подозрительным литералам в `frontend/lib/features/chat/` (см. команды ниже).

| Область | Примеры ключей / примечание |
|--------|----------------|
| Заголовок ошибки (функция **11.4**) | через **`chatErrorTitle`**: `chatErrorGeneric`, `chatErrorConversationNotFound`, `chatErrorRateLimited`, плюс общие `errorForbidden`, `errorUnauthorized`, `errorRequestCancelled`, `errorNetwork`, `errorServer`. **`chatErrorDetail`** — **не ключ ARB**, сырой текст бэкенда — **исключение из аудита l10n**. |
| Экран чата | `chatScreenAppBarFallbackTitle`, `chatScreenSelectConversationHint`, `chatScreenLoadingOlder`, `chatScreenPendingSending`, `chatScreenPendingRetry`, `chatScreenNotFoundBack`, семантика `chatScreenMessageSemantic*` |
| Ввод (**11.8**) | `chatInputHint`, `chatInputSendTooltip`. Ключи `chatInputStopTooltip`, `chatInputAttachTooltip`, `chatInputAttachDisabledHint` — сохранены в ARB как UI-словарь для будущего использования (11.11+), пока не передаются из `chat_screen`. |
| Сообщение (**11.6**) | `chatMessageCopyCode`, `chatMessageStreamingPlaceholder`, `chatMessageImagePlaceholder`, `chatMessageMarkdownImageAlt` |
| Карточка задачи (**11.7**) | `taskStatusCardFallbackTitle`, `taskStatus*`, `taskCardAgentRole*` |
| Примечание о реалтайме | `chatLinkedTasksRealtimeNote` — **пересмотреть** после **11.9** (текст + `@…description` в template — см. «Цель»). |

**Имя переменной локали в коде:** на момент составления задачи вся фича **`frontend/lib/features/chat/`** уже использует **`final l10n = AppLocalizations.of(context)!;`** (без `loc` / `t` / прямых цепочек `AppLocalizations.of…` в UI). Для приёмки **11.10** достаточно **первой** команды ниже. Если правки **11.5–11.9** введут другой стиль — **вернуть единообразие `l10n`** до закрытия **11.10**. **Вторая** команда — **запасной сторож**: если появятся вызовы **`AppLocalizations.of(…)!.getter`** без переменной `l10n`, объединить вывод **обеих** команд перед сравнением с ARB (иначе дельта по «мёртвым» ключам может соврать).

Извлечение имён геттеров из **`l10n.*`**:

```bash
rg -oN --no-filename -r '$1' 'l10n\.(\w+)' \
  frontend/lib/features/chat --type dart | sort -u
```

**Почему не `sed` после `rg -o`:** без **`--no-filename`** (`-I`) ripgrep дописывает путь к файлу (`path:l10n.key`), из‑за чего **`^l10n\.`** в `sed` не срабатывает и в списке остаются дубликаты одного ключа из разных файлов.

Запасной вариант — любые **`AppLocalizations.of(…)!.getter`** одним проходом (**replace** по capture, без `sed`; корректно при пробелах внутри скобок и при наличии/отсутствии **`!`**):

```bash
rg -oN --no-filename -r '$1' 'AppLocalizations\.of\([^)]*\)!?\.(\w+)' \
  frontend/lib/features/chat --type dart | sort -u
```

**Follow-up (не блокирует 11.10):** в **10.8** для роутера упоминается **`requireAppLocalizations`** из `frontend/lib/core/l10n/require.dart` — явный **`StateError`** с **`where:`**, если в дереве нет делегатов. Перевести экраны чата на этот хелпер — отдельный рефакторинг инициализации, не часть l10n-аудита ARB.

Поиск захардкоженного пользовательского текста (точечно, ручная классификация; одинарные и двойные кавычки):

```bash
rg -n "Text\(\s*['\"]|(hintText|labelText|tooltip|helperText|errorText|semanticsLabel):\s*['\"]" \
  frontend/lib/features/chat --type dart
```

## Архитектурные решения

1. **Template locale** — **`app_ru.arb`**; новые ключи **сначала** в `ru`, затем в `en` (повтор якоря: § «Предусловия», п. 1 и **10.8**). Ключ только в `en` → геттер **не** появится.

2. **Единый источник для тестов (11.11)** — предпочитать **`AppLocalizations`** из дерева с **`AppLocalizations.delegate`** и **`supportedLocales`**, а не голые литералы в ожиданиях там, где строка уже в ARB (**как 10.8 → 10.9**).

3. **Разграничение «локализуем / не локализуем»** — заголовки ошибок и статичный UI → ARB; опциональный хвост от API в **`chatErrorDetail`** → вне ARB (см. п. 4 «Предусловия»).

## Чек-лист приёмки

- [ ] Множества ключей сообщений в **`app_en.arb`** и **`app_ru.arb`** совпадают (**`make frontend-l10n-check`**).
- [ ] Для каждой строки с **`{placeholder}`** в обоих ARB есть **`@messageKey`** с одинаковыми **`placeholders.<name>.type`**.
- [ ] В **`frontend/lib/features/chat/`** нет новых пользовательских литералов вне **`AppLocalizations`** (контент сообщений чата из модели — исключение; **`chatErrorDetail`** — исключение).
- [ ] Строки, завязанные на «до 11.9», приведены в соответствие с текущим поведением (**WebSocket** / live-обновления), либо удалены из UI и ARB; для сохранённых ключей обновлены **`description`** в template (**`app_ru.arb`**), где нужно контексту переводчикам.
- [ ] **`make frontend-codegen`** и **`make frontend-analyze`** проходят.
- [ ] **`make frontend-test`** в корневом **`Makefile`** по-прежнему соблюдает порядок: **`dart run build_runner … --delete-conflicting-outputs`** → **`flutter gen-l10n`** → **`flutter test`** (как **10.8**; нарушение порядка часто ломает PR с правками ARB).
- [ ] Инвентаризация **мёртвых** ключей с префиксом **`chat`** (после рефакторингов **11.5–11.9**): сравнить ключи `chat*` из ARB с множеством геттеров, извлечённых **`rg`** по фиче (обязательно — первая команда; при смешанном стиле вызовов — **объединить** с выводом второй, см. § выше). Ключи, не используемые нигде кроме **`lib/l10n/*`** и сгенерированных **`app_localizations*.dart`**, — удалить **или** явно оформить **follow-up** в описании PR.
- [ ] Согласование с **11.11**: тестовое дерево с **`AppLocalizations.delegate`**; при добавлении ключей — обновить ожидания тестов, где проверяется видимый текст.
