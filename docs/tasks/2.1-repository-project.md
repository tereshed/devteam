## Описание

Создать репозиторий для CRUD-операций с проектами. Следует конвенциям проекта: интерфейс + приватная struct, `*gorm.DB` через конструктор, `context.Context` во всех методах, кастомные ошибки, пагинация для списков. Repository — единственный слой, который работает с БД.

---

## Интерфейс

```go
type ProjectRepository interface {
    Create(ctx context.Context, project *models.Project) error
    GetByID(ctx context.Context, id uuid.UUID) (*models.Project, error)
    List(ctx context.Context, filter ProjectFilter) ([]models.Project, int64, error)
    ListByUserID(ctx context.Context, userID uuid.UUID, filter ProjectFilter) ([]models.Project, int64, error)
    Update(ctx context.Context, project *models.Project) error
    Delete(ctx context.Context, id uuid.UUID) error
}
```

---

## Методы

### Create

Создаёт новый проект. Возвращает `ErrProjectNameExists` при дублировании имени в рамках пользователя (ловим pgErr 23505).

| Параметр | Тип | Описание |
|----------|-----|----------|
| `project` | `*models.Project` | Заполненная модель (UserID обязателен) |

### GetByID

Получает проект по UUID. Preload `GitCredential` (без `EncryptedValue`). Возвращает `ErrProjectNotFound` при отсутствии.

| Параметр | Тип | Описание |
|----------|-----|----------|
| `id` | `uuid.UUID` | ID проекта |

**Preload:** `GitCredential` (для отображения label/provider без секретов).

### List

Список всех проектов с пагинацией и фильтрами. Для admin-уровня.

### ListByUserID

Список проектов конкретного пользователя с пагинацией и фильтрами. Основной метод для обычных пользователей.

### Update

Обновляет проект (`gorm.Save`). `Save` перезаписывает **все** поля структуры, включая нулевые значения.

**Контракт с Service-слоем:** Service **ОБЯЗАН** передавать полностью заполненную модель:
1. `GetByID` — загрузить текущее состояние
2. Изменить нужные поля
3. `Update` — сохранить

Это гарантирует, что неизменённые поля не затрутся пустыми значениями. Service также проверяет, что проект принадлежит пользователю (ABAC).

### Delete

Мягкое удаление не используется — полное удаление. Каскадно удалятся `teams`, `tasks`, `conversations`, `mcp_server_configs` (через ON DELETE CASCADE в БД).

## Фильтры и пагинация

```go
type ProjectFilter struct {
    Status      *models.ProjectStatus // Фильтр по статусу (active/paused/archived)
    GitProvider *models.GitProvider   // Фильтр по git-провайдеру
    Search      *string               // Поиск по name/description (ILIKE)
    Limit       int                   // Размер страницы (default: 20, max: 100)
    Offset      int                   // Смещение
    OrderBy     string                // Поле сортировки (default: "created_at")
    OrderDir    string                // ASC / DESC (default: "DESC")
}
```

**Реализация пагинации:** аналогично `LLMRepository.ListLogs` — сначала `Count`, потом `Find` с `Limit/Offset`.

**Поиск (Search):** `WHERE (name ILIKE '%query%' ESCAPE '\' OR description ILIKE '%query%' ESCAPE '\')`; в репозитории символы `\`, `%`, `_` в строке поиска экранируются, чтобы не работали как шаблоны ILIKE.

### Разделение ответственности по слоям

| Ответственность | Слой | Описание |
|-----------------|------|----------|
| Дефолты пагинации | **Service / Handler** (и **Repository**, defense in depth) | Сервис/handler: если `Limit == 0` → 20, если `> 100` → 100. Репозиторий дублирует нормализацию лимита (`normalizeProjectListLimit`), чтобы `GORM Limit(0)` никогда не снимал лимит и не отдавал всю таблицу, если сервис ошибётся. |
| Валидация `OrderBy` | **Repository** | Whitelist допустимых колонок внутри метода (защита от SQL-инъекций). |
| Валидация `OrderDir` | **Repository** | Только `ASC` / `DESC`. При невалидном значении — fallback на `DESC`. |
| Валидация enum-фильтров | **Service / Handler** | `Status.IsValid()`, `GitProvider.IsValid()` проверяются до вызова репозитория. |

### Защита от SQL-инъекций (OrderBy / OrderDir)

**КРИТИЧНО:** `OrderBy` и `OrderDir` — строки, подставляемые в `db.Order(...)`. Прямая подстановка пользовательского ввода = SQL-инъекция.

**Реализация в репозитории:**

```go
var allowedOrderColumns = map[string]bool{
    "created_at": true,
    "updated_at": true,
    "name":       true,
    "status":     true,
}

func sanitizeOrder(orderBy, orderDir string) string {
    if !allowedOrderColumns[orderBy] {
        orderBy = "created_at"
    }
    if orderDir != "ASC" && orderDir != "asc" {
        orderDir = "DESC"
    }
    return orderBy + " " + orderDir
}
```

Вызов: `db.Order(sanitizeOrder(filter.OrderBy, filter.OrderDir))`.

## Кастомные ошибки

```go
var (
    ErrProjectNotFound   = errors.New("project not found")
    ErrProjectNameExists = errors.New("project with this name already exists")
)
```

Обработка ошибок PostgreSQL:
- `gorm.ErrRecordNotFound` → `ErrProjectNotFound`
- pgErr code `23505` (unique_violation) → `ErrProjectNameExists`

Конвенция: аналогично `user_repository.go` (проверка `gorm.ErrDuplicatedKey`, `pgconn.PgError`, fallback по тексту).

## Preload стратегия

| Метод | Preload | Причина |
|-------|---------|---------|
| `GetByID` | `GitCredential` | UI показывает label/provider привязанного credential |
| `List` / `ListByUserID` | — | Не нужен для списка, уменьшаем JOIN'ы |

**Важно:** `GitCredential.EncryptedValue` помечен как `json:"-"` в модели — не утекает в JSON.

## Конвенции (по существующему коду)

1. **Интерфейс + приватная struct** — как `PromptRepository`, `UserRepository`
2. **Конструктор `NewProjectRepository(db *gorm.DB) ProjectRepository`**
3. **`r.db.WithContext(ctx)`** — во всех запросах
4. **`fmt.Errorf("failed to ...: %w", err)`** — оборачивание ошибок
5. **Кастомные sentinel errors** — `ErrProjectNotFound` (как `ErrUserNotFound`)
6. **Paging: `Count` + `Limit/Offset`** — как `LLMRepository.ListLogs`
