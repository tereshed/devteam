## Порядок выполнения с 10.7 (SSOT — один паттерн навигации)

**Инвариант:** задача **10.7 выполняется до или в одном PR с 10.6**. Не допускается сценарий «сначала `IndexedStack` без URL, потом миграция на `StatefulShellRoute`» — это двойная работа, двойной ребилд тестов и расхождение с SSOT спринта.

**Единственный целевой паттерн:** вложенные маршруты под `/projects/:id` через **`StatefulShellRoute`** (или эквивалент в `go_router`, дающий отдельный виджет на ветку и синхронизацию с URL: `/projects/:id/chat`, `/projects/:id/tasks`, `/projects/:id/team`, `/projects/:id/settings`). Каждая ветка — **отдельный** child route; при переходе Sprint 11/12 тяжёлые экраны (WebSocket, стрим задач) **не** остаются все смонтированными в одном `IndexedStack` — жизненный цикл привязан к ветке shell, что исключает фоновые соединения на неактивных вкладках.

Детали регистрации путей — в спецификации задачи **10.7**; в 10.6 описывается только поведение **shell** и **placeholder-тел** веток до Sprint 11–12.

## Ошибки и 404

**Слои (см. `docs/rules/review.md` §2):** presentation **не** импортирует `dio` и **не** проверяет `DioException` / `statusCode`. Различение «404 vs прочая ошибка» — только по **типизированным исключениям** из слоя данных (репозиторий маппит HTTP 404 → например **`ProjectNotFoundException`**; остальное → существующие типы из `ProjectRepository` / домена). Если в **10.2** такого типа для 404 ещё нет — **расширить 10.2** и реализацию `getProject` **до** приёмки 10.6.

| Случай | Поведение |
|--------|-----------|
| **Сеть / 5xx / прочие ошибки** | Экран ошибки + **Retry** — см. **«Retry (SSOT): `ref.invalidate`»** ниже |
| **`404` / проект удалён** | **Отдельный UI:** сообщение из `projectDashboardNotFoundTitle` + кнопка **`projectDashboardNotFoundBackToList`** → **`context.go('/projects')`**. **Не** показывать только Retry — иначе бесконечный цикл, если проект удалён в другой вкладке. В UI различать по **`ProjectNotFoundException`** (или принятому в проекте имени из 10.2). |

## Архитектурные решения

### Retry (SSOT): `ref.invalidate`

Для кнопки **Retry** при ошибке загрузки (не 404) использовать **`ref.invalidate(projectProvider(projectId))`**. Последующий `watch` перезапустит провайдер и запрос. **`ref.refresh`** для этого сценария **не** использовать — иначе расходится контракт с другими экранами на `AsyncValue`.

### Данные и провайдер

| Аспект | Решение |
|--------|---------|
| Источник данных | `ref.watch(projectProvider(projectId))` |
| Повтор при ошибке (не 404) | **`ref.invalidate(projectProvider(projectId))`** |
| 404 | Не retry на «тот же id» без смены маршрута; увести на список по кнопке |
| Уход со страницы | autoDispose + `CancelToken` + `ref.onDispose` (инвариант 10.3) |

### Разделение UI

| Компонент | Ответственность |
|-----------|-----------------|
| `ProjectDashboardScreen` | Только `projectId` из конструктора; `watch` провайдера; развилка loading / not-found / error / data; встраивание shell |
| Shell | Навигация между ветками `go_router` + `body` активной ветки — **без** прямых вызовов репозитория |
| `ProjectDestinationPlaceholder` | Общий виджет для тел плейсхолдеров (обязателен при ≥ 2 одинаковых заглушках) |

### Производительность и будущие спринты

Shell на **`StatefulShellRoute`** держит в дереве **только активную ветку** (поведение `go_router`), а не все четыре экрана сразу — это сознательное отличие от **`IndexedStack`**, где все children остаются смонтированными. **IndexedStack** в 10.6 **не** использовать как основной паттерн hub.

### Не входит в MVP 10.6

- Редактирование/удаление проекта, WebSocket, реальные списки команды/настроек
- **SEO `title`** для web — опционально в отдельной задаче

| Смена раздела (shell) | При навигации на другую ветку отображается соответствующий child (smoke; один набор ожиданий) |
| **CancelToken при dispose** | Smoke: после `pumpWidget` экрана с провайдером и последующего перехода на другой виджет/снятия дерева — для использованного токена **`isCancelled == true`** (или эквивалентная проверка через mock `Dio` / перехват `cancelToken.cancel()`), чтобы инвариант 10.3 был покрыт сценарием из фичи |
