## Импорты: `lib/` vs `test/`

- **`package:frontend/...`** резолвится в **`lib/`** пакета **`frontend`**. Импортировать **`*.mocks.dart`** из **`test/`** через **`package:frontend/...` невозможно** — таких URI нет.
- **Из файлов под `test/`** разрешены и **нормальны** **относительные** импорты **между тестами и хелперами** (например **`../../../helpers/chat_mocks.mocks.dart`** из `presentation/screens/`, **`../../helpers/...`** из `presentation/widgets/`). Это **не** нарушение правила «не `../` из `lib/`»: правило **`docs/rules/frontend.md`** относится к коду в **`lib/`**, где импорты — только **`package:frontend/...`**.
- **Запрещено:** тянуть **`presentation/controllers/chat_controller_test.mocks.dart`** из экранных/виджетных тестов — **не потому что «относительный импорт плох»**, а из‑за **связности с чужим юнит-файлом** (переименование/удаление сломает widget-слой). **Работа 11.11:** заменить в [chat_screen_test.dart](../../frontend/test/features/chat/presentation/screens/chat_screen_test.dart) строку импорта мока на **`../../../helpers/chat_mocks.mocks.dart`** (после появления генерации).
- **Запрещено:** импортировать **`lib/`** из теста относительным путём — только **`package:frontend/...`**.

### Конфликт с линтером: `always_use_package_imports` и `test/`

В [analysis_options.yaml](../../frontend/analysis_options.yaml) включены **`always_use_package_imports: true`** и **`avoid_relative_lib_imports: true`**. Первое применяется **ко всему дереву**, включая **`test/`**. URI **`package:frontend/...`** мапится только на **`lib/`**, поэтому импорт **`helpers/chat_mocks.mocks.dart`** из **`presentation/screens/`** технически **не может** быть `package:`-импортом — остаётся **относительный** путь. Это создаёт **потенциальное** замечание анализатора на относительные импорты между файлами **`test/`**.

**До старта работ по 11.11** (или в том же PR, но **явным решением в описании**): прогнать **`make frontend-analyze`** на текущей ветке; если на **`test/features/chat/**`** есть **info/warning/error** по **`always_use_package_imports`** — зафиксировать **базовое** действие и **дополнительно** одну из стратегий для линта:

| Уровень | Действие |
|---------|----------|
| **Базовое (обязательно)** | **`helpers/chat_mocks.dart`** + генерация **`chat_mocks.mocks.dart`** — снимает зависимость widget-слоя от **`chat_controller_test.mocks.dart`**. Относительный импорт мока из **`presentation/screens/`** остаётся технической необходимостью. |
| **Дополнительно (а)** | В **`analyzer.exclude`** — шаблон **`**/*.mocks.dart`** (см. [analysis_options.yaml](../../frontend/analysis_options.yaml)): mockito не обязан удовлетворять `prefer_const_constructors` / `prefer_single_quotes` и т.д., аналогично **`*.g.dart`**. |
| **Дополнительно (б)** | Точечно ослабить **`always_use_package_imports`** для **`test/**`** через **`analyzer.errors`** / override (если команда предпочитает анализировать моки, но разрешить относительные импорты в тестах). |

Итог (**базовое + при необходимости (а) или (б)**) — зафиксирован в PR **11.11** (или в отдельном коммите конфигурации до миграции тестов).

## Ошибки UI: ARB-ключи vs функции `chatErrorTitle` / `chatErrorDetail`

В **`app_en.arb` / `app_ru.arb`** **нет** геттеров **`l10n.chatErrorTitle`** / **`l10n.chatErrorDetail`**. Заголовки ошибок чата задаются **ключами** вроде **`chatErrorGeneric`**, **`chatErrorConversationNotFound`**, **`chatErrorRateLimited`**, плюс общие ключи; хелпер **`String chatErrorTitle(AppLocalizations l10n, Object error)`** — **top-level** функция в [chat_controller.dart](../../frontend/lib/features/chat/presentation/controllers/chat_controller.dart) (рядом с **`class ChatController`**), публичный API файла; **`import`** с **`show chatErrorTitle`** допустим. **Внутри** хелпер мапит ошибку на **`l10n.<key>`**.

**Два разных «retry»:** **`l10n.chatScreenPendingRetry`** — подпись **inline**-повтора у **pending**-сообщения в ленте (сценарий Ru-smoke § ниже). **`l10n.retry`** — подпись кнопки повтора на **полноэкранной** ошибке начальной загрузки (**`AsyncError`**, как в **10.6**). В одном тесте не смешивать: искать в дереве тот узел, который сценарий реально строит.

**`chatErrorDetail`** — **функция** **`String? chatErrorDetail(Object error)`**, возвращающая **опциональный сырой** текст ответа API ([11.10](./11.10-l10n-chat-strings.md) § Предусловия п. 4). Это **не** ключ ARB и **не** «переводимый литерал» в смысле **`l10n.chatErrorDetail`**.

**В тестах:**

- Для заголовка на экране ошибки: **`expect(find.text(chatErrorTitle(l10n, err)), …)`** (или с тем же **`Object error`**, что в сценарии), с **`import`** хелпера из **`package:frontend/features/chat/presentation/controllers/chat_controller.dart`** — **не** `find.text(l10n.chatErrorTitle)` и **не** `find.text(l10n.chatErrorDetail)` (**такого геттера нет** — тест упадёт на компиляции).
- Для **`chatErrorDetail`**: проверять либо **`expect(find.text(chatErrorDetail(err)!), …)`** при известном непустом значении в сценарии, либо **`findsNothing`** для ветки **`null`**, либо поле состояния контроллера/модели, если деталь отражается там. Сырой английский хвост (**`HTTP 503: …`**, **`connection reset`**) — **легитимен** и попадает в § **Whitelist**, а не в ARB.
- Кнопка повтора на полноэкранной ошибке: **`l10n.retry`** (ключ **`retry`** в ARB) — через **`find.text(l10n.retry)`**.
- Строки **AppBar** / подсказки из **ARB** — по § **Инвентаризация ARB-ключей** и фактической разметке виджета (**`chatInputHint`** vs **`chatScreenInputHint`** — использовать тот ключ, который реально передаётся в **ChatInput**).

### Никаких ожиданий на «дублированные переводы» для детали ошибки

Заголовок ошибки — через **`chatErrorTitle`** / косвенно ключи из § **Инвентаризация**. **`chatErrorDetail`** не выравнивается с ARB: не ожидать, что тот же текст можно достать **`l10n.<key>`**, и не расширять latin-grep на обязательный перенос сырого хвоста в `.arb`.

**Gate:** пока **11.10** не закрыта и **`make frontend-l10n-check`** красный, нельзя полагаться на новые ключи в тестах; канон ключей — задача **11.10**.

## `MediaQuery` в обвязке теста

**Анти-паттерн:** подменять **`MediaQueryData(...)` целиком** конструктором с одним полем (**`size`**, **`textScaler`**) — теряются **`padding`**, **`viewInsets`**, **`platformBrightness`**, что расходится с продом и ломает **`MediaQuery.textScalerOf`** / **`SafeArea`**.

**Нужно:** внутри дерева с **`MaterialApp`** — **`Builder`** → **`MediaQuery.of(context).copyWith(textScaler: …, size: …)`** (или эквивалент с сохранением остальных полей). Зафиксировать это в **`test_wrappers`** при миграции [chat_input_test.dart](../../frontend/test/features/chat/presentation/widgets/chat_input_test.dart).

## `pumpAndSettle` vs одноразовая анимация

- **Периодические таймеры / незавершённый стрим** — `pumpAndSettle` может зависнуть или скрыть семантику; использовать явные **`tester.pump(Duration)`**.
- **Одноразовая** анимация **`ScrollController.animateTo`** (auto-scroll к низу) — **не** то же самое: допустимо **`await tester.pump(scrollToBottomDuration); await tester.pump(const Duration(milliseconds: 16));`** где **`scrollToBottomDuration`** — константа из **`ChatScreenScroll`** (после выноса из [chat_screen.dart](../../frontend/lib/features/chat/presentation/screens/chat_screen.dart)), либо **`pumpAndSettle` с жёстким `timeout`**, если в дереве **нет** бесконечных тиков. Не использовать «запрет pumpAndSettle» как оправдание не дождаться **animateTo**.

## Whitelist латинских литералов в `expect(find.text('…'))`

Пользовательский UI из **ARB** — только **`l10n.<key>`**. Разрешённые литералы (новый = константа в **`chat_fixtures.dart`** + строка здесь + PR):

- **Тела сообщений из фикстур** — **`kChatFixtureAssistantHelloWorld`**, **`kChatFixtureUserTypedText`**, константы для wall-текста; в **`expect`** — **имя константы**, не голая строка.
- **Заголовок беседы из API** — например **`kChatFixtureConversationTitle`**.
- **Markdown / href / fence** (`chat_message_href_test`, `chat_message_fence_test`) — **`https://example.com`**, **`#### Heading`**, фрагменты кода — только **именованные константы** теста или **`chat_fixtures.dart`**.
- **Код статуса задачи в данных API**, если UI показывает сырой код — **`kChatFixtureTaskStatusPending`** и т.д.; если строка локализована — **`l10n`**.
- **`ValueKey`** / технические идентификаторы — не через **`find.text`**.
- **Сырой хвост ошибки API** при проверке **`chatErrorDetail`** — литералы вида **`HTTP 503: …`**, **`upstream timeout`**, **`connection reset`** и т.п.; **не** переносятся в ARB; см. § **Ошибки UI**.

Интерполяции **`'Line $i…'`** — только внутри фикстур, не в **`expect(find.text('Line 0…'))`** без константы.
