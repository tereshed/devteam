# 7.7 Роут: `GET /api/v1/projects/:id/ws` (WebSocket)

**Sprint:** 7 — WebSocket и Реалтайм (Backend)
**Файл:** `backend/internal/server/server.go`
**Статус:** ⬜

---

## Описание

Зарегистрировать публичный HTTP-эндпоинт `GET /api/v1/projects/:id/ws`, по которому клиенты выполняют **HTTP → WebSocket upgrade** для подписки на стрим событий конкретного проекта (см. **7.2** — handler, **7.3** — формат сообщений, **7.4** — EventBus).

Этот тикет — **только проводка**: создаём `ws.Hub`, `ws.EventBus`, `ws.HubBridge`, `ws.WebSocketHandler` в `cmd/api/main.go`, прокидываем через `Dependencies` в `server.New(...)`, добавляем роут в `setupRoutes`.

---

## Контекст использования

```
Browser/Mobile
   │  GET /api/v1/projects/{uuid}/ws
   │  Authorization: Bearer <JWT>          (или Sec-WebSocket-Protocol: bearer.<JWT>)
   │  Origin: https://app.devteam.io
   ▼
Gin router (server.go) ─► AuthMiddleware (4.x) ─► WebSocketHandler.Connect (7.2)
                                                       │
                                                       └─► ws.Hub.Register (7.1)
```

Согласно `README.md` (Sprint 7) — без 7.7 фронт **физически не может** подключиться, и весь стек 7.1–7.6 неактивен.

## Функциональные требования

### 1. Регистрация роута в `server.setupRoutes`

В существующей группе `projects := api.Group("/projects"); projects.Use(authMW)` добавить **до** `projects.GET("/:id", ...)` (иначе Gin сматчит «ws» как :id, см. комментарий в `server.go` про team-роут):

```go
projects.GET("/:id/ws", deps.WebSocketHandler.Connect)
```

**Порядок маршрутов** в Gin — критичен. Конкретные пути с дополнительным сегментом (`/:id/ws`, `/:id/team`, `/:id/tasks`) **обязаны** регистрироваться раньше «общего» `/:id`.

### 2. Аутентификация (auth-middleware)

- Маршрут **обязан** быть под `projects.Use(authMW)` — JWT/API Key аутентификация выполняется **до** handler'а.
- **API Key (`X-API-Key`) допустимо**: middleware ставит `userID` так же, как для JWT (см. `auth_middleware.go`). WebSocket-клиенты на бэке (например, MCP-боты) могут пользоваться этим путём.
- **⚠️ Браузерные клиенты (Critical):** Стандартный `new WebSocket(url, protocols)` не позволяет устанавливать кастомные HTTP-заголовки. Единственный безопасный способ передать JWT из браузера — `Sec-WebSocket-Protocol: bearer.<JWT>`.
- **Требование к authMW:** Middleware обязан фоллбечиться на чтение токена из `Sec-WebSocket-Protocol` (префикс `bearer.`), если заголовок `Authorization` отсутствует. Если authMW из спринта 4.x этого не умеет — доработать **до** начала интеграции. Иначе браузерные клиенты получат `401` даже с валидным токеном.
- Дополнительные проверки доступа к `projectID` — **в handler'е (7.2)**, не в middleware (там нет `projectID`).

### 3. CORS / Origin

- Текущий CORS-конфиг в `server.New` использует `AllowOrigins: ["*"]` — **этого недостаточно для WS** (CORS не применяется к WebSocket-handshake; защита — `Upgrader.CheckOrigin`).
- Whitelist разрешённых origin'ов **обязан** прокидываться в `ws.HandlerConfig.AllowedOrigins` через конфиг (env `ALLOWED_ORIGINS`).
- Для production — `["*"]` **запрещён**: см. CSWSH в **7.2**.

### 4. Dependencies, DI и lifecycle

В `cmd/api/main.go`:

```go
// корневой ctx приложения для long-living компонентов
rootCtx, rootCancel := context.WithCancel(context.Background())
defer rootCancel()

hub := ws.NewHub()
go hub.Run(rootCtx)

bus := ws.NewInMemoryBus()
defer bus.Close()

bridge := ws.NewHubBridge(bus, hub, secretScrubber, logger)
go bridge.Run(rootCtx)

wsHandler := ws.NewWebSocketHandler(hub, projectService, ws.HandlerConfig{
    AllowedOrigins:         cfg.AllowedOrigins,
    MaxConnsPerUserProject: cfg.WebSocket.MaxConnsPerUserProject, // дефолт 5
})

deps := server.Dependencies{
    // ... существующие
    WebSocketHandler: wsHandler,
}

srv := server.New(db, srvCfg, deps)

// graceful shutdown:
//   1. http server Shutdown(ctx)
//   2. rootCancel() → hub.Run и bridge.Run завершаются
//   3. bus.Close()
```

В `internal/server/server.go` **расширить** `Dependencies`:

```go
type Dependencies struct {
    // ... существующие поля
    WebSocketHandler *ws.WebSocketHandler
}
```

### 5. Graceful shutdown (КРИТИЧНО — Race Condition)

Текущий `Server.Shutdown(ctx)` — пустой. В рамках 7.7 минимально необходимое:

- HTTP-сервер должен останавливаться через `http.Server.Shutdown(ctx)` (не через `Close`), чтобы дать время отправить `close` фреймы.
- **⚠️ Race Condition:** `hub.Run` и `bridge.Run` запускаются в горутинах (`go hub.Run(rootCtx)`). Если `main()` просто вызовёт `rootCancel()` и завершится — процесс может выйти **до того**, как hub успеет разослать `{"type":"close","reason":"server_shutdown"}` и закрыть TCP-соединения. Клиенты получат грязный обрыв (`EOF` / `Connection reset by peer`).
- **Требование:** Использовать `sync.WaitGroup` в `main.go` для синхронного ожидания завершения фоновых горутин:

```go
var wg sync.WaitGroup

wg.Add(1)
go func() {
    defer wg.Done()
    hub.Run(rootCtx)
}()

wg.Add(1)
go func() {
    defer wg.Done()
    bridge.Run(rootCtx)
}()

// при шатдауне:
rootCancel()
wg.Wait() // ждём, пока hub и bridge реально отправят close-фреймы и завершатся
```

- После остановки HTTP — `rootCancel()` → `hub.shutdown()` (см. 7.1) разошлёт всем клиентам `{"type":"close","reason":"server_shutdown"}` и закроет соединения.
- `bus.Close()` останавливает Bus.
- Если `Server.Shutdown` уже изменён в другом тикете — **не дублировать**, лишь убедиться, что `wg.Wait()` вызван **после** `rootCancel()` и **до** завершения `main()`.

### 6. Без MCP-инструмента

Согласно `backend.mdc` §7.2: WebSocket-стрим **не подходит** под MCP tool-контракт (MCP-tools — request/response). MCP **не создаём**.

### 7. Swagger

- Аннотации живут на handler'е (см. **7.2**).
- В этом тикете — выполнить `make swagger`, чтобы swagger.json подхватил новый эндпоинт.
- В `@Router` указывать **без** префикса `/api/v1` (см. `backend.mdc` §6.3): `/projects/{id}/ws [get]`.
- Note: WebSocket-эндпоинты не работают через Swagger UI (тот шлёт обычный HTTP), но документация важна для клиентов и MCP-описаний.

### 8. Конфиг

В `internal/config/config.go` добавить секцию:

```go
type WebSocketConfig struct {
    AllowedOrigins         []string      `env:"WS_ALLOWED_ORIGINS" envSeparator:","`
    MaxConnsPerUserProject int           `env:"WS_MAX_CONNS_PER_USER_PROJECT" envDefault:"5"`
    PingPeriod             time.Duration `env:"WS_PING_PERIOD" envDefault:"54s"`
    PongWait               time.Duration `env:"WS_PONG_WAIT" envDefault:"60s"`
}
```

`AllowedOrigins`: для dev — `http://localhost:8080,http://localhost:5173`; для prod — конкретные домены.

**⚠️ Валидация при старте:** Если `WS_ALLOWED_ORIGINS` пуста или отсутствует в .env, парсер может создать слайс `['']` (один пустой элемент). Убедиться, что логика проверки Origin в хендлере 7.2 корректно обрабатывает пустые списки и **не разрешает доступ всем подряд** при ошибке конфигурации. Рекомендуется явная проверка длины слайса и валидности URL при инициализации (early fail на старте, а не silent fallback на AllowAll).

### 9. ЗАПРЕТЫ

- **Запрещено** делать `projects.GET("/:id", ...)` **раньше** `/:id/ws` (Gin сматчит первым).
- **Запрещено** регистрировать роут вне `projects.Use(authMW)` (доступ без аутентификации).
- **Запрещено** использовать query-параметр `?token=...` в качестве источника JWT (см. 7.2).
- **Запрещено** оставлять `AllowOrigins: ["*"]` в production (см. 7.2 CSWSH).
- **Запрещено** забывать про graceful shutdown — Hub без shutdown оставит коннекты «болтаться».
- **Запрещено** завершать `main()` без `sync.WaitGroup.Wait()` после `rootCancel()` (приведёт к race condition: клиенты получат грязный обрыв вместо `close`-фрейма).
- **Запрещено** использовать `Sec-WebSocket-Protocol` для передачи JWT без доработки authMW (браузерные клиенты получат `401`).

## Дизайн-решения

1. **Размещение под `/projects/:id/`:** соответствует доменной модели — пользователь подписывается на конкретный проект, не на «глобальный» поток. Авторизация уже привязана к проекту.
2. **Регистрация перед `/:id`:** прямое следствие Gin matching ordering — копируем уже работающий паттерн из `team`/`tasks` роутов в `server.go`.
3. **Lifecycle Hub/Bus/Bridge в `main.go`, не в `server.go`:** это long-running компоненты приложения, а не HTTP-инфраструктура. `server` отвечает только за роуты.
4. **`HandlerConfig` через env:** соответствует `backend.mdc` §3 / `internal/config` — никаких глобалов и hardcoded значений.
5. **Без MCP:** реалтайм-стрим — антипаттерн для tool-контракта (см. `backend.mdc` §7.2 «❌ НЕТ» для long-poll/stream).

- [ ] В `server.setupRoutes` зарегистрирован `projects.GET("/:id/ws", deps.WebSocketHandler.Connect)`.
- [ ] Маршрут зарегистрирован **до** `projects.GET("/:id", ...)` (подтверждено в коде комментарием).
- [ ] Маршрут под `projects.Use(authMW)` — анонимный доступ невозможен.
- [ ] `Dependencies` расширен полем `WebSocketHandler *ws.WebSocketHandler`.
- [ ] В `cmd/api/main.go` созданы `hub`, `bus`, `bridge`, `wsHandler` с корректным DI; глобальных переменных нет (`backend.mdc` §2.2).
- [ ] `hub.Run(rootCtx)` и `bridge.Run(rootCtx)` запущены в горутинах; `bus.Close()` оформлен `defer`.
- [ ] **Graceful shutdown:** HTTP-сервер останавливается через `Shutdown(ctx)` **до** `rootCancel`; клиенты получают `close`-фрейм от `hub.shutdown()`.
- [ ] **Race condition предотвращён:** `sync.WaitGroup` используется в `main.go`; `wg.Wait()` вызван **после** `rootCancel()` и **до** завершения процесса; клиенты получают корректный `close`-фрейм, а не `EOF`.
- [ ] **authMW поддерживает Sec-WebSocket-Protocol:** Если `Authorization` заголовок отсутствует, middleware читает токен из `Sec-WebSocket-Protocol` с префиксом `bearer.`; запросы из браузера с токеном в `Sec-WebSocket-Protocol` НЕ падают с `401`.
- [ ] **AllowedOrigins валидация:** При пустом или некорректном `WS_ALLOWED_ORIGINS` сервер падает на старте с early fail; НЕ разрешает доступ всем подряд.
- [ ] `WebSocketConfig` добавлен в `internal/config`, читается из env с разумными дефолтами; `AllowedOrigins` без `*` для production.
- [ ] `WS_ALLOWED_ORIGINS` и `WS_MAX_CONNS_PER_USER_PROJECT` задокументированы в `.env.example` / README.
- [ ] **MCP-инструмент НЕ создан** (стрим не подходит под tool-контракт, см. `backend.mdc` §7.2).
- [ ] `make swagger` выполнен; в `swagger.json` появилась запись для `/projects/{id}/ws`.
- [ ] `make test-unit` и интеграционные тесты `server_test.go` зелёные.
- [ ] Smoke-проверка: `wscat -c 'ws://localhost:8080/api/v1/projects/<uuid>/ws' -H 'Authorization: Bearer <jwt>' -H 'Origin: http://localhost:8080'` возвращает `Connected (press CTRL+C to quit)` и принимает события при изменении статуса задачи.
- [ ] Smoke-проверка: тот же запрос **без** JWT → `401`; **без** доступа к проекту → `403`; с чужим Origin → `403`; UUID не валиден → `400`.
