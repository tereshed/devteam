## Описание

Реализовать `GitHubProvider` — конкретную имплементацию интерфейса `GitProvider` (4.1) для работы с GitHub. Провайдер реализует **все 27 методов** интерфейса, используя:

- **GitHub REST API v3** (через библиотеку `go-github/v67`) — для remote-операций (PR, ветки, метаданные).
- **git CLI** (через `exec.Command`) — для локальных операций (clone, commit, push, diff, file content).

## Структура GitHubProvider

```go
package gitprovider

import (
    "context"
    "io"
    "os/exec"

    "github.com/google/go-github/v67/github"
    "golang.org/x/oauth2"
)

// GitHubProvider реализует GitProvider для GitHub REST API v3.
// Remote-операции (PR, ветки, метаданные) выполняются через go-github.
// Локальные операции (clone, commit, push, diff) выполняются через git CLI (exec.Command).
//
// Credentials (Token) хранятся в экземпляре и используются:
// - Для API-вызовов: через oauth2.TokenSource → http.Client → github.Client.
// - Для git CLI: инжектируются в URL (https://x-access-token:<token>@github.com/...).
type GitHubProvider struct {
    client *github.Client
    creds  Credentials
}

// NewGitHubProvider создаёт экземпляр GitHubProvider.
// Вызывается из Factory (4.5), НЕ напрямую.
//
// ⚠️ oauth2.NewClient использует context.Background() — это безопасно,
// т.к. контекст здесь нужен только для конфигурации HTTP-транспорта (TLS),
// а не для запросов. Каждый метод GitHubProvider прокидывает ctx запроса
// напрямую в go-github вызов (g.client.*.Method(ctx, ...)),
// что гарантирует корректную работу таймаутов и отмен.
func NewGitHubProvider(creds Credentials) *GitHubProvider {
    ts := oauth2.StaticTokenSource(&oauth2.Token{AccessToken: creds.Token})
    httpClient := oauth2.NewClient(context.Background(), ts)
    client := github.NewClient(httpClient)

    return &GitHubProvider{
        client: client,
        creds:  creds,
    }
}
```

### Группа 2: Local CLI (exec.Command) — 9 методов

Все локальные методы работают через хелпер `gitExec`:

```go
// gitExec выполняет git-команду в указанной директории.
// Возвращает stdout и error. При ошибке в error включается stderr.
func gitExec(ctx context.Context, workDir string, args ...string) (string, error) {
    cmd := exec.CommandContext(ctx, "git", args...)
    cmd.Dir = workDir
    
    var stdout, stderr bytes.Buffer
    cmd.Stdout = &stdout
    cmd.Stderr = &stderr
    
    if err := cmd.Run(); err != nil {
        return "", fmt.Errorf("git %s: %s: %w", args[0], stderr.String(), err)
    }
    return stdout.String(), nil
}
```

#### 2.6 Push

```go
func (g *GitHubProvider) Push(ctx context.Context, workDir string, opts PushOptions) error
```

| Шаг | Действие |
|-----|----------|
| 1 | Получить remote URL: `gitExec(ctx, workDir, "remote", "get-url", remote)` |
| 2 | Если URL уже содержит Token (Clone из Go-кода) → использовать как есть |
| 3 | Если URL без Token (Clone из sandbox) → собрать authenticated push URL |
| 4 | `remote` = opts.Remote, если пусто → `"origin"` |
| 5 | args: `["push", pushURL, opts.Branch]` (pushURL вместо remote name — **без** изменения .git/config) |
| 6 | Если `opts.Force` → `args = append(args, "--force")` |
| 7 | `gitExec(ctx, workDir, args...)` |
| 8 | "rejected" в stderr → `ErrConflict` |
| 9 | "Permission denied" / "403" → `ErrPermissionDenied` |

**⚠️ Token injection для Push — два подхода:**

| Сценарий | Подход | Безопасность |
|---|---|---|
| **Ephemeral workDir** (sandbox, импорт) | Token в remote URL при Clone → Push «as is» | ✅ Безопасно: workDir удаляется после использования |
| **Persistent workDir** (LocalGitProvider, dev) | `git push https://x-access-token:{TOKEN}@github.com/... branch` — Token передаётся **в команде**, **НЕ** через `git remote set-url` | ✅ Безопасно: Token не сохраняется в `.git/config` |

**⚠️ КРИТИЧНО:** **ЗАПРЕЩЕНО** использовать `git remote set-url origin https://x-access-token:{TOKEN}@...` в persistent workDir — это оставит Token на диске в `.git/config`. Вместо этого передавать authenticated URL напрямую в `git push <URL> <branch>` (git принимает URL вместо remote name).

### mapGitHubError

```go
// mapGitHubError конвертирует ошибки go-github в sentinel-ошибки gitprovider.
func mapGitHubError(err error) error
```

**Маппинг:**

| GitHub HTTP Status | Sentinel Error |
|---|---|
| 401 Unauthorized | `ErrAuthFailed` |
| 403 Forbidden | Проверить header `X-RateLimit-Remaining`: `0` → `ErrRateLimited`, иначе → `ErrPermissionDenied` |
| 404 Not Found | Зависит от контекста: `ErrRepoNotFound` / `ErrBranchNotFound` / `ErrPRNotFound` / `ErrFileNotFound` |
| 409 Conflict | `ErrConflict` |
| 422 Unprocessable | Проверить message: "already exists" → `ErrBranchAlreadyExists` / `ErrPRAlreadyExists` |
| 429 Too Many Requests | `ErrRateLimited` |

**Реализация:** Использовать `github.ErrorResponse` type assertion:

```go
func mapGitHubError(err error, notFoundErr error) error {
    var ghErr *github.ErrorResponse
    if !errors.As(err, &ghErr) {
        return err
    }
    switch ghErr.Response.StatusCode {
    case 401:
        return ErrAuthFailed
    case 403:
        if ghErr.Response.Header.Get("X-RateLimit-Remaining") == "0" {
            return ErrRateLimited
        }
        return ErrPermissionDenied
    case 404:
        return notFoundErr // Передаётся контекстно: ErrRepoNotFound, ErrPRNotFound и т.д.
    case 409:
        return ErrConflict
    case 422:
        return mapUnprocessableError(ghErr)
    case 429:
        return ErrRateLimited
    default:
        return err
    }
}
```

**⚠️ ВАЖНО:** `notFoundErr` передаётся аргументом, т.к. 404 может означать разные вещи (репо не найден, PR не найден, файл не найден). Каждый вызывающий метод передаёт свой sentinel error.

## Безопасность

### Token injection

| Операция | Как инжектируется Token |
|---|---|
| API (go-github) | Через `oauth2.TokenSource` в HTTP Client (ctx прокидывается в каждый метод) |
| Clone | URL: `https://x-access-token:{TOKEN}@github.com/owner/repo.git` |
| Push (ephemeral) | Token уже в remote URL после Clone |
| Push (persistent) | `git push https://x-access-token:{TOKEN}@github.com/... branch` — **НЕ** `git remote set-url` |

### Защита от утечки Token

1. **Логирование:** Все ошибки `exec.Command` пропускаются через `sanitizeToken` перед возвратом.
2. **Stderr:** Если `git push` пишет URL в stderr (при ошибке) — Token заменяется на `***`.
3. **Время жизни:** GitHubProvider — ephemeral (создаётся на время операции, не кэшируется глобально).

### Rate Limiting

GitHub API: 5000 req/hr для authenticated requests. Провайдер **не** реализует retry/backoff на уровне MVP. При получении 429/403 (rate limit) — возвращает `ErrRateLimited`, вызывающий код решает, как обработать.

**Post-MVP:** Добавить configurable retry с exponential backoff (через `golang.org/x/time/rate`).

## Дизайн-решения

### 1. go-github vs raw HTTP

**Выбор:** `go-github/v67` — типизированная обёртка над GitHub REST API.

**Плюсы:** Типобезопасность, встроенная пагинация, обработка rate limits, поддержка всех endpoint'ов.
**Минусы:** Большая зависимость (~40MB). Для MVP оправдано — ускоряет разработку.

### 2. git CLI vs go-git для локальных операций

**Выбор:** `exec.Command("git", ...)` — вызов системного git.

**Альтернатива:** `go-git` — чистая Go-реализация git.

**Обоснование:**
- `go-git` не поддерживает все git-операции (неполная совместимость).
- `exec.Command` — предсказуем, поведение идентично тому, что делает `entrypoint.sh` в sandbox.
- Система (Docker / CI) всегда имеет `git` в PATH.
- Проще отлаживать (те же команды, что в терминале).

### 3. Контекстный 404

GitHub API возвращает 404 для разных сущностей. `mapGitHubError` принимает `notFoundErr` как параметр, чтобы вызывающий метод мог передать конкретный sentinel error (`ErrRepoNotFound`, `ErrPRNotFound`, `ErrFileNotFound`, `ErrBranchNotFound`).

### 4. readCloserWithWait — предотвращение zombie-процессов

Для `GetLocalDiff` и `GetLocalFileContent` возвращается `io.ReadCloser`, чей `Close()` вызывает `cmd.Wait()`. Это предотвращает zombie-процессы при использовании `exec.Command` с `StdoutPipe`.

### 5. Два файла: github.go + github_helpers.go

Основной файл (`github.go`) содержит структуру и реализацию 27 методов.
Вспомогательный файл (`github_helpers.go`) содержит утилиты: `parseRepoURL`, `mapGitHubError`, `mapPullRequest`, `gitExec`, `sanitizeToken`. Разделение — для читаемости (github.go ~500-700 строк).
