## Описание

На экране деталей задачи пользователь должен иметь явные действия **Pause**, **Cancel**, **Resume** в соответствии с FSM в **`task_service.go`** ([`allowedTransitions`](../../backend/internal/service/task_service.go)) и документом [3.6](./3.6-state-machine.md).

Данные и побочные эффекты мутаций — **только** через **`TaskDetailController`**. **Запрещено:** **`TaskRepository`** или **`Dio`** в **`TaskDetailScreen`**.

**Инвариант:** состояние «идёт HTTP lifecycle-мутация» (**inflight**) — **не** локальный **`setState`** без синхронизации с контроллером; только **`TaskDetailState`** + обновления из **`TaskDetailController`** (§ «Inflight»).

**Вне скоупа 12.8:** массовые действия на **12.4**, изменение FSM на бэкенде, **12.9** (подписка WS), полный пакет **12.10** (если не блокирует минимальный smoke).

### Размещение (**вариант A**, без противоречий)

Адаптивная раскладка по ширине; **в одном breakpoint дубля нет**.

| Условие | Где кнопки |
|---------|------------|
| **`MediaQuery.size.width < 600`** (mobile) | Горизонтальная группа (**`Row`** / **`Wrap`**) в **`SliverToBoxAdapter`** **сразу под** блоком статуса/метаданных (**до** секций описание / результат). |
| **`≥ 600 dp`** (tablet / desktop) | Только **`AppBar.actions`** (компактные **`IconButton`** или **`FilledButton.tonal`**). **Без** второй копии группы в теле экрана. |

**Запрещено:** одновременно показывать те же три действия и в **`AppBar.actions`**, и в теле на **одном** breakpoint.

---

| Тема | Решение |
|------|--------|
| Видимость по **`status`** | Канон — таблица § «Видимость кнопок». Кнопку **скрывать**, если переход недопустим; **не** показывать **disabled** «на всякий случай», если канон — «скрыть». **Два независимых основания для disabled:** (1) FSM — не путать с **скрыть**; (2) **inflight** / **`realtimeMutationBlocked`** — кнопки **уже смонтированы**, но временно **disabled** (§ «Inflight», § «Связь с realtime»). |
| **Pause** | Статусы, из которых **`canTransition(·, paused)`** = true: **`planning`**, **`in_progress`**, **`review`**, **`changes_requested`**, **`testing`** (см. **`allowedTransitions`** в **`task_service.go`**). |
| **Cancel** | Статусы, из которых **`canTransition(·, cancelled)`** = true: **`pending`**, **`planning`**, **`in_progress`**, **`review`**, **`changes_requested`**, **`testing`** — **подтверждено** строками **47–53** **`allowedTransitions`** (**`pending`** включает **`cancelled`**). |
| **Resume** | Один HTTP-контракт **`TaskRepository.resumeTask`** / **`Resume`** на бэкенде обрабатывает **`paused`** и **`failed`**: переход в **`pending`** при **`canTransition`**. UI: показывать **Resume** для **`paused`** и **`failed`** (одна и та же кнопка, подпись из l10n). |
| Терминальные статусы | **`completed`**, **`cancelled`** — панель **Pause/Cancel/Resume** **не** монтировать. |
| **`taskDeleted`** / **`AsyncError`** / нет **`task`** | Как **12.5**: панель мутаций **не** показывать. |

### Inflight (mutex lifecycle-мутаций)

- **Место состояния:** только **`TaskDetailState`**. **Канон поля:** не один **`bool` «идёт мутация»** — он **не** говорит, *какая* кнопка нажата, и **ломает** § «Семантика UI при inflight» (спиннер на **нажатой**). Использовать **`TaskLifecycleMutation?`** (или эквивалент: **`enum`** **`pause` / `cancel` / `resume`** + **`null`** = idle / Freezed union) — поле в **`task_states.dart`**, обновление **только** в **`TaskDetailController.pauseTask` / `resumeTask` / `cancelTask`** (установка до **`await`**, сброс в **`finally`** / после завершения).  
- **Запрещено:** любой флаг inflight **только** в **`State`** **`TaskDetailScreen`** без зеркала в контроллере — при пересборке / навигации возможны race и повторный HTTP.  
- Если на старте PR полей ещё нет — **блокер реализации**: сначала расширить **`TaskDetailState`** + контроллер, затем UI.

### Семантика UI при inflight (**disabled + индикатор**)

| Вопрос | Канон |
|--------|--------|
| Где спиннер | **Один** активный индикатор на **нажатой** кнопке: **`CircularProgressIndicator`** (small) **внутри** этой кнопки **или** рядом в **`Row`** только для неё. **Запрещено** вешать отдельный «глобальный» **`CircularProgressIndicator`** посередине панели без привязки к кнопке (если только дизайн не сводится к одной кнопке на всю ширину панели). |
| Остальные кнопки панели | **`onPressed: null`** (**disabled**), без спиннера на каждой. **Это не противоречит** правилу «FSM: скрывать недопустимое»: здесь кнопки **уже видимы** по FSM, но временно заблокированы **мутацией в полёте**. |
| Внешний вид | **Disabled** — стандартная тема (**`onPressed: null`**). Активная кнопка с мутацией — визуально «в работе» за счёт индикатора на ней. |
| **Back** / **`pop`** | **Не** блокировать **`WillPopScope`** ради inflight (пользователь может выйти). **Факт (код 12.3 на момент описания):** **`_mutationSurfaceGuard()`** только **`AsyncData`**; lifecycle после **`await`** дергает **`_patchState`** без проверки «провайдер ещё жив». **`_wsRefetchInFlight`** к lifecycle не относится. **Дефолтный способ (Riverpod 3.x):** сразу после **`await`** репозитория — **`if (!ref.mounted) return;`** и только затем **`_patchState`** — покрывает «ушли с экрана», без **`CancelToken`** и без epoch. **Fallback:** **`int _lifecycleEpoch`** (или аналог), если нужно отличать «сменился **`taskId`** family / другая мутация стартовала» от простого **pop** — тогда сравнение epoch после **`await`** дополняет **`ref.mounted`**. **`CancelToken`** на lifecycle — опционально, не замена **`ref.mounted`**. |

### `TaskMutationOutcome` (**полный enum** в `task_errors.dart`)

| Значение | Поведение UI при возврате из **`pauseTask`/`resumeTask`/`cancelTask`** |
|----------|----------------------------------------------------------------------|
| **`completed`** | Успех: **`task`** уже обновлён в контроллере; дополнительный snack **не** обязателен. |
| **`validationFailed`** | Для lifecycle-методок в **12.3** не используется; если вернулось — трактовать как программную ошибку интеграции (assert/dev log), не как UX-путь. |
| **`notReady`** | Нет **`AsyncData`**: **не** переводить провайдер в **`AsyncError`**; **не** вызывать повтор самим экраном. |
| **`blockedByRealtime`** | **Не** **`AsyncError`**. Обязательно: уже видимый баннер **`taskDetailRealtimeMutationBlocked`**. Если вызов всё же дошёл до notifier и вернул **`blockedByRealtime`** (гонка): показать **`SnackBar`** с **`taskActionBlockedByRealtimeSnack`** (**один** раз на событие). **Запрещено** оставлять поведение «молча / опционально». |
| Транспорт / API после **`await`**, исключение в **`catch`** | **`AsyncError`** по канону **12.3**; заголовок **`taskDetailErrorTitle`**, деталь **`taskErrorDetail`** — **без** **`error.toString()`** в **`Text`**. |

---

| Тема | Решение |
|------|--------|
| **Cancel — подтверждение** | **Обязательно** диалог **`AlertDialog`** (или **`showModalBottomSheet`** на узкой ширине) с отменой / подтверждением. **Pause/Resume** — без модалки (дефолт). **Клавиатура:** явный **`autofocus`** на кнопке подтверждения и Submit по **Enter** — **не** требуются (**Material `AlertDialog`** по умолчанию не подтверждает по Enter); закрытие по **Esc** — типичное поведение платформы. При продуктовом требовании «Enter = подтвердить» — отдельная мини-задача. |
| Доступность | **`Semantics(label: …)`** или **`Tooltip`** на иконках; подписи из **arb**. |

### Видимость кнопок (сводка по **`allowedTransitions`** + **`Pause`/`Cancel`/`Resume`**)

| `status` | Pause | Cancel | Resume |
|----------|-------|--------|--------|
| `pending` | — | ✅ | — |
| `planning` | ✅ | ✅ | — |
| `in_progress` | ✅ | ✅ | — |
| `review` | ✅ | ✅ | — |
| `changes_requested` | ✅ | ✅ | — |
| `testing` | ✅ | ✅ | — |
| `paused` | — | — | ✅ |
| `failed` | — | — | ✅ |
| `completed`, `cancelled` | — | — | — |

**Обоснование `Resume` для `failed`:** `Resume` в [`task_service.go`](../../backend/internal/service/task_service.go) допускает **`task.Status == Failed`** (проверка перед **`canTransition` к `pending`**). Один и тот же **`TaskRepository.resumeTask`** на клиенте.

## Связь с realtime (**12.5** / **12.9**)

**Двойная линия защиты (канон):**

1. **Превентивно:** UI читает **`ref.watch` → `TaskDetailState.realtimeMutationBlocked`** и ставит **`onPressed: null`** на все lifecycle-кнопки, пока **`true`**, плюс баннер (**12.5**).
2. **Реактивно:** если между **`build`** и **`onPressed`** флаг успел смениться **или** остался race, вызов notifier всё же вернёт **`TaskMutationOutcome.blockedByRealtime`** — показать **`SnackBar`** (см. таблицу § «`TaskMutationOutcome`»). Snack — в основном для этой редкой гонки, не замена баннеру.

- Не дублировать **`error.toString()`** для исхода **`blockedByRealtime`**.

## Сценарии widget-тестов

- **`planning`:** видны Pause и Cancel; Resume отсутствует.
- **`paused`:** виден Resume; Pause/Cancel отсутствуют.
- **`failed`:** виден Resume (как для **`paused`**); Pause/Cancel отсутствуют.
- **`completed`:** панель lifecycle отсутствует.
- **`pending`:** только Cancel.
- **`realtimeMutationBlocked: true`:** кнопки неактивны; при сценарии гонки — появление **`SnackBar`** с **`taskActionBlockedByRealtimeSnack`** (если тест моделирует возврат **`blockedByRealtime`**).
- Диалог отмены: dismiss без **`cancelTask`**; подтверждение — один вызов **`cancelTask`** (stub контроллера).
- **Inflight:** при активной мутации в state из контроллера — остальные действия disabled; индикатор на кнопке, совпадающей с типом активной мутации (**`TaskLifecycleMutation`** / enum из § «Inflight»).
- **Двойной tap Pause:** при замедленном stub **`pauseTask`** (или state с inflight после первого tap) — **`pauseTask`** вызывается **ровно один раз** (второй tap не инициирует второй HTTP / второй вызов notifier).

## Архитектурная связь

```
TaskDetailScreen
  → taskDetailControllerProvider.notifier
    → pauseTask | resumeTask | cancelTask
      → TaskRepository.pauseTask | cancelTask | resumeTask
        → POST …/tasks/:id/{pause|cancel|resume}
```

Путь префикса API — как в **`TaskRepository`** (**12.2**).

## Частые ошибки

1. Локальный **`setState`**-флаг inflight без **`TaskDetailState`**.
2. Путаница «FSM: скрыть» vs «inflight: disabled» — см. строку таблицы про два основания для disabled и § «Inflight».
3. **`Text('$error')`** при **`AsyncError`**.
4. Разная семантика **`blockedByRealtime`** у двух разработчиков (snack не зафиксирован).
5. Таблица видимости без **`failed` → Resume**.
6. Жёсткая ссылка на приватный **`_postTaskAction`** в документации вместо публичных методов репозитория.
