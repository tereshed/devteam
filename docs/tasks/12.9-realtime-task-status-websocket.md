## Описание

Закрыть разрыв между **инфраструктурой WebSocket (11.2)** и **экранами задач (12.4–12.8)**: пока пользователь в контексте проекта и смонтированы контроллеры списка и/или детали, клиент получает **`WsClientEvent`** и обновляет **`TaskListState` / `TaskDetailState`** без ручного **`refresh()`**.

**Запрещено:** дублировать FSM **`WebSocketService`** или разбор сырого текста кадра — только **`Stream<WsClientEvent>`** и **`switch`**, как в **11.9** (**парсинг envelope** — **`parseWsServerEnvelope(String)`** в **`websocket_service.dart`**, не в контроллере).

### Зачем подписка в **обоих** контроллерах

| Контроллер | **`keepAlive`** | Зачем WS |
|------------|-----------------|----------|
| **`TaskListController`** | **`true`** (**12.3**) | Обновление строк Kanban/списка по **`taskStatus`** |
| **`TaskDetailController`** | **autoDispose** | Лента **`taskMessage`**, карточка задачи и подзадач по **`taskStatus`** для **открытой** задачи |

**Кейс deeplink:** пользователь открывает **`/projects/:projectId/tasks/:taskId`** без предварительного захода на список — провайдер списка может быть **не** построен. Подписка **только** в списке **не** покрывает деталь. И наоборот: подписка **только** в детали не обновляет Kanban, когда деталь закрыта, а список открыт.

**Повторный `connect(projectId)`:** допустим (несколько экранов / контроллеров того же проекта); не вызывать **`disconnect()`** из **`TaskListController` / `TaskDetailController`** при **`onDispose`** — только отмена **`StreamSubscription`**, иначе обрушится сокет для чата и других подписчиков (**11.9**, § **Смена projectId**).

## Полная таблица `WsClientEvent` (**TaskList** / **TaskDetail**)

Источник вариантов: **`websocket_events.dart`** (**строки ~240–249**). Поведение согласовано с **`ChatController._onWsClientEvent`** (**`chat_controller.dart`**, ~295–327), с отличиями только там, где у задач есть **`applyWsTaskStatus`** / **`applyWsTaskMessage`**.

| `WsClientEvent` | `TaskListController` | `TaskDetailController` |
|-----------------|------------------------|---------------------------|
| **`server` → `taskStatus`** | **`applyWsTaskStatus`** | **`applyWsTaskStatus`** |
| **`server` → `taskMessage`** | Игнор | **`applyWsTaskMessage`** |
| **`server` → `error`** | Если **`err.projectId != family projectId`** → **silent ignore**. Если тот же проект и **`needsRestRefetch == true`** → **`requestRestRefetch`**. Иначе (**`needsRestRefetch == false`**) → **silent ignore**. | То же |
| **`server` → `agentLog`** | Silent ignore (**12.9**) | Silent ignore (**12.9**) |
| **`server` → `unknown`** | Silent ignore; опционально **`debugPrint`** в debug-сборке | То же |
| **`parseError`** | Нет исключения из listener; сброс только **`realtimeServiceFailure`**, если был transient (**аналог** **`ChatController`** `_clearRealtimeTransientFailure` по смыслу) | То же |
| **`serviceFailure`** | **`applyRealtimeFailure(failure)`** | **`applyRealtimeFailure(failure)`** |
| **`authFailure`** | Терминал: **`realtimeSessionFailure: RealtimeSessionFailure.authenticationLost()`**, **`realtimeMutationBlocked: true`**, **`realtimeServiceFailure: null`** (параллель **`ChatController`** ~300–303, но типы state — задачи) | То же |
| **`subprotocolMismatch`** | Одна ветка с **`parseError`**: сброс transient **`realtimeServiceFailure`**; **без** **`refresh()`**; **без** смены **`realtimeSessionFailure`** | То же |

**Без хардкода пользовательских строк** в контроллере — только типизированные поля state (**12.5** / **l10n** на экране).

## Lifecycle, смена проекта и смена family детали

- **`connect(projectId)`** при первой подписке контроллера для данного **`projectId`** (или следовать тому же порядку, что **11.9** для чата).
- **`ref.onDispose`**: только **`await _wsSubscription?.cancel()`**, **без** **`webSocketService.disconnect()`** (**T10**).
- Смена проекта **A → B** при навигации: фильтр **`event.projectId == family projectId`** обязателен; чужие события — silent ignore.
- **Family **`(projectId, taskId)`** у `TaskDetailController`:** при переходе **`/projects/A/tasks/T1` → `/projects/B/tasks/T2`** старый инстанс family **(A, T1)** **autoDispose** — его подписка отменяется **автоматически**; новый **(B, T2)** создаёт **свою** подписку. **Не** вводить общий shared-binding на несколько family: каждая family **сама** владеет **`StreamSubscription`**; никаких ручных мостов между family.

## Частые ошибки

1. **`disconnect()`** в **`onDispose`** контроллера задач → рвётся сокет для чата и других фич.
2. Подписка только в списке → deeplink на деталь без списка не получает **`task_message`**.
3. Копирование разбора сырых кадров / **`parseWsServerEnvelope`** из **`websocket_service.dart`** в контроллер вместо потребления **`WsClientEvent.server`**.
4. Игнор фильтра **`projectId`** → применение чужих событий при мультипроекте / тестах.
