# 7.1 WebSocket Hub: менеджер подключений

**Sprint:** 7 — WebSocket (Backend)
**Файл:** `backend/internal/ws/hub.go`
**Статус:** ✅ Выполнена

---

## Описание

Создать **`Hub`** — центральный менеджер WebSocket-подключений, организованный по **`project_id`**. Hub управляет жизненным циклом подключений, обеспечивает отправку сообщений клиентам и оповещает подписчиков о событиях внутри проекта.

**Основная задача:** Реализовать in-memory хранилище подключений с возможностью broadcast-рассылки по `project_id`. Hub должен быть безопасным для конкурентного доступа (goroutine-safe).

---

## Контекст использования

WebSocket-подключения в DevTeam используются для real-time уведомлений клиента о:

- Изменении статуса задачи (`task_status`)
- Новых сообщениях агента (`task_message`)
- Логах агента (`agent_log`)
- Ошибках (`error`)

Согласно `README.md` (Sprint 7):

```
WebSocket Hub (7.1) → WebSocket Handler (7.2) → Типы сообщений (7.3)
```

Hub — это **нижний уровень**, не зависящий от HTTP и аутентификации. Handler (7.2) использует Hub для регистрации подключений и отправки сообщений.

---

## Архитектура: Actor Model (единый цикл)

**КРИТИЧНО:** Hub использует **чистый Actor Model** — ВСЕ операции с состоянием выполняются внутри одной горутины `Run()`. Это классический паттерн из Gorilla WebSocket и он исключает race conditions.

| Что | Где происходит |
|-----|---------------|
| Мутация `clients` | Только внутри `Run()` |
| Регистрация клиента | Через канал `register` |
| Удаление клиента | Через канал `unregister` |
| Бродкаст | Через канал `broadcast` |
| Отправка одному клиенту | Через канал `unicast` |

**Мьютекс `mu` НЕ используется** — состояние изменяется только в сериализованном цикле `Run()`. `sync.RWMutex` убран за ненадобностью.

## Интерфейс и типы

### hub.go

```go
package ws

import (
	"context"
)

// ErrEmptyProjectID — projectID не может быть пустым.
var ErrEmptyProjectID = errors.New("ws: project_id is empty")

// ErrClientNotFound — клиент с указанным ID не найден.
var ErrClientNotFound = errors.New("ws: client not found")

// Hub — центральный менеджер WebSocket-подключений, организованный по project_id.
// Использует Actor Model: ВСЕ мутации состояния происходят в единственной горутине Run().
// Это исключает race conditions при работе с clients.
//
// Контракт:
//   - Register: добавляет Client в группу project_id (отправляет в канал register)
//   - Unregister: удаляет Client из всех групп (отправляет в канал unregister)
//   - SendToProject: отправляет сообщение всем клиентам в проекте (неблокирующая запись)
//   - SendToClient: отправляет сообщение конкретному клиенту (неблокирующая запись)
type Hub struct {
	// Каналы для сериализованного взаимодействия с Run()
	register   chan *RegisterMessage // ← Actor Model: данные летят вместе с событием, не мутируем клиента извне
	unregister chan *Client
	broadcast  chan *Message
	unicast    chan *UnicastMessage

	// Состояние — мутируется ТОЛЬКО внутри Run()
	// Индекс: project_id → клиенты проекта (O(1) lookup для broadcast)
	projects map[string]map[*Client]bool
	// Обратный индекс: client → его project_ids (для быстрого unregister)
	clientProjects map[*Client]map[string]bool
	// Поиск клиента по ID: clientID → Client
	clientsByID map[string]*Client
}

// Message — сообщение для broadcast в проект.
type Message struct {
	ProjectID string
	Type      string
	Payload   []byte
}

// UnicastMessage — сообщение для конкретного клиента.
type UnicastMessage struct {
	ClientID string
	Type     string
	Payload  []byte
}

// RegisterMessage — сообщение для регистрации клиента ( Actor Model: данные летят вместе с клиентом).
type RegisterMessage struct {
	Client     *Client
	ProjectIDs []string
}

// NewHub создаёт новый Hub.
func NewHub() *Hub {
	return &Hub{
		register:       make(chan *RegisterMessage),
		unregister:    make(chan *Client),
		broadcast:     make(chan *Message, 256),  // Буфер для предотвращения блокировки
		unicast:       make(chan *UnicastMessage, 256),
		projects:      make(map[string]map[*Client]bool),
		clientProjects: make(map[*Client]map[string]bool),
		clientsByID:   make(map[string]*Client),
	}
}

// Run запускает основной цикл Hub. ВСЕ мутации состояния — здесь.
// Блокируется до закрытия контекста или закрытия каналов.
func (h *Hub) Run(ctx context.Context) {
	for {
		select {
		case <-ctx.Done():
			h.shutdown()
			return
		case reg := <-h.register:
			h.addClient(reg.Client, reg.ProjectIDs)
		case client := <-h.unregister:
			h.removeClient(client)
		case msg := <-h.broadcast:
			h.broadcastToProject(msg)
		case msg := <-h.unicast:
			h.sendToClient(msg)
		}
	}
}

// Register добавляет клиента в Hub. Вызывается из Handler (7.2).
// Hub НЕ валидирует права доступа — это обязанность Handler.
// Handler ОБЯЗАН проверить, что пользователь имеет доступ к projectIDs
// (через сервис/БД), прежде чем вызывать Register.
//
// Actor Model: projectIDs летят вместе с событием регистрации, НЕ мутируем клиента.
func (h *Hub) Register(client *Client, projectIDs []string) {
	h.register <- &RegisterMessage{Client: client, ProjectIDs: projectIDs}
}

// Unregister удаляет клиента из Hub. Вызывается при закрытии соединения.
func (h *Hub) Unregister(client *Client) {
	h.unregister <- client
}

// SendToProject отправляет сообщение всем клиентам проекта.
// НЕБЛОКИРУЮЩАЯ операция: если канал broadcast переполнен (медленные клиенты),
// сообщение дропается для этого проекта (slow client isolation).
func (h *Hub) SendToProject(projectID, msgType string, payload []byte) error {
	if projectID == "" {
		return ErrEmptyProjectID
	}
	select {
	case h.broadcast <- &Message{ProjectID: projectID, Type: msgType, Payload: payload}:
		return nil
	default:
		// Канал переполнен — slow client problem, дропаем сообщение
		return nil
	}
}

// SendToClient отправляет сообщение конкретному клиенту по ClientID.
// НЕБЛОКИРУЮЩАЯ операция.
func (h *Hub) SendToClient(clientID, msgType string, payload []byte) error {
	if clientID == "" {
		return ErrEmptyProjectID // reuse error, clientID == projectID semantics
	}
	select {
	case h.unicast <- &UnicastMessage{ClientID: clientID, Type: msgType, Payload: payload}:
		return nil
	default:
		return nil
	}
}
```

### Внутренние методы Hub (мутируют состояние — вызываются ТОЛЬКО из Run)

```go
// addClient регистрирует клиента во всех его проектах.
// ВЫЗЫВАТЬ ТОЛЬКО ИЗ Run()
func (h *Hub) addClient(client *Client, projectIDs []string) {
	// Регистрируем клиента по ID
	h.clientsByID[client.ID] = client

	// Регистрируем в каждом проекте
	for _, projectID := range projectIDs {
		if h.projects[projectID] == nil {
			h.projects[projectID] = make(map[*Client]bool)
		}
		h.projects[projectID][client] = true
	}

	// Запоминаем подписки для быстрого unregister
	h.clientProjects[client] = make(map[string]bool)
	for _, pid := range projectIDs {
		h.clientProjects[client][pid] = true
	}
}

// removeClient удаляет клиента из Hub и всех его проектов.
// Закрывает канал Send клиента.
// ВЫЗЫВАТЬ ТОЛЬКО ИЗ Run()
// Idempotent: повторный вызов на уже удалённом клиенте — no-op.
func (h *Hub) removeClient(client *Client) {
	// Проверяем, что клиент ещё не удалён ( idempotent )
	if _, ok := h.clientProjects[client]; !ok {
		return
	}

	// Удаляем из projects map
	for projectID := range h.clientProjects[client] {
		delete(h.projects[projectID], client)
		if len(h.projects[projectID]) == 0 {
			delete(h.projects, projectID)
		}
	}
	delete(h.clientProjects, client)

	// Удаляем из clientsByID
	delete(h.clientsByID, client.ID)

	// Закрываем канал отправки клиента (только отправитель закрывает канал!)
	close(client.Send)
}

// broadcastToProject отправляет сообщение всем клиентам проекта.
// НЕБЛОКИРУЮЩАЯ запись в каждый client.Send.
// ВЫЗЫВАТЬ ТОЛЬКО ИЗ Run()
func (h *Hub) broadcastToProject(msg *Message) {
	clients, ok := h.projects[msg.ProjectID]
	if !ok || len(clients) == 0 {
		return
	}

	for client := range clients {
		select {
		case client.Send <- msg.Payload:
		default:
			// Клиент не успевает читать — slowloris risk.
			// Отключаем клиента: он всё равно не получит сообщение вовремя.
			h.removeClient(client)
		}
	}
}

// sendToClient отправляет сообщение конкретному клиенту.
// НЕБЛОКИРУЮЩАЯ запись.
// ВЫЗЫВАТЬ ТОЛЬКО ИЗ Run()
func (h *Hub) sendToClient(msg *UnicastMessage) {
	client, ok := h.clientsByID[msg.ClientID]
	if !ok {
		return
	}
	select {
	case client.Send <- msg.Payload:
	default:
		// Клиент завис — отключаем
		h.removeClient(client)
	}
}

// shutdown корректно завершает все подключения при остановке Hub.
func (h *Hub) shutdown() {
	for client := range h.clientProjects {
		select {
		case client.Send <- []byte(`{"type":"close","reason":"server_shutdown"}`):
		default:
		}
		h.removeClient(client)
	}
}
```

## Структура Client

```go
package ws

import (
	"github.com/gorilla/websocket"
)

// Client — WebSocket-клиент, зарегистрированный в Hub.
// Не мутируется после создания — подписки передаются через RegisterMessage в канале.
type Client struct {
	ID   string
	Conn *websocket.Conn
	Send chan []byte // Буферизованный канал для отправки (Hub пишет сюда)
	Hub  *Hub
}

// NewClient создаёт клиента с буферизованным каналом отправки.
func NewClient(id string, conn *websocket.Conn, hub *Hub) *Client {
	return &Client{
		ID:   id,
		Conn: conn,
		Send: make(chan []byte, 256), // Буфер для медленных клиентов
		Hub:  hub,
	}
}

// WritePump читает из Send и пишет в Conn. Запускается в горутине.
// При ошибке записи (включая closed channel) вызывает Unregister.
func (c *Client) WritePump() {
	defer func() {
		c.Conn.Close()
		c.Hub.Unregister(c) // Убираем из Hub
	}()

	for {
		msg, ok := <-c.Send // Блокируется здесь, пока не придёт сообщение или канал закроется
		if !ok {
			return // Канал закрыт — завершаем
		}
		if err := c.Conn.WriteMessage(websocket.TextMessage, msg); err != nil {
			return // Ошибка записи — соединение разорвано
		}
	}
}

// ReadPump читает из Conn в бесконечном цикле. В MVP можно оставить минимальную обработку.
// При разрыве соединения вызывает Unregister.
func (c *Client) ReadPump() error {
	defer func() {
		c.Hub.Unregister(c)
		c.Conn.Close()
	}()
	// MVP: просто читаем и игнорируем входящие сообщения
	for {
		if _, _, err := c.Conn.ReadMessage(); err != nil {
			return err // Разрыв соединения — вызываем defer и выходим
		}
		// Игнорируем payload входящих сообщений (MVP)
	}
}
```

## Структура данных: O(1) Broadcast

**КРИТИЧНО (Performance):**

| Структура | Назначение | Сложность поиска |
|-----------|------------|-----------------|
| `projects map[string]map[*Client]bool` | project_id → клиенты | **O(1)** для broadcast |
| `clientProjects map[*Client]map[string]bool` | client → его project_ids | **O(k)** для unregister (k = число проектов) |
| `clientsByID map[string]*Client` | clientID → Client | **O(1)** для unicast |

**Почему НЕ `clients map[*Client]map[string]bool`:** Итерация по всем клиентам для бродкаста в одном проекте — это O(N), где N = общее число подключений. При 10 000 онлайн и 2 клиентах в проекте — 10 000 проверок. **Инвертированная мапа** (`projects`) даёт O(k) где k = число клиентов в проекте.

## Дизайн-решения

1. **Actor Model (единый цикл Run):** Все мутации состояния в одной горутине — это классический и проверенный паттерн Gorilla WebSocket. Исключает race conditions без мьютексов. **Мьютексы НЕ используются.**

2. **Буферизованные каналы:** `broadcast` и `unicast` имеют буфер 256. Запись в `Send` клиента — тоже буфер 256. Это предотвращает блокировку Hub при slow clients (slowloris protection).

3. **Неблокирующая запись (slow client isolation):** `select { case h.broadcast <- msg: ... default: ... }` — если клиент не успевает читать, Hub не блокируется, а дропает сообщение. При повторном срабатывании — клиент отключается. **Это корректное поведение для real-time системы.**

4. **Канал закрывает только отправитель:** `client.Send` закрывается ВНУТРИ Hub (в `removeClient`), а не в `Client`. Это предотвращает panic: send on closed channel.

5. **Graceful Shutdown:** При отмене контекста Hub отправляет всем клиентам `close`-сообщение, затем отключает их. Это позволяет клиенту корректно закрыть соединение.

6. **Без HTTP/Gin:** Hub — чистый in-memory менеджер. Не знает о HTTP upgrade, JWT или аутентификации.

7. **IDOR Protection — ОБЯЗАННОСТЬ Handler (7.2):** Hub НЕ валидирует права доступа. **Handler (7.2) ОБЯЗАН** проверить принадлежность пользователя к `projectIDs` (через сервис/БД) перед вызовом `Register`. Иначе злоумышленник подпишется на чужой проект.

## Обработка ошибок

| Ошибка | Сценарий | Поведение |
|--------|----------|-----------|
| `ErrEmptyProjectID` | Пустой `projectID` в `SendToProject` | Early return, ошибка возвращается |
| Повторный `removeClient` | Клиент уже удалён (e.g. двойной Unregister) | Idempotent no-op (без паники) |
| Клиент не найден (`clientsByID`) | `SendToClient` по несуществующему ID | Молча silently ignore (ID может устареть) |
| Slow client (переполнен `Send`) | Клиент не читает | Отключение клиента через `removeClient` |
| TCP connection broken | `WriteMessage` error | `WritePump` вызывает `Unregister` |

**Важно:** `SendToProject` и `SendToClient` возвращают `error` только для пустого ID. Остальные "ошибки" (slow client, not found) обрабатываются внутри без возврата — это корректное поведение для real-time messaging.
