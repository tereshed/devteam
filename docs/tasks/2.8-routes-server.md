**Sprint:** 2 — CRUD API для Project и Team
**Файлы:**
- `backend/internal/server/server.go` — регистрация роутов
- `backend/cmd/api/main.go` — DI-сборка
**Статус:** ✅ Выполнена

---

## Описание

Финальная задача-интегратор Sprint 2. Убедиться, что все новые handler'ы (ProjectHandler, TeamHandler) корректно зарегистрированы в `server.go`, DI-граф в `main.go` собран правильно, и вся цепочка `Repository → Service → Handler → Route` работает end-to-end. Задача также включает ревизию текущего кода: удаление `nil`-guard'ов (handlers больше не опциональны), правильный порядок роутов и проверку, что Swagger UI отображает новые эндпоинты.

---

## Текущее состояние

В `server.go` уже **частично** зарегистрированы роуты для проектов и команд (добавлены при создании стабов). Однако:

1. Используются `nil`-guard'ы (`if deps.ProjectHandler != nil`) — после Sprint 2 handlers гарантированно существуют, guard'ы нужно убрать
2. Порядок роутов `/projects/:id/team` и `/projects/:id` может вызвать конфликт в Gin — вложенные роуты `/team` должны быть зарегистрированы ДО параметрических `:id`
3. DI в `main.go` может использовать заглушки — нужно убедиться, что все реальные зависимости подключены

## Целевой вид роутов (server.go)

```go
func (s *Server) setupRoutes(deps Dependencies) {
    authMW := middleware.AuthMiddleware(deps.JWTManager, deps.ApiKeyService)

    // Health & Swagger
    s.router.GET("/health", s.healthCheck)
    s.router.GET("/swagger/*any", ginSwagger.WrapHandler(swaggerFiles.Handler))

    api := s.router.Group("/api/v1")
    {
        // --- Auth (публичные) ---
        auth := api.Group("/auth")
        {
            auth.POST("/register", deps.AuthHandler.Register)
            auth.POST("/login", deps.AuthHandler.Login)
            auth.POST("/refresh", deps.AuthHandler.Refresh)

            authProtected := auth.Group("")
            authProtected.Use(authMW)
            {
                authProtected.GET("/me", deps.AuthHandler.Me)
                authProtected.POST("/logout", deps.AuthHandler.Logout)
                authProtected.POST("/api-keys", deps.ApiKeyHandler.Create)
                authProtected.GET("/api-keys", deps.ApiKeyHandler.List)
                authProtected.GET("/api-keys/mcp-config", deps.ApiKeyHandler.GetMCPConfig)
                authProtected.POST("/api-keys/:id/revoke", deps.ApiKeyHandler.Revoke)
                authProtected.DELETE("/api-keys/:id", deps.ApiKeyHandler.Delete)
            }
        }

        // --- Projects (авторизованный пользователь) ---
        projects := api.Group("/projects")
        projects.Use(authMW)
        {
            projects.POST("", deps.ProjectHandler.Create)
            projects.GET("", deps.ProjectHandler.List)

            // Вложенный ресурс: Team (ДО /:id чтобы избежать конфликта)
            projects.GET("/:id/team", deps.TeamHandler.GetByProjectID)
            projects.PUT("/:id/team", deps.TeamHandler.Update)

            projects.GET("/:id", deps.ProjectHandler.GetByID)
            projects.PUT("/:id", deps.ProjectHandler.Update)
            projects.DELETE("/:id", deps.ProjectHandler.Delete)
        }

        // --- LLM, Prompts, Workflows, Webhooks (admin only) ---
        // ... (без изменений)
    }
}
```

**Ключевые изменения:**
1. Убраны `if deps.ProjectHandler != nil` / `if deps.TeamHandler != nil`
2. Порядок: `/:id/team` зарегистрирован **до** `/:id`
3. Единый блок `projects` без вложенных условий

## Целевой вид DI (main.go)

### Repositories

```go
projectRepo := repository.NewProjectRepository(db)
teamRepo := repository.NewTeamRepository(db)
gitCredRepo := repository.NewGitCredentialRepository(db)
txManager := repository.NewTransactionManager(db)
```

### Services

```go
projectService := service.NewProjectService(
    projectRepo, teamRepo, gitCredRepo, txManager, cfg.Encryption.Key,
)
teamService := service.NewTeamService(teamRepo)
```

**Важно:** Параметр `cfg.Encryption.Key` (AES-256 ключ) — вместо текущего `nil`. Если конфиг ещё не поддерживает это поле, добавить в `config.go`:

```go
type EncryptionConfig struct {
    Key string `env:"ENCRYPTION_KEY" envDefault:""`
}
```

### Handlers

```go
projectHandler := handler.NewProjectHandler(projectService)
teamHandler := handler.NewTeamHandler(teamService, projectService)
```

### Dependencies struct

```go
type Dependencies struct {
    AuthHandler     *handler.AuthHandler
    ApiKeyHandler   *handler.ApiKeyHandler
    LLMHandler      *handler.LLMHandler
    PromptHandler   *handler.PromptHandler
    ProjectHandler  *handler.ProjectHandler   // обязателен
    TeamHandler     *handler.TeamHandler       // обязателен
    WorkflowHandler *handler.WorkflowHandler
    WebhookHandler  *handler.WebhookHandler
    JWTManager      *jwt.Manager
    ApiKeyService   service.ApiKeyService
}
```

Поля `ProjectHandler` и `TeamHandler` — **не указатели** или без `nil`-проверки (handlers всегда создаются).

### 3. Smoke-test (curl)

```bash
# Без токена → 401
curl -s http://localhost:8080/api/v1/projects | jq .

# С токеном → 200 (пустой список)
TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","password":"admin123"}' | jq -r '.access_token')

curl -s http://localhost:8080/api/v1/projects \
  -H "Authorization: Bearer $TOKEN" | jq .

# Создание проекта → 201
curl -s -X POST http://localhost:8080/api/v1/projects \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Test Project","description":"Test"}' | jq .

# Получение команды → 200
PROJECT_ID=$(curl -s http://localhost:8080/api/v1/projects \
  -H "Authorization: Bearer $TOKEN" | jq -r '.data[0].id')

curl -s http://localhost:8080/api/v1/projects/$PROJECT_ID/team \
  -H "Authorization: Bearer $TOKEN" | jq .
```
