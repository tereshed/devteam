## Описание

Создать три Freezed-модели для десериализации ответов backend API. Модели **1:1 соответствуют** типам DTO из `backend/internal/handler/dto/`:
- `ProjectResponse` (project_dto.go:72–87) → `ProjectModel`
- `TeamResponse` (team_dto.go:10–17) → `TeamModel`
- `AgentResponse` (team_dto.go:21–29) → `AgentModel`

**Критичные требования:**
- ✅ Синтаксис Freezed v3: `abstract class X with _$X` (не `implements`)
- ✅ Все JSON-ключи — `@JsonKey(name: 'snake_case')` (backend отдаёт snake_case)
- ✅ Даты — `DateTime` (не `String`), json_serializable парсит автоматически
- ✅ Nullable поля обозначены `?` (например, `String?` для опциональных)
- ✅ `const factory` конструктор + `fromJson`
- ✅ **НУЛЕВЫЕ выдуманные поля** — только то, что отдаёт backend
- ✅ Зависимости: `freezed: ^3.2.3`, `freezed_annotation: ^3.1.0`, `json_serializable: ^6.7.1`

## ProjectModel

Модель проекта. **1:1 соответствует** `ProjectResponse` из backend (project_dto.go:72–87).

```dart
import 'package:freezed_annotation/freezed_annotation.dart';

part 'project_model.freezed.dart';
part 'project_model.g.dart';

@freezed
abstract class ProjectModel with _$ProjectModel {
  const factory ProjectModel({
    /// UUID проекта
    required String id,
    
    /// Название проекта
    required String name,
    
    /// Описание проекта
    required String description,
    
    /// Тип провайдера: 'github', 'gitlab', 'bitbucket', 'local'
    @JsonKey(name: 'git_provider')
    required String gitProvider,
    
    /// HTTPS URL Git-репозитория
    @JsonKey(name: 'git_url')
    required String gitUrl,
    
    /// Ветка по умолчанию (например, 'main' или 'master')
    @JsonKey(name: 'git_default_branch')
    required String gitDefaultBranch,
    
    /// Git-кредентиал (краткие данные без секретов, может быть null)
    @JsonKey(name: 'git_credential')
    GitCredentialModel? gitCredential,
    
    /// Имя вектор-коллекции (для Weaviate)
    @JsonKey(name: 'vector_collection')
    required String vectorCollection,
    
    /// Технологический стек (JSON объект: {"backend": "Go", "frontend": "Flutter"})
    @JsonKey(name: 'tech_stack')
    required Map<String, dynamic> techStack,
    
    /// Статус проекта: 'active', 'paused', 'archived', 'indexing', 'indexing_failed', 'ready'
    required String status,
    
    /// Дополнительные настройки (JSON объект)
    required Map<String, dynamic> settings,
    
    /// Дата создания
    @JsonKey(name: 'created_at')
    required DateTime createdAt,
    
    /// Дата последнего обновления
    @JsonKey(name: 'updated_at')
    required DateTime updatedAt,
  }) = _ProjectModel;

  const ProjectModel._();

  factory ProjectModel.fromJson(Map<String, dynamic> json) =>
      _$ProjectModelFromJson(json);
}
```

### Поля
| Поле | JSON-ключ | Тип | Nullable | Описание |
|------|----------|-----|----------|----------|
| `id` | `id` | String | NO | UUID проекта |
| `name` | `name` | String | NO | Название |
| `description` | `description` | String | NO | Описание |
| `gitProvider` | `git_provider` | String | NO | 'github' \| 'gitlab' \| 'bitbucket' \| 'local' |
| `gitUrl` | `git_url` | String | NO | HTTPS URL репо |
| `gitDefaultBranch` | `git_default_branch` | String | NO | Ветка по умолчанию |
| `gitCredential` | `git_credential` | GitCredentialModel? | YES | Данные git-ключа (опционально) |
| `vectorCollection` | `vector_collection` | String | NO | Имя коллекции Weaviate |
| `techStack` | `tech_stack` | Map<String, dynamic> | NO | JSON объект стека |
| `status` | `status` | String | NO | Статус проекта |
| `settings` | `settings` | Map<String, dynamic> | NO | JSON объект настроек |
| `createdAt` | `created_at` | DateTime | NO | Дата создания |
| `updatedAt` | `updated_at` | DateTime | NO | Дата обновления |

## AgentModel

Модель агента в составе команды. **1:1 соответствует** `AgentResponse` из backend (team_dto.go:21–29).

⚠️ **Важно:** AgentResponse — это краткое представление агента (используется в TeamResponse). Полные данные агента загружаются отдельно через backend/internal/models.Agent. Здесь моделируем то, что приходит в составе TeamResponse. Для редактирования агента в будущих спринтах потребуется отдельная `AgentDetailModel` с полным набором полей.

```dart
import 'package:freezed_annotation/freezed_annotation.dart';

part 'agent_model.freezed.dart';
part 'agent_model.g.dart';

@freezed
abstract class AgentModel with _$AgentModel {
  const factory AgentModel({
    /// UUID агента
    required String id,
    
    /// Название агента
    required String name,
    
    /// Роль агента в pipeline: 'planner', 'developer', 'reviewer', 'tester', 'orchestrator', 'worker', 'supervisor', 'devops'
    required String role,
    
    /// Имя модели LLM (например: 'claude-opus-4-7')
    /// Nullable: если не установлена, используется по умолчанию
    String? model,
    
    /// Имя промпта (относится к отдельной сущности Prompt в backend)
    /// Nullable: может быть не установлен
    @JsonKey(name: 'prompt_name')
    String? promptName,
    
    /// Бэкенд для выполнения (claude-code | aider | custom)
    /// Nullable: не все агенты выполняют код в sandbox
    @JsonKey(name: 'code_backend')
    String? codeBackend,
    
    /// Активен ли агент
    @JsonKey(name: 'is_active')
    required bool isActive,
  }) = _AgentModel;

  const AgentModel._();

  factory AgentModel.fromJson(Map<String, dynamic> json) =>
      _$AgentModelFromJson(json);
}
```

### Поля
| Поле | JSON-ключ | Тип | Nullable | Описание |
|------|----------|-----|----------|----------|
| `id` | `id` | String | NO | UUID агента |
| `name` | `name` | String | NO | Название агента |
| `role` | `role` | String | NO | Роль в pipeline |
| `model` | `model` | String? | YES | Модель LLM (null = default) |
| `promptName` | `prompt_name` | String? | YES | Имя промпта |
| `codeBackend` | `code_backend` | String? | YES | Бэкенд sandbox (claude-code \| aider \| custom) |
| `isActive` | `is_active` | bool | NO | Активен ли |

## Тесты (unit)

### Тест ProjectModel

Файл `frontend/test/features/projects/domain/models/project_model_test.dart`:

```dart
import 'package:flutter_test/flutter_test.dart';
import 'package:frontend/features/projects/domain/models.dart';

void main() {
  group('ProjectModel', () {
    final validJson = {
      'id': 'project-123',
      'name': 'Test Project',
      'description': 'Test description',
      'git_provider': 'github',
      'git_url': 'https://github.com/test/repo',
      'git_default_branch': 'main',
      'git_credential': null,
      'vector_collection': 'DevTeam_Project_123',
      'tech_stack': {'backend': 'Go', 'frontend': 'Flutter'},
      'status': 'active',
      'settings': {},
      'created_at': '2026-04-27T10:00:00Z',
      'updated_at': '2026-04-27T10:30:00Z',
    };

    test('fromJson десериализует поля с правильными JSON-ключами', () {
      final project = ProjectModel.fromJson(validJson);
      
      expect(project.id, 'project-123');
      expect(project.name, 'Test Project');
      expect(project.gitProvider, 'github');
      expect(project.gitUrl, 'https://github.com/test/repo');
      expect(project.techStack, {'backend': 'Go', 'frontend': 'Flutter'});
      expect(project.status, 'active');
    });

    test('fromJson парсит DateTime из RFC3339-строк', () {
      final project = ProjectModel.fromJson(validJson);
      
      expect(project.createdAt, isA<DateTime>());
      expect(project.updatedAt, isA<DateTime>());
      expect(project.createdAt.year, 2026);
      expect(project.createdAt.month, 4);
    });

    test('toJson сериализует поля в snake_case JSON-ключи', () {
      final now = DateTime.now();
      final project = ProjectModel(
        id: 'p1',
        name: 'Test',
        description: 'Desc',
        gitProvider: 'github',
        gitUrl: 'https://github.com/test/repo',
        gitDefaultBranch: 'main',
        gitCredential: null,
        vectorCollection: 'vec-123',
        techStack: {},
        status: 'active',
        settings: {},
        createdAt: now,
        updatedAt: now,
      );

      final json = project.toJson();
      
      expect(json.containsKey('git_provider'), true);
      expect(json.containsKey('git_url'), true);
      expect(json.containsKey('created_at'), true);
      expect(json['git_provider'], 'github');
      expect(json['status'], 'active');
    });

    test('fromJson обрабатывает опциональное git_credential (присутствует)', () {
      final jsonWithCred = {...validJson};
      jsonWithCred['git_credential'] = {
        'id': 'cred-1',
        'provider': 'github',
        'auth_type': 'token',
        'label': 'My GitHub',
      };

      final project = ProjectModel.fromJson(jsonWithCred);
      expect(project.gitCredential, isNotNull);
      expect(project.gitCredential!.provider, 'github');
    });

    test('fromJson обрабатывает git_credential = null', () {
      final jsonWithNull = {...validJson};
      jsonWithNull['git_credential'] = null;

      final project = ProjectModel.fromJson(jsonWithNull);
      expect(project.gitCredential, isNull);
    });

    test('fromJson не падает если git_credential отсутствует в JSON (omitempty)', () {
      // Backend отправляет omitempty: если null/не установлен → ключ пропадает
      final jsonWithoutCred = {...validJson}..remove('git_credential');

      final project = ProjectModel.fromJson(jsonWithoutCred);
      expect(project.gitCredential, isNull);
    });

    test('copyWith создаёт новый экземпляр без изменения оригинала', () {
      final original = ProjectModel.fromJson(validJson);
      final updated = original.copyWith(status: 'paused');

      expect(updated.status, 'paused');
      expect(original.status, 'active');
    });
  });
}
```

      'model': 'claude-opus-4-7',
      'prompt_name': 'Developer',
      'code_backend': 'claude-code',
      'is_active': true,
    };

    test('fromJson десериализует все поля с правильными JSON-ключами', () {
      final agent = AgentModel.fromJson(validJson);
      
      expect(agent.id, 'agent-1');
      expect(agent.name, 'DevAgent');
      expect(agent.role, 'developer');
      expect(agent.model, 'claude-opus-4-7');
      expect(agent.promptName, 'Developer');
      expect(agent.codeBackend, 'claude-code');
      expect(agent.isActive, true);

    test('fromJson обрабатывает nullable поля (model, prompt_name, code_backend)', () {
      final jsonWithNulls = {
        'id': 'agent-2',
        'name': 'ReviewAgent',
        'role': 'reviewer',
        'model': null,
        'prompt_name': null,
        'code_backend': null,
        'is_active': true,
      };

      final agent = AgentModel.fromJson(jsonWithNulls);
      
      expect(agent.model, isNull);
      expect(agent.promptName, isNull);
      expect(agent.codeBackend, isNull);
    });

    test('fromJson преобразует is_active в isActive', () {
      final json = {...validJson};
      json['is_active'] = false;

      final agent = AgentModel.fromJson(json);
      expect(agent.isActive, false);
    });

    test('toJson сериализует в snake_case (prompt_name, code_backend, is_active)', () {
      final agent = AgentModel(
        id: 'a1',
        name: 'Agent',
        role: 'developer',
        model: 'claude-opus-4-7',
        promptName: 'Dev',
        codeBackend: 'claude-code',
        isActive: true,
      );

      final json = agent.toJson();
      
      expect(json.containsKey('prompt_name'), true);
      expect(json.containsKey('code_backend'), true);
      expect(json.containsKey('is_active'), true);
      expect(json['prompt_name'], 'Dev');
      expect(json['is_active'], true);
    });

    test('copyWith создаёт новый экземпляр без изменения оригинала', () {
      final original = AgentModel.fromJson(validJson);
      final updated = original.copyWith(isActive: false);

      expect(updated.isActive, false);
      expect(original.isActive, true);
      expect(updated.name, original.name); // не изменилось
    });
  });
}

## Частые ошибки и как их избежать

### ❌ Ошибка 1: Freezed v2 синтаксис при v3 в pubspec

**Неправильно:**
```dart
abstract class ProjectModel implements _$ProjectModel {  // ❌ implements
  const factory ProjectModel({ ... }) = _ProjectModel;
}
```

**Правильно:**
```dart
abstract class ProjectModel with _$ProjectModel {        // ✅ with
  const factory ProjectModel({ ... }) = _ProjectModel;

  const ProjectModel._();  // ✅ Приватный конструктор

  factory ProjectModel.fromJson(...) => _$ProjectModelFromJson(...);
}
```

### ❌ Ошибка 2: Забыли `@JsonKey(name: '...')` на snake_case полях

**Неправильно:**
```dart
required String gitProvider,  // ❌ JSON ключ: "gitProvider", но backend отдаёт "git_provider"
```

**Правильно:**
```dart
@JsonKey(name: 'git_provider')
required String gitProvider,  // ✅ JSON ключ: "git_provider"
```

**Результат ошибки:** `Exception: Exception: NoSuchMethodError: The getter 'git_provider' was not found`

### ❌ Ошибка 3: Даты как String вместо DateTime

**Неправильно:**
```dart
required String createdAt,  // ❌ Нужны парсинги везде: DateTime.parse(createdAt)
```

**Правильно:**
```dart
@JsonKey(name: 'created_at')
required DateTime createdAt,  // ✅ json_serializable парсит автоматически
```

**Бонус:** DateTime позволяет сортировать, сравнивать, форматировать напрямую.

### ❌ Ошибка 4: Required поле, которое backend может не отправить в JSON

**Неправильно:**
```dart
required String? model,  // ❌ fromJson упадёт если ключ в JSON отсутствует
```

**Правильно:**
```dart
String? model,  // ✅ Если ключ отсутствует → model = null
```

**Почему:** В Go tag `*string` без `omitempty` сериализуется как `"key": null`, а с `omitempty` — ключ пропадает совсем.

**Реальная ситуация в backend:**
- `AgentResponse` (team_dto.go:25-27) — **БЕЗ omitempty**:
  ```go
  Model       *string `json:"model"`         // nil → сериализуется как "model": null
  PromptName  *string `json:"prompt_name"`   // nil → сериализуется как "prompt_name": null
  CodeBackend *string `json:"code_backend"`  // nil → сериализуется как "code_backend": null
  ```
  **JSON всегда содержит эти ключи**, просто значение может быть `null`.

- `ProjectResponse` (project_dto.go:80) — **С omitempty только для git_credential**:
  ```go
  GitCredential *GitCredentialSummary `json:"git_credential,omitempty"`  // nil → ключ пропадает
  ```
  **JSON может не содержать** `git_credential` вообще.

**Поэтому:**
- Для `model`, `promptName`, `codeBackend` → `String?` (поле **всегда есть** в JSON, но может быть null)
- Для `gitCredential` → `GitCredentialModel?` (поле **может не быть** в JSON)
- С `required String? model` → `fromJson` упадёт если ключа нет. Но ключ **всегда есть**, поэтому можно даже `required` (но nullable предпочтительнее).
- С `String? model` → `json_serializable` подставит `null` для omitempty-полей и для null-значений (безопасно).

**Итог:** `required` и `?` совместимы в Dart 3, но для опциональных в JSON лучше без `required`, иначе отсутствие ключа сломает парсинг.

### ❌ Ошибка 5: Забыли генерировать код после изменения модели

**Результат:** Ошибки `The method 'fromJson' isn't defined` или `The class '_$ProjectModel' isn't defined`.

**Фикс:**
```bash
make frontend-codegen
```

### ❌ Ошибка 6: techStack как String вместо Map

Backend отдаёт JSON объект, а не строку:

**Неправильно:**
```dart
required String techStack,  // ❌ {"backend": "Go"} → не parse-сится в String
```

**Правильно:**
```dart
@JsonKey(name: 'tech_stack')
required Map<String, dynamic> techStack,  // ✅ {"backend": "Go", "frontend": "Flutter"}
```

### ❌ Ошибка 7: Выдуманные поля, которых нет в backend

**Неправильно:**
```dart
required String userId,           // ❌ ProjectResponse не содержит userId
required TeamModel team,          // ❌ ProjectResponse не содержит полную Team
@Default('Default Team') String name,  // ❌ но TeamResponse всегда имеет name (no default)
```

**Правильно:** Сверить с ProjectResponse и TeamResponse в backend. Оставить только поля, которые backend реально отдаёт.
