# 2.7 Handler: TeamHandler (настройка команды проекта)

**Sprint:** 2 — CRUD API для Project и Team
**Файл:** `backend/internal/handler/team_handler.go`
**Тест:** `backend/internal/handler/team_handler_test.go` (unit)
**Статус:** ⬜ Не начата

---

## Описание

Создать HTTP-обработчик для управления командой проекта. Команда — вложенный ресурс проекта (`/projects/:id/team`). Два эндпоинта: получение команды с агентами и обновление (переименование, управление составом агентов). Перед любым действием проверяется, что проект принадлежит пользователю (ABAC через ProjectService).

---

## Структура

```go
type TeamHandler struct {
    teamService    service.TeamService
    projectService service.ProjectService
}

func NewTeamHandler(teamService service.TeamService, projectService service.ProjectService) *TeamHandler
```

**Зависимость от `ProjectService`** — для проверки владения проектом (ABAC). Альтернатива: `TeamService` сам проверяет владение через `ProjectRepository`, но это дублирует логику.

---

## Эндпоинты

| Метод | Путь | Handler метод | Описание |
|-------|------|---------------|----------|
| GET | `/api/v1/projects/:id/team` | `GetByProjectID` | Получить команду с агентами |
| PUT | `/api/v1/projects/:id/team` | `Update` | Обновить команду |

Оба эндпоинта требуют авторизации (`AuthMiddleware`).

---

## Методы

### GetByProjectID (`GET /projects/:id/team`)

1. `uuid.Parse(c.Param("id"))` — projectID из path
2. `c.Get("userID")` → userID из JWT
3. `projectService.GetByID(ctx, userID, projectID)` — ABAC-проверка владения проектом
4. `teamService.GetByProjectID(ctx, projectID)` — получение команды с Preload Agents + Prompt
5. Ответ: `200 OK` + `TeamResponse`

**Маппинг ошибок:**
| Ошибка | HTTP | apierror code |
|--------|------|---------------|
| `ErrProjectNotFound` | 404 | `not_found` |
| `ErrProjectForbidden` | 403 | `forbidden` |
| `ErrTeamNotFound` | 404 | `not_found` |

### Update (`PUT /projects/:id/team`)

1. `uuid.Parse(c.Param("id"))` — projectID из path
2. `c.Get("userID")` → userID
3. `projectService.GetByID(ctx, userID, projectID)` — ABAC
4. `ShouldBindJSON(&dto.UpdateTeamRequest)` — парсинг body
5. `teamService.Update(ctx, projectID, req)` — обновление
6. Ответ: `200 OK` + `TeamResponse`

**Маппинг ошибок:**
| Ошибка | HTTP | apierror code |
|--------|------|---------------|
| `ErrProjectNotFound` | 404 | `not_found` |
| `ErrProjectForbidden` | 403 | `forbidden` |
| `ErrTeamNotFound` | 404 | `not_found` |
| binding error | 400 | `bad_request` |


## TeamService (минимальный)

Handler зависит от `TeamService`. Создаётся минимальный интерфейс:

```go
type TeamService interface {
    GetByProjectID(ctx context.Context, projectID uuid.UUID) (*models.Team, error)
    Update(ctx context.Context, projectID uuid.UUID, req dto.UpdateTeamRequest) (*models.Team, error)
}
```

**Реализация** — тонкая обёртка над `TeamRepository`:
- `GetByProjectID` → `teamRepo.GetByProjectID` (с Preload Agents + Prompt)
- `Update` → `teamRepo.GetByProjectID` + применить DTO + `teamRepo.Update`

**Файл:** `backend/internal/service/team_service.go`

## Конвенции

1. **ABAC через ProjectService** — проверка владения проектом до любого действия с командой
2. **`c.Get("userID")`** — извлечение userID из JWT
3. **`apierror.JSON()`** — для всех ошибок
4. **Swagger-аннотации** — без `/api/v1` в `@Router`
5. **DTO конвертеры** — `dto.ToTeamResponse()`, `dto.ToAgentResponse()`
